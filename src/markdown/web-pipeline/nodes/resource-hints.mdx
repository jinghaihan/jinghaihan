## 概念定位

资源预提示（Resource Hints）节点用于在浏览器“真正需要资源之前”提前做准备，减少关键请求等待时间。

它属于典型的“前置优化”：做对了缩短关键链路，做错了会浪费带宽与连接资源。

## 工作机制

1. 浏览器解析到预提示标签。
2. 根据提示类型执行 DNS、建连或资源预取。
3. 后续真实请求触发时复用已准备好的链路或副本。
4. 进入正常解析与渲染流程。

## 关键知识点

### 常见预提示类型对比

| 类型 | 作用 | 典型场景 | 风险 |
| --- | --- | --- | --- |
| `dns-prefetch` | 提前 DNS 解析 | 可能访问但不确定时机的跨域 | 误判会造成无效解析 |
| `preconnect` | 提前 DNS+TCP+TLS | 首屏关键第三方域 | 过多会抢连接池 |
| `preload` | 高优先级预加载当前页必需资源 | 首屏关键字体/CSS/脚本 | 类型不匹配会浪费下载 |
| `prefetch` | 低优先级预取未来可能资源 | 下一路由资源预热 | 命中不确定，易浪费流量 |
| `modulepreload` | 预加载模块依赖图 | 模块化前端应用 | 依赖变化需同步维护 |

### 使用优先级

- 优先保障“当前页面关键路径”而非未来可能路径。
- 先做少量高价值提示，再逐步扩展。

### 与缓存策略协同

预提示并不替代缓存策略，二者应配合：

- 预提示负责“更早开始”
- 缓存负责“重复复用”

## 对页面性能的影响

- 合理预提示可降低关键资源启动延迟。
- 过度预提示会抢占带宽和连接，反而拖慢关键请求。
- 弱网环境下误用 `prefetch` 成本更高。

## 观测与排障

### 观测要点

1. 看关键资源的请求启动时机是否提前。
2. 看预提示资源是否真正被后续请求命中。
3. 看是否出现连接池拥塞与无效下载。

### 排障步骤

1. 标定首屏关键资源，保留必要 `preload/preconnect`。
2. 审查 `prefetch` 命中率，移除低价值项。
3. 校验 `as` 属性与资源类型匹配。
4. 回归验证 FCP（First Contentful Paint）/LCP（Largest Contentful Paint）变化。

## 实战优化清单

- 关键跨域域名优先 `preconnect`。
- 首屏关键字体、关键 CSS 可用 `preload`。
- 未来路由资源用 `prefetch` 并持续评估命中率。
- 预提示配置纳入构建与发布检查。

## 常见误区

- “提示越多越快”：错误，过量提示会挤占真正关键资源。
- “`prefetch` 对当前页一定有收益”：错误，它主要面向未来导航。
- “`preload` 不用管类型”：错误，`as` 不匹配会导致重复下载或优先级异常。

## 面试补充

### 30 秒说明

资源预提示是把等待前移的优化手段。`preconnect` 适合关键跨域建连，`preload` 适合当前页必需资源，`prefetch` 适合未来导航预热。关键是控制提示数量和命中率，避免为了“提前加载”反而拖慢关键路径。

### 高频追问

1. `preload` 和 `prefetch` 最核心区别是什么？

`preload` 面向当前页面关键资源，优先级更高；`prefetch` 面向未来可能访问资源，优先级更低。

2. 为什么 `preconnect` 不能全站乱加？

会提前占用连接资源，过多时可能拖慢真正关键域请求。

3. 如何判断预提示是否有效？

看关键资源请求启动是否提前，以及预提示资源命中率是否稳定。

4. `modulepreload` 适合什么场景？

模块化依赖较深的应用，可提前拉取模块依赖减少执行前等待。

5. 预提示和缓存哪个优先？

两者协同，先保证缓存策略正确，再做高价值预提示。


## 补充视角（标准节点）

### 与上下游节点边界

- 上游节点（高频）：前端工程化实践、发布回归与性能门禁。
- 本节点重点：聚焦“资源预提示优化”本身的机制与决策，不替代下游渲染/监控职责。
- 下游节点（高频）：浏览器 DNS 缓存、TCP/TLS 建连、响应流接收。

### 落地动作清单

1. 先定义该节点的关键输入和关键输出，避免职责漂移。
2. 再绑定至少 1 个可观测指标，确保问题可量化定位。
3. 将该节点的异常处理写入动作库，确保值班时可执行。

### 高频追问

1. 这个节点优化应该先看什么？

先看它的关键输入是否稳定，再看关键输出是否满足下游依赖和时延预算。

2. 如何避免“优化该节点却拖慢其他节点”？

做上下游联动回归，对比关键链路分阶段指标，防止成本转移。

3. 这个节点如何进入发布门禁？

把它的核心指标加入灰度对比阈值，超阈值时触发暂停放量或回滚。
