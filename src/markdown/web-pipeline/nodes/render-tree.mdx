## 概念定位

渲染树是 DOM 与 CSSOM 合并后的“可渲染对象树”。它只保留需要参与显示的节点，为布局计算提供输入。

## 工作机制

1. 以 DOM 为结构基础，结合 CSSOM 计算样式。
2. 过滤不参与渲染的节点（如 `display: none`）。
3. 生成可渲染对象并组织成渲染树。
4. 输出给 Layout 阶段做几何计算。

## 关键知识点

- **不是 DOM 原样复制**：两者不一一对应。
- **样式计算成本高**：选择器复杂度与节点数都会放大开销。
- **继承与层叠**：决定最终样式结果。

## 对页面性能的影响

- 节点过多或选择器过重会拖慢样式计算。
- 频繁改类名/内联样式会触发渲染树更新。
- 大规模动态内容会增加渲染树重建压力。

## 观测与排障

### 浏览器侧

- Performance 观察 Style Recalculation 时间占比。
- Rendering 工具辅助定位频繁样式失效区域。

## 常见误区

- “所有 DOM 节点都会进入渲染树”：不准确，不可见节点会被剔除。
- “样式计算只是小成本”：不准确，在复杂页面中常是大头。

## 面试补充

### 30 秒说明

渲染树是浏览器把 DOM 结构和 CSS 样式融合后的可绘制模型。它决定哪些元素真正参与布局和绘制，是渲染流水线的中枢节点。

### 高频追问

1. `display: none` 和 `visibility: hidden` 在渲染树中的差异？

`display: none` 通常不进入渲染树；`visibility: hidden` 仍占位并参与布局，只是不显示。

2. 为什么复杂 CSS 选择器会影响性能？

浏览器需要在大量节点上做匹配与回溯，复杂选择器会增加样式计算时间，进而拖慢渲染树构建。
