## 概念定位

CDN（Content Delivery Network，内容分发网络）边缘节点用于把资源尽量“放到离用户更近的位置”，减少回源链路与跨地域时延。

它是 Web 性能和稳定性的关键放大器：配置正确时大幅提速，配置错误时会放大故障。

## 工作机制

1. 用户请求先到 CDN 边缘节点。
2. 边缘节点按缓存键查找资源是否命中。
3. 命中直接返回，未命中则回源获取并回填缓存。
4. 按 TTL（Time To Live）和失效策略维护副本新鲜度。

## 关键知识点

### 缓存键设计

缓存键通常由路径、查询参数、协议、设备类型等维度组成。键过细会命中碎片化，键过粗会内容错配。

### 回源链路

- 回源性能受源站容量、网络路径、压缩与协议配置影响。
- 回源异常会在边缘放大成大面积慢请求或 5xx。

### 边缘能力扩展

- 边缘重写与重定向
- 边缘压缩与图片格式协商
- 边缘安全防护（WAF、限速）

## 对页面性能的影响

- 命中边缘可显著降低 TTFB（Time To First Byte）。
- 高回源率会导致尾延迟与波动上升。
- 边缘规则不一致可能导致局部地区体验异常。

## 观测与排障

### 关键指标

- 边缘命中率
- 回源率与回源带宽
- 边缘 4xx/5xx
- 各地域 P75/P95 TTFB

### 排障步骤

1. 先确认问题是全局还是局部地域。
2. 检查命中率变化与缓存键是否发生变更。
3. 检查回源耗时、源站健康和限流策略。
4. 对比规则版本，排查边缘配置漂移。

## 实战优化清单

- 关键静态资源开启长缓存与预热。
- 动态接口谨慎缓存，明确缓存键与失效策略。
- 发布前验证多地域边缘规则一致性。
- 将 CDN 指标纳入发布门禁。

## 常见误区

- “上了 CDN 就一定快”：错误，命中率和回源质量决定真实收益。
- “缓存键越细越安全”：错误，容易把命中率打散。
- “CDN 只管静态文件”：错误，动态加速和边缘策略同样常见。

## 面试补充

### 30 秒说明

CDN 边缘节点通过就近缓存和回源控制降低请求延迟。优化核心是缓存键设计、命中率治理和回源链路稳定性。上线时必须关注多地域一致性，否则容易出现“部分地区慢或异常”的问题。

### 高频追问

1. 如何判断 CDN 是否真正生效？

看命中率、回源率和 TTFB 分位数是否同步改善。

2. 命中率高但用户仍慢，可能是什么问题？

可能是边缘节点到用户链路差、连接问题或关键资源未命中。

3. 缓存键设计的核心原则是什么？

既能区分内容差异，又不把请求维度切得过碎。

4. CDN 故障为什么常呈地域性？

边缘节点分布和运营商路径差异导致问题先在局部区域暴露。

5. 如何降低回源峰值风险？

预热热点资源、合理 TTL、回源限流和多级缓存协同。


## 补充视角（标准节点）

### 与上下游节点边界

- 上游节点（高频）：磁盘缓存检查、缓存策略优化。
- 本节点重点：聚焦“CDN 边缘节点”本身的机制与决策，不替代下游渲染/监控职责。
- 下游节点（高频）：响应流接收、浏览器 DNS 缓存。

### 落地动作清单

1. 先定义该节点的关键输入和关键输出，避免职责漂移。
2. 再绑定至少 1 个可观测指标，确保问题可量化定位。
3. 将该节点的异常处理写入动作库，确保值班时可执行。

### 高频追问

1. 这个节点优化应该先看什么？

先看它的关键输入是否稳定，再看关键输出是否满足下游依赖和时延预算。

2. 如何避免“优化该节点却拖慢其他节点”？

做上下游联动回归，对比关键链路分阶段指标，防止成本转移。

3. 这个节点如何进入发布门禁？

把它的核心指标加入灰度对比阈值，超阈值时触发暂停放量或回滚。
