## 概念定位

系统 DNS 缓存是操作系统层面的域名解析缓存。浏览器缓存未命中时，会继续尝试命中系统层缓存记录。

## 工作机制

1. 浏览器向系统 resolver 发起域名解析请求。
2. 系统检查本地 DNS 缓存（受 TTL（Time To Live） 约束）。
3. 命中则直接返回 IP。
4. 未命中则继续查 `hosts` 或向递归 DNS 发起请求。

## 关键知识点

- **跨浏览器共享**：同机多个应用可受益于系统缓存。
- **缓存刷新**：受系统策略和网络切换影响。
- **DNS Client 服务**：不同系统实现不同，但核心逻辑一致。

## 对页面性能的影响

- 可避免重复外部 DNS 查询，降低建连前等待。
- 对多浏览器或多应用并发访问同域收益明显。
- 网络切换后缓存失配会引发额外解析开销。

## 观测与排障

### 命令行

```sh
# macOS
sudo dscacheutil -flushcache
# Windows
ipconfig /displaydns
ipconfig /flushdns
```

### 浏览器侧

- 对比 flush 前后同域名首次请求的 DNS 耗时。

## 常见误区

- “只要重启浏览器就能清系统 DNS 缓存”：不准确，系统缓存独立存在。
- “系统 DNS 缓存一定比浏览器缓存慢”：不一定，命中后都很快。

## 面试补充

### 30 秒说明

系统 DNS 缓存是浏览器缓存后的第二层命中点。它由操作系统维护，可跨应用复用，命中时同样能省去外部 DNS 往返。

### 高频追问

1. 为什么同一台机器不同浏览器访问同域都很快？

可能是系统 DNS 缓存生效，多个浏览器都通过系统 resolver 共享了域名到 IP 的映射。

2. 清 DNS 缓存时应该清哪一层？

排障时通常要分层看：浏览器层、系统层都可能命中；只清一层可能仍看到旧解析结果。
