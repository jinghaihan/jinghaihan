## 概念定位

顶级域（TLD）服务器负责管理某个顶级域下的委派信息。它在 DNS 链路中承接根服务器，继续把查询引导到具体权威服务器。

## 工作机制

1. 递归 DNS 根据根返回线索访问对应 TLD 服务器。
2. TLD 返回目标二级域的权威 NS 记录。
3. 递归 DNS 再向权威服务器查询最终资源记录。

## 关键知识点

- **委派关系**：TLD 管理“谁是权威”，不直接管理业务主机记录。
- **NS 与 TTL（Time To Live）**：决定中间结果缓存时间和切换收敛速度。
- **DNS 分层职责**：层次越高，信息越抽象。

## 对页面性能的影响

- 首次冷解析会多一跳查询延迟。
- TLD 线索通常可被递归缓存，后续访问成本低。
- 委派配置错误会导致整域不可解析。

## 观测与排障

### 命令行

```sh
dig NS example.com
# 或完整链路
 dig +trace example.com
```

## 常见误区

- “TLD 服务器能直接给我业务域名 A 记录”：通常不能，它返回权威服务器线索。
- “换权威 DNS 后会立刻全球生效”：不准确，受各层缓存 TTL 影响。

## 面试补充

### 30 秒说明

TLD 服务器在 DNS 链路中承担“中间委派”职责：告诉递归 DNS 该去哪个权威服务器查最终记录。它是根与权威之间的关键转发层。

### 高频追问

1. 为什么改 NS 后用户还在访问旧解析？

因为根/TLD/递归层都可能缓存旧委派，必须等待相关 TTL 逐步过期。

2. TLD 返回的是什么类型记录？

核心是 NS 委派信息，常伴随必要的附加记录用于加速下一跳查询。
