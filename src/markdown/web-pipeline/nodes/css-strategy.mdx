## 概念定位

CSS 架构与渲染策略是 `cssom-js -> layout -> paint` 的汇总节点。目标是让样式系统在可维护的同时，控制选择器成本、布局成本和重绘成本。

## 工作机制

1. 先定义样式分层：基础层、组件层、工具层、覆盖层。
2. 控制选择器复杂度和作用域，减少样式重算开销。
3. 按关键路径拆分 CSS：首屏关键样式优先、非关键样式延后。
4. 对高频动画/交互区域制定“合成友好”样式策略。
5. 通过可视化工具定位重绘热点和布局热点，持续修正。

## 关键知识点

- **样式重算成本**：复杂选择器和大范围失效会放大 Recalculate Style 成本。
- **布局稳定性**：尺寸未预留、晚到字体会导致布局波动。
- **绘制复杂度**：阴影、滤镜、渐变、大面积透明层会增加 Paint 成本。
- **层级治理**：z-index 和图层策略应可控，避免无序叠加。
- **CSS 体积治理**：无用样式会拖慢下载、解析、执行多个阶段。

## 对页面性能的影响

- CSS 架构合理可显著降低样式重算与回流扩散。
- 首屏关键样式策略直接影响 LCP。
- 动画和滚动场景中，CSS 策略决定掉帧风险。

## 观测与排障

### 浏览器侧

- Coverage 看未使用 CSS 比例。
- Performance 看 `Recalculate Style` 与 `Paint` 占比。
- Paint Flashing 定位高频重绘区域。

### 排障步骤

1. 先清理无用样式，降低首屏必须解析体积。
2. 优化选择器层级，减少跨层级大范围匹配。
3. 为动态内容预留尺寸，降低 CLS。
4. 动画改用 `transform/opacity`，减少布局/绘制参与。
5. 回归验证 LCP、CLS、FPS（Frames Per Second）变化。

## 常见误区

- “CSS 问题只影响视觉，不影响性能”：不准确，样式系统直接影响主线程预算。
- “选择器越精确越好”：不绝对，过深链路会增加匹配成本。
- “视觉效果都可以直接上”：不准确，复杂效果需要按设备分级。

## 面试补充

### 30 秒说明

CSS 性能优化的核心是三件事：降样式重算成本、控布局稳定性、减重绘面积。先从架构和关键样式拆分入手，再针对交互热点做合成友好优化。

### 高频追问

1. 为什么“减少选择器复杂度”有意义？

浏览器样式匹配需要在大量节点上运行，复杂链式选择器会放大匹配与回溯成本。

2. 字体加载会影响哪些指标？

字体晚到可能造成文本重排，影响 CLS，也可能推迟主要内容稳定呈现，影响 LCP 体感。

3. CSS 优化如何和组件化协同？

通过样式作用域约束、分层规则和命名约定，让组件可复用同时避免全局污染和重算扩散。
