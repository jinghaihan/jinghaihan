## 概念定位

`hosts` 文件是系统级静态域名映射表。该节点表示在走外部 DNS 前，先看本地是否有人工指定的域名到 IP 映射。

## 工作机制

1. 系统 resolver 读取 `hosts` 中的域名映射。
2. 若命中，直接返回映射 IP，跳过外部 DNS 查询。
3. 未命中才进入递归 DNS 解析流程。
4. 命中结果通常不受 DNS TTL（Time To Live） 约束，而受本地文件内容控制。

## 关键知识点

- **最高优先级本地覆盖**：常用于开发联调与灰度验证。
- **静态映射**：不会自动随线上 DNS 变更更新。
- **生效范围**：作用于该机器上依赖系统解析的请求。

## 对页面性能的影响

- 命中时可省去 DNS 递归流程。
- 配错 IP 会直接导致访问异常或证书校验失败。
- 长期遗留映射可能造成“本机与线上行为不一致”。

## 观测与排障

### 命令行

```sh
# 查看 hosts（macOS/Linux）
cat /etc/hosts
# Windows
notepad C:\\Windows\\System32\\drivers\\etc\\hosts
```

### 浏览器侧

- 当访问结果与同事机器不同，优先检查 hosts 覆盖。

## 常见误区

- “nslookup 一定反映 hosts 结果”：不总是，很多场景它直接查 DNS 服务器。
- “hosts 只影响浏览器”：不准确，系统解析路径上的应用都可能受影响。

## 面试补充

### 30 秒说明

`hosts` 是本地静态解析覆盖层，优先级高于外部 DNS。命中时可直接得到 IP，但也最容易引入环境偏差和隐蔽性问题。

### 高频追问

1. 为什么我本机能访问，其他人不行？

典型原因是你机器存在 hosts 映射，把域名指向了特定测试 IP，而其他机器走的是线上 DNS 结果。

2. hosts 改完不生效怎么办？

先确认文件路径和权限，再清浏览器/系统 DNS 缓存，必要时重启相关网络服务或浏览器进程。
