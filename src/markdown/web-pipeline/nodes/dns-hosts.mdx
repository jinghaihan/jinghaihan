## 概念定位

Hosts 文件匹配是 DNS 递归前的本地静态映射层，常用于开发联调、灰度验证和临时路由覆盖。

## 工作机制

1. 系统 resolver 在外部 DNS 前读取 hosts 映射。
2. 命中时直接返回指定 IP，不再继续 DNS 分层查询。
3. 未命中时才进入递归 DNS 路径。
4. hosts 生效通常不依赖 TTL，依赖本地文件配置与缓存刷新。

## 关键知识点

- **优先级高**：可覆盖线上权威 DNS 结果。
- **静态性强**：不会自动随线上变更更新。
- **风险可隐蔽**：本机“看似正常”但与线上真实环境不一致。
- **排障第一步**：同域行为分歧时优先排查 hosts。

## 对页面性能的影响

- 命中 hosts 可省去 DNS 查询链，解析阶段更快。
- 错误映射会导致建连失败、证书不匹配、地区路由异常。
- 遗留测试映射常导致“只有我机器有问题”。

## 观测与排障

### 检查命令

```sh
# macOS / Linux
cat /etc/hosts
# Windows
notepad C:\\Windows\\System32\\drivers\\etc\\hosts
```

### 排障步骤

1. 比对团队机器 hosts 是否一致。
2. 检查是否遗留历史测试域名映射。
3. 清浏览器与系统 DNS 缓存后重试。
4. 对比权威 DNS 结果和本机访问结果差异。
5. 建立 hosts 变更登记，避免长期漂移。

## 常见误区

- “nslookup 一定走 hosts”：不准确，很多实现直接查 DNS 服务器。
- “hosts 只影响浏览器”：不准确，依赖系统解析的应用都会受影响。
- “改完 hosts 立即全生效”：不一定，缓存层可能还在复用旧值。

## 面试补充

### 30 秒说明

hosts 是本地静态覆盖层，优先级高于外部 DNS。它很适合调试，也很容易造成环境偏差。线上问题排障时，要把 hosts 检查作为第一步。

### 高频追问

1. 为什么我本机正常，其他人不正常？

常见是你本机 hosts 把域名指到了特殊 IP，而其他人走的是线上 DNS 结果。

2. hosts 覆盖会影响 HTTPS 吗？

会。若映射 IP 对应证书与域名不匹配，TLS 握手会失败。

3. 如何降低 hosts 误配置风险？

建立最小化使用原则、变更记录和定期清理流程。
