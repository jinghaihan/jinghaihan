## 概念定位

Paint 阶段负责把布局结果转成像素绘制指令，包括文本、颜色、边框、阴影、图片等视觉内容。

## 工作机制

1. 浏览器根据布局结果确定需要绘制的可见区域。
2. 生成绘制命令（paint records）并提交给栅格化流程。
3. 发生样式变化时触发重绘区域更新。
4. 结果交给合成阶段进行图层组合。

## 关键知识点

- **重绘（Repaint）**：仅视觉变化，不一定触发布局。
- **无效区域（Invalidation）**：决定重绘范围大小。
- **绘制复杂度**：阴影、滤镜、渐变等会增加成本。

## 对页面性能的影响

- 大面积重绘会导致帧率下降。
- 高频动画若落在 Paint 阶段，容易出现卡顿。
- 复杂视觉效果在低性能设备上代价明显。

## 观测与排障

### 浏览器侧

- 开启 Paint Flashing 查看重绘区域。
- Performance 面板关注 Paint 耗时与频率。

## 常见误区

- “只要没回流就没性能问题”：不准确，重绘本身也可能很重。
- “所有样式动画都一样快”：不准确，触发 Paint 的动画通常更贵。

## 面试补充

### 30 秒说明

Paint 是把布局后的元素真正画到位图上的阶段。它不一定涉及几何重算，但大面积或高频重绘依然会拖垮帧率。

### 高频追问

1. 为什么动画推荐用 `transform`/`opacity`？

它们更容易在合成阶段处理，能减少 Layout/Paint 参与，通常更平滑。

2. 如何判断页面是否有过度重绘？

用 Paint Flashing 看可视化重绘范围，再结合 Performance 分析高耗时绘制任务和触发来源。
