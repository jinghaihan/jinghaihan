## 概念定位

合成阶段把多个图层按顺序混合输出到屏幕，是渲染流水线的最终展示环节。

## 工作机制

1. 浏览器按策略把页面拆分为多个合成层。
2. 合成线程按层级、透明度、变换关系做图层混合。
3. 生成最终帧并提交 GPU 显示。
4. 下一帧重复该流程。

## 关键知识点

- **图层提升**：可减少主线程重绘参与。
- **GPU 合成**：适合处理位移、缩放、透明度动画。
- **内存开销**：图层越多，占用越大。

## 对页面性能的影响

- 合理图层可显著提升动画流畅度。
- 过度图层化会增加内存和合成调度成本。
- 图层抖动会导致不稳定帧时间。

## 观测与排障

### 浏览器侧

- Layers 面板查看图层数量与层级关系。
- Performance 观察 Composite Layers 阶段耗时。

## 常见误区

- “图层越多越好”：错误，过多图层会引发额外成本。
- “用了 `will-change` 一定更快”：不一定，滥用会导致显存压力。

## 面试补充

### 30 秒说明

Composite 是渲染输出的最后一步。它把各图层交给合成线程/GPU 混合成最终画面。优化重点是合理图层化，而不是盲目提升图层数量。

### 高频追问

1. `will-change` 什么时候该用？

用于明确即将发生动画/变化的少量关键元素，临时提升优化机会；不该长期全局滥用。

2. 为什么 `top/left` 动画常比 `transform` 卡？

`top/left` 更容易触发布局与重绘；`transform` 常可在合成阶段直接处理，主线程压力更小。
