## 概念定位

合成（Composite）阶段是渲染链路最后一步，负责把多个图层按顺序混合并输出到屏幕，直接决定动画和滚动的帧稳定性。

## 工作机制

1. 浏览器根据策略把页面内容拆分为若干图层。
2. 合成线程按层级、透明度、变换关系进行图层混合。
3. 生成最终帧并提交 GPU（Graphics Processing Unit） 显示。
4. 下一帧重复调度，形成连续视觉输出。

## 关键知识点

- **图层提升**：让高频变化元素更靠近合成阶段处理。
- **图层数量控制**：层太少难优化，层太多会增加内存和调度成本。
- **合成友好动画**：`transform`、`opacity` 更容易稳定在合成链路。
- **帧稳定性**：平均帧率高不代表卡顿少，关键看长尾帧。

## 对页面性能的影响

- 合理图层化可显著改善滚动与动画流畅度。
- 过度图层化会提高显存占用，导致反向卡顿。
- 合成链路抖动会直接表现为交互不跟手。

## 观测与排障

### 浏览器侧

- Layers 面板查看图层数量、边界和层级关系。
- Performance 看 Composite 阶段时长与帧分布。
- 结合 FPS（Frames Per Second）和 Long Task 观察卡顿根因。

### 排障步骤

1. 定位需要高频更新的关键元素。
2. 仅对关键元素做图层优化，避免全局提升。
3. 清理冗余 `will-change`，减少无效图层常驻。
4. 观察显存和帧时间，找到图层数量平衡点。
5. 回归验证滚动和动画的长尾帧改善。

## 常见误区

- “图层越多越快”：错误，过量图层会带来管理和内存负担。
- “合成阶段不占主线程”：不准确，调度与提交仍会影响整体时序。
- “只要 60 FPS 就万事大吉”：不准确，用户感知常由卡顿尖峰决定。

## 面试补充

### 30 秒说明

Composite 阶段是最终出帧环节。优化核心不是盲目加图层，而是让关键交互元素进入合成友好路径，并控制图层数量和显存开销，保障帧稳定性。

### 高频追问

1. 为什么 `will-change` 不能全局滥用？

它会提前申请优化资源，滥用会增加显存和调度成本，反而可能造成性能回退。

2. 合成优化和布局优化怎么配合？

先减少不必要布局与重绘，再把高频动画留在合成阶段，才能形成稳定收益。

3. 如何判断卡顿来自合成还是脚本？

看 Performance：若 Script/Long Task 高则偏脚本问题；若 Composite 峰值和帧抖动高则偏图层与合成链路问题。
