## 概念定位

内存缓存是浏览器进程内最快的资源复用层。该节点表示请求先尝试从运行期内存中直接读取资源副本。

## 工作机制

1. 根据请求 URL、方法、Vary 维度等生成缓存匹配键。
2. 检查资源是否仍在有效期内（`Cache-Control`/`Expires`）。
3. 命中且新鲜时直接返回，不进入磁盘与网络。
4. 未命中或已过期时进入下一层缓存或协商验证流程。

## 关键知识点

- **速度极快**：省去磁盘 IO 与网络 RTT（Round-Trip Time）。
- **生命周期短**：通常随标签页/进程生命周期结束而失效。
- **受策略约束**：并非所有响应都可进入内存缓存。

## 对页面性能的影响

- 首次加载后同会话复访可明显降低资源读取延迟。
- 热路径资源命中内存缓存时，首屏稳定性更好。
- 内存压力高时缓存被回收，会导致性能波动。

## 观测与排障

### 浏览器侧

- Network 面板查看资源是否显示来自 memory cache。
- 对比同会话首次/二次访问的请求耗时差异。
- 观察在高内存占用场景下命中率是否下降。

## 常见误区

- “内存缓存和 BFCache（Back-Forward Cache） 是同一件事”：不是，BFCache 缓的是页面快照状态。
- “清空硬盘缓存会清掉内存缓存”：不完全等价，行为依浏览器而异。

## 面试补充

### 30 秒说明

内存缓存是最前置、最快速的浏览器缓存层。命中时几乎可以零成本返回资源，但它生命周期短，容易受进程回收和内存压力影响。

### 高频追问

1. 内存缓存与磁盘缓存最核心的区别是什么？

内存缓存更快但不持久，通常仅在当前运行期有效；磁盘缓存更持久但存在 IO 成本。

2. 为什么有时二次访问还是没命中内存缓存？

可能是资源不满足缓存条件、缓存已过期、进程重启或内存回收导致条目被淘汰。
