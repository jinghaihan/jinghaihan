## 概念定义

- 本质定义：Promise 与 `async/await` 是 JavaScript 异步编排与错误传播的核心模型。
- 解决问题：解决回调地狱、链式流程控制、并发组合和异常兜底问题。
- 边界说明：不处理渲染线程调度与网络缓存策略。

## 核心机制拆解

### 1) 机制/流程

1. Promise 通过 `pending/fulfilled/rejected` 管理状态。
2. `then/catch/finally` 形成可组合链。
3. `async` 函数返回 Promise，`await` 让异步流程线性表达。
4. 并发场景通过 `all/race/allSettled/any` 建模。

### 2) 关键条件与触发点

- 条件 A：`Promise.all` 任一失败会整体失败。
- 条件 B：`await` 需要 `try/catch` 或上层统一捕获异常。

### 3) 结果与影响

- 对可维护性的影响：流程更直观，错误链更可控。
- 对稳定性的影响：无统一异常处理会产生未捕获拒绝。

## 相似概念对比

| 维度 | Promise 链 | async/await | 结论 |
|---|---|---|---|
| 可读性 | 链式 | 同步风格 | 复杂流程更偏向 `async/await` |
| 并发表达 | 组合方法丰富 | 需配合 Promise 组合 | 不是替代关系 |
| 错误处理 | `catch` 链 | `try/catch` | 都可统一兜底 |
| 调试难度 | 链较长时偏高 | 断点更直观 | 团队可混合使用 |

## 代码

```ts
async function queryUserAndPosts(userId: string) {
  try {
    const [user, posts] = await Promise.all([
      fetch(`/api/users/${userId}`).then(r => r.json()),
      fetch(`/api/users/${userId}/posts`).then(r => r.json()),
    ])
    return { user, posts }
  }
  catch (err) {
    return { user: null, posts: [], error: String(err) }
  }
}
```

### 示例说明

- 示例展示了并发聚合和统一异常处理。
- `Promise.all` 适合“全成功才能继续”的场景。
- 常见错误是只写 `await` 不处理异常，导致未捕获拒绝。

## 相关面试题

1. 对 Promise 的理解
- 标准答案：Promise 是异步结果容器，状态只能从 `pending` 单向变为 `fulfilled` 或 `rejected`。它通过链式调用实现流程组合和错误透传。
- 考察点：是否理解状态机模型。

2. Promise 的基本用法
- 标准答案：通过构造函数创建 Promise，使用 `then/catch/finally` 处理结果；组合场景使用 `all`、`race`、`allSettled` 等静态方法。
- 考察点：是否具备组合能力而非只会单个 Promise。

3. Promise 解决了什么问题
- 标准答案：主要解决回调地狱和错误处理分散问题，提供可组合的异步流程表达。它并不自动消除并发竞态，需要额外控制策略。
- 考察点：是否能说清“解决了什么、没解决什么”。

4. Promise.all 和 Promise.race 的区别的使用场景
- 标准答案：`all` 适合依赖全成功场景，`race` 适合超时抢占或首响应策略。选择依据是“要全量结果还是首个结果”。
- 考察点：是否能按业务语义选方法。

5. 对 async/await 的理解
- 标准答案：`async/await` 是 Promise 语法糖，让异步流程以同步风格书写。底层仍基于 Promise 和微任务调度。
- 考察点：是否理解语法糖本质。

6. async/await 如何捕获异常
- 标准答案：推荐 `try/catch` 包裹关键 `await` 片段，或在外层统一捕获并上报。并发场景下可结合 `allSettled` 降低整体失败概率。
- 考察点：是否有工程级异常治理意识。

### 代码输出题

1. 代码输出结果
```js
const promise = new Promise((resolve, reject) => {
  console.log(1);
  console.log(2);
});
promise.then(() => {
  console.log(3);
});
console.log(4);
```
- 输出结果：
```text
1
2
4
```
- 关键机制：Promise 状态一旦变为 `fulfilled/rejected` 就不可逆，`then/catch/finally` 都返回新 Promise；非函数参数会发生值透传。
- 边界条件：若返回 Promise 自身会触发链路循环错误；未捕获拒绝会出现 `Uncaught (in promise)`。
