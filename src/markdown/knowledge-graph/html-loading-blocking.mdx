## 概念定义

- 本质定义：加载与阻塞模型描述浏览器在解析 HTML（HyperText Markup Language）时，如何发现资源、建立依赖关系，并在必要时暂停解析或推迟渲染。
- 解决问题：解释“白屏为什么长”“资源为什么到了但页面还没出来”“标签位置为什么会影响首屏”。
- 边界说明：本节点关注资源关系和阻塞链路，不展开 `defer` / `async` 的执行策略细节。

## 核心机制拆解

### 1) 机制/流程

1. 浏览器一边解析文档，一边发现 `link`、`script`、`img`、`iframe`、`a` 等标签上的资源引用。
2. `href` 更偏“建立关系或跳转目标”，常见于样式表和链接；`src` 更偏“把资源嵌入当前位置”，常见于脚本、图片和 `iframe`。
3. 同步脚本会暂停后续 HTML 解析，某些样式资源会推迟可见渲染。
4. 文档结构和关键资源稳定后，页面才真正进入可见和可交互阶段。

### 2) 关键条件与触发点

- 条件 A：把同步脚本放在首屏关键路径上，会直接拉长解析阻塞时间。
- 条件 B：样式表过大或关键样式加载过慢时，页面会出现“资源到了但内容迟迟不显示”的情况。
- 条件 C：误把 `src` 和 `href` 当成同一种“资源地址”，会导致语义和加载行为一起写错。

### 3) 结果与影响

- 对性能/稳定性/可维护性的影响：资源关系写对了，首屏优化更有抓手；资源关系写乱了，后面很容易靠经验碰运气。

## 相似概念对比

| 维度 | `src` | `href` | 结论 |
|---|---|---|---|
| 语义 | 把资源嵌入当前文档或上下文 | 建立关系或定义跳转目标 | 不能混用 |
| 常见标签 | `script`、`img`、`iframe` | `link`、`a` | 取决于标签职责 |
| 对页面的影响 | 可能影响解析和资源执行时机 | 常影响样式关系或导航跳转 | 阻塞点不同 |
| 适用场景 | 脚本、图片、嵌入内容 | 样式表、站内外链接 | 先看语义，再看性能 |

## 代码

```html
<head>
  <link rel="stylesheet" href="/assets/main.css" />
</head>
<body>
  <a href="/detail/1001">查看详情</a>
  <script src="/assets/main.js"></script>
</body>
```

### 示例说明

- 这段代码解决什么问题：展示 `href` 用于关系和跳转，`src` 用于资源嵌入，同时把同步脚本挪到文档尾部以减少阻塞。
- 关键语句或参数含义：`a + href` 表示导航目标；`link + href` 表示样式关系；`script + src` 表示脚本资源。
- 常见错误写法：把首屏并不需要的同步脚本堆在 `head` 里，或者把 `href` / `src` 的职责讲成一回事。

## 真实场景示例

- 场景描述：活动页接口并不慢，但白屏时间很长，且偶尔出现 DOM（Document Object Model）找不到节点的报错。
- 问题表现：样式迟迟不生效，脚本在元素还没解析出来之前就执行。
- 处理方案：梳理关键样式和同步脚本位置，减少首屏阻塞，把非关键逻辑后移或改成更合适的脚本策略。
- 验证方式：对比 FCP（First Contentful Paint）、长任务数量和首屏可见时间。

## 常见误区

- 误区：`src` 和 `href` 都是地址字段，随便替换没区别。
- 正解：它们表达的是不同关系，写法改了，浏览器行为也会跟着变。
- 误区：脚本下载很快，就不会影响首屏。
- 正解：问题不只是下载速度，解析暂停和执行时机同样会拖慢页面。
- 误区：所有 `script` 都该丢到 `head`。
- 正解：默认同步脚本放在 `head` 很容易阻塞，除非它真的必须在最前面执行。

## 关联知识点

- 依赖：文档声明与 head 配置。
- 影响：script 加载策略（defer / async）、媒体与图形、浏览器解析链路。
- 对比：解析阻塞与渲染阻塞。
- 组合：资源优先级、懒加载、预加载。

## 相关面试题

1. src 和 href 的区别
- 标准答案：`src` 表示把资源嵌入当前文档位置，比如脚本、图片、`iframe`；`href` 表示建立关系或声明跳转目标，比如样式表和链接。机制上两者语义不同，所以浏览器的加载和处理阶段也不同。场景上 `script`、`img` 常用 `src`，`link`、`a` 常用 `href`。
- 考察点：是否能把语义差异和浏览器行为联系起来。

2. HTML中如何创建链接？
- 标准答案：标准做法是使用 `a` 标签配 `href`。机制上 `href` 可以指向站内路径、外部地址、锚点、邮件地址或电话协议。场景上新窗口跳转还要结合 `target="_blank"` 与 `rel="noopener noreferrer"` 一起使用，避免安全和性能问题。
- 考察点：是否掌握链接的基础写法和安全细节。

3. script 标签位置的性能影响：为什么建议写在 body 末尾？
- 标准答案：因为默认同步 `script` 在解析到标签时会暂停后续 HTML 解析，放在 `body` 末尾可以让浏览器先把主要结构和样式跑出来。机制上这能减少白屏时间，也能避免脚本执行时 DOM 还没生成。场景上如果脚本没有必须前置的理由，后移或改用 `defer` / `async` 都比把它直接堆在 `head` 里更稳妥。
- 考察点：是否理解脚本位置、解析时序和用户感知性能之间的关系。
