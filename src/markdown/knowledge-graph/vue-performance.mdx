## 代码

```vue
<script setup lang="ts">
import { computed, defineAsyncComponent, ref } from 'vue'

const HeavyPanel = defineAsyncComponent(() => import('./HeavyPanel.vue'))
const keyword = ref('')
const list = ref<string[]>([])

const filtered = computed(() =>
  list.value.filter(item => item.includes(keyword.value)),
)
</script>

<template>
  <input v-model="keyword" />
  <HeavyPanel v-if="keyword.length > 0" :items="filtered" />
</template>
```

### 示例说明

- 这段代码解决什么问题：把首屏低频模块拆分懒加载，降低初始包体积。
- 关键语句或参数含义：`defineAsyncComponent` 延迟加载重量组件，`computed` 避免模板重复计算。
- 常见错误写法：所有组件首屏同步加载，导致白屏时间和首包体积上升。

## 相关面试题

1. Vue 的优点
- 标准答案：结论是优势主要有上手成本低、响应式开发体验好、生态完善。机制上模板编译 + 依赖追踪降低手写 DOM 与状态同步成本。场景上中后台和内容型应用能快速形成稳定交付。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

2. Vue 的性能优化有哪些
- 标准答案：结论是要分层优化：构建层（分包、懒加载）、渲染层（稳定 key、减少深层响应式）、交互层（节流防抖、虚拟列表）。机制上目标是减少无效渲染和大对象追踪。场景上先用性能面板定位热点，再做最小改动优化。
- 考察点：是否能完整列举关键项，并说明每一项的适用场景。

3. vue 初始化页面闪动问题
- 标准答案：结论是闪动通常来自首屏样式晚到或数据占位不稳定。机制上 CSS 未及时生效、骨架屏缺失或 hydration 前后不一致都会导致闪烁。场景上可通过关键 CSS 内联、骨架屏与首屏数据预取治理。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

5. Vue 3 相比 Vue 2 在性能上做了哪些核心优化？
- 标准答案：结论是 Vue 3 在编译优化、运行时调度和响应式实现上都有提升。机制上包含静态提升、Patch Flag、更快的 Proxy 响应式与更高效的更新路径。场景上复杂模板和中大型组件树在 Vue 3 下通常能获得更稳定吞吐。
- 考察点：是否能把“编译期优化 + 运行时优化”分层讲清楚。

6. 在日常 Vue 3 开发中，有哪些常用的性能优化技巧？
- 标准答案：结论是常用技巧包括稳定 key、合理拆分组件、按需异步加载、控制深层响应式、避免模板重复计算。机制上核心是减少无效依赖追踪与无关重渲染。场景上先定位热点再做最小优化，避免提前复杂化代码。
- 考察点：是否具备可执行的日常性能治理清单。
