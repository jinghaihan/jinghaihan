## 概念定位

合成（Composite）节点把多个图层合并成最终画面，核心是图层管理和合成开销控制。

合理图层可提升动画流畅度，图层失控会带来内存和合成成本。

## 工作机制

1. 浏览器根据样式和属性提升部分元素为独立图层。
2. 合成线程按图层顺序合并最终画面。
3. transform/opacity 动画常可在合成阶段完成。
4. 图层变化时更新合成指令。
5. 过多图层会增加内存与管理开销。

## 关键知识点

### 必须掌握

- 合成层不是越多越好。
- 合成优化常与 GPU 内存消耗绑定。
- 合理图层隔离可减少重绘影响范围。
- will-change 需谨慎使用并及时回收。
- 合成阶段问题也会影响 INP。
- 动画性能要结合布局和绘制共同看。

### 选型与取舍

- 图层提升提升流畅度，但内存压力增大。
- 少图层内存低，但可能增加重绘成本。
- 手工图层策略可控，但维护复杂。

## 对页面性能与稳定性的影响

- 节点稳定时，Knowledge Graph 主线 的网络与渲染链路更可预测，性能治理收益更稳定。
- 节点失稳时，问题会向后续阶段扩散，常表现为首屏变慢、交互卡顿或版本一致性风险。
- 优化动作应结合上下游联动验证，避免局部优化把成本转移到其他节点。

## 观测与排障

### 核心指标

- 图层数量与内存占用
- 合成耗时
- 动画掉帧率
- GPU 资源占用

### 排障顺序

1. 先查看图层分布和异常增长。
2. 检查动画属性是否走合成路径。
3. 检查 will-change 是否滥用。
4. 收敛不必要图层提升。
5. 回归低端设备表现。

### 常见故障场景

- 动画流畅但设备发热明显。
- 页面内存持续上涨。
- 图层过多导致合成卡顿。
- 切场景后图层未回收。

## 常见误区

- 把图层提升当万能优化。
- 忽视 GPU 内存预算。
- 动画优化只看单一阶段。

## 题库深挖补充

### 高频对比与边界（基于题库）

- Layout 负责几何计算，Paint 负责像素绘制，Composite 负责图层合成。
- 回流（Reflow）通常比重绘（Repaint）成本高，合成通常成本更低但不是免费。
- top/left 动画常触发布局与绘制，transform/opacity 更可能走合成路径。
- 强制同步布局（Forced Synchronous Layout）来自“写样式后立刻读布局”。
- will-change 能提速也能增内存，滥用会导致图层膨胀。
- 面试常问“为什么动画卡”，核心看属性选择与任务调度。

### 线上排障剧本

1. 在性能面板拆分 Scripting/Layout/Paint/Composite 占比。
2. 定位触发高频布局或重绘的 DOM 操作。
3. 调整动画属性到合成友好路径并验证视觉一致性。
4. 检查图层数量、GPU 内存和掉帧峰值。
5. 在真实设备上回归而非只看本机。

### 高频追问补充（附答案）

1. transform 为什么常更流畅？

因为更容易在合成阶段完成，减少布局和绘制开销。

2. 什么场景会触发强制同步布局？

连续读写布局属性交错，浏览器被迫立即计算几何信息。

3. 合成层越多越好吗？

不是，过多图层会增加内存和合成管理成本。

## 进阶追问（补充）

1. 如果网络指标正常但用户仍感知慢，优先看哪一层？

优先看主线程任务和渲染阶段（Layout/Paint/Composite）是否拥塞，网络快不代表交互快。

2. 缓存策略和发布策略为什么必须一起设计？

缓存决定资源复用路径，发布决定资源版本切换；两者脱节会出现版本错配。

3. 如何验证一次链路优化不是偶然收益？

至少做对照组、分位数对比，并覆盖冷缓存与弱网样本。

## 案例追问加深（第三轮）

### 真实故障片段

- 故障表现：同一版本在不同地区首屏耗时差异明显，且刷新方式不同结果不同。
- 排查结论：网络链路、缓存路径和渲染调度叠加导致体验分化，单点优化无法彻底解决。
- 治理动作：建立链路分层观察面板，按导航类型、缓存命中层和渲染阶段联合分析。

### 追问补充

1. 为什么链路问题经常不是单点根因？

Knowledge Graph 本身是多阶段因果链，一个阶段的波动会在后续阶段被放大。

2. 如何快速缩小排障范围？

先固定触发方式和样本维度，再按网络、缓存、渲染三层逐步排除。

## 面试补充

### 30 秒说明

合成节点优化的核心是“用更少但更有效的图层完成流畅渲染”，避免图层泛滥。

### 高频追问

1. 为什么 transform 常更流畅？

更容易在合成阶段完成，减少布局与绘制压力。

2. will-change 为什么要谨慎？

它可能长期占用图层和内存资源。

3. 图层多一定更快吗？

不一定，图层过多会增加合成成本和内存压力。

4. 合成问题怎么排查？

看图层面板、合成耗时和 GPU 内存变化。
