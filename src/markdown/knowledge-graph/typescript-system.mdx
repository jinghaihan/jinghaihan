## 知识体系概览

TypeScript 体系解决“如何在不改变 JavaScript 运行时语义的前提下，为大型前端项目补充类型约束、声明能力和工程级安全网”的问题。

## 能力边界

- 覆盖范围：基础类型、对象建模、函数签名、泛型、类型编程、类与装饰器、声明文件、模块系统和 `tsconfig` 工程配置。
- 非覆盖范围：JavaScript 运行时规则、浏览器渲染机制、打包器实现细节，这些分别属于 JavaScript、浏览器原理和工程化体系。

## 核心子主题

1. 基础类型与类型推断：`any`、`unknown`、`never`、元组、字面量类型和推断规则。
2. 对象建模与接口：对象类型、`interface`、`type`、索引签名和结构化类型系统。
3. 函数类型与重载：参数约束、返回值、`this`、箭头函数和重载设计。
4. 泛型与泛型约束：复用类型逻辑，同时保留类型安全。
5. 类型守卫与类型编程：`keyof`、`typeof`、`infer`、映射类型、条件类型和工具类型。
6. 枚举、类与装饰器：访问修饰符、抽象类、枚举和装饰器等运行时相关能力。
7. 声明文件与工程集成：`declare`、`.d.ts`、模块系统、`tsconfig`、构建与项目接入。

## 推荐学习路径

- 入门顺序：基础类型与类型推断 -> 对象建模与接口 -> 函数类型与重载 -> 泛型与泛型约束 -> 类型守卫与类型编程 -> 枚举、类与装饰器 -> 声明文件与工程集成
- 面试优先顺序：基础类型与类型推断 -> 对象建模与接口 -> 泛型与泛型约束 -> 类型守卫与类型编程 -> 声明文件与工程集成 -> 函数类型与重载 -> 枚举、类与装饰器
- 项目落地顺序：先补基础类型和对象建模 -> 再补泛型和工具类型 -> 再治理声明文件、模块和工程配置 -> 最后按业务需要使用类、装饰器和高级类型技巧

## 常见混淆边界

- `any` 和 `unknown`：前者关闭检查，后者保留检查。
- `type` 和 `interface`：前者表达能力更广，后者更适合对象契约和扩展。
- 类型断言 vs 类型守卫：前者是“告诉编译器你信我”，后者是“先判断再收窄”。
- TypeScript 和 JavaScript：前者主要工作在编译阶段，后者决定最终运行时行为。

## 相关知识岛

- JavaScript 体系：TypeScript 建立在 JavaScript 语法和运行时之上。
- 前端工程化体系：`tsconfig`、模块系统和构建链路要和工程化配置一起工作。
- Vue 体系：组件 `props`、`emits` 和组合式函数高度依赖类型建模。
- React 体系：组件 `props`、Hook 返回值和状态建模都受类型系统影响。
