## 知识体系概览

前端工程化体系解决“代码如何被稳定构建、发布和维护”的问题，位于研发效率与线上稳定性之间的承接层。

## 能力边界

- 覆盖范围：版本协作、构建工具选型、Webpack 构建机制、热更新、产物优化、构建提速、多页面配置、Babel 编译原理。
- 非覆盖范围：浏览器渲染细节、网络协议细节、框架运行时机制。

## 核心子主题

1. Git 协作与历史整合：版本管理、分支协作与历史整理的基础规则。
2. 构建工具选型：Webpack、Rollup、Parcel 的能力边界与选型标准。
3. Webpack 模块图与构建流程：从 entry 到产物输出的主链路。
4. Loader 与 Plugin 扩展机制：文件转换与编译生命周期扩展。
5. Webpack 热更新运行时：开发期热替换链路与回退条件。
6. Webpack 产物与性能优化：体积、缓存、加载时序的系统优化。
7. Webpack 构建提速策略：全量构建和增量构建的提速方法。
8. 单页与多页构建配置：SPA 与 MPA 的入口和产物组织方式。
9. Babel 编译三阶段：Parse、Transform、Generate 的职责拆分。

## 推荐学习路径

- 入门顺序：Git 协作与历史整合 -> 构建工具选型 -> Webpack 模块图与构建流程 -> Loader 与 Plugin 扩展机制 -> Webpack 产物与性能优化。
- 面试优先顺序：Webpack 模块图与构建流程 -> Loader 与 Plugin 扩展机制 -> Webpack 热更新运行时 -> Webpack 构建提速策略 -> Git 协作与历史整合。
- 项目落地顺序：Webpack 产物与性能优化 -> Webpack 构建提速策略 -> 单页与多页构建配置 -> Babel 编译三阶段。

## 常见混淆边界

- 构建性能优化 vs 运行时性能优化：前者优化构建耗时和产物结构，后者优化页面执行和渲染成本。
- 工具选型 vs 配置调优：选型决定能力上限，调优决定当前项目收益上限。

## 相关知识岛

- 性能优化体系：构建产物拆分、缓存与预加载策略最终体现在性能指标上。
- JavaScript 体系：Babel 转换目标、模块语义和异步加载策略依赖 JS 基础。
- 计算机网络体系：静态资源缓存策略和发布一致性依赖 HTTP 缓存语义。
