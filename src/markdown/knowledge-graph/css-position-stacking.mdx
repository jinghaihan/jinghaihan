## 概念定义

- 本质定义：定位与层叠上下文决定元素在二维平面中的位置和前后层级关系。
- 解决问题：处理遮挡、悬浮、吸顶、弹层和复杂层叠冲突。
- 边界说明：它不处理浮动清理，浮动问题在 BFC 节点展开。

## 核心机制拆解

### 1) 机制/流程

1. 先确定定位类型：`static`、`relative`、`absolute`、`fixed`、`sticky`。
2. 再识别是否创建层叠上下文（`position + z-index`、`opacity < 1`、`transform` 等）。
3. 最后按层叠规则比较绘制顺序。

### 2) 关键条件与触发点

- 条件 A：未创建预期层叠上下文时，`z-index` 看似“失效”。
- 条件 B：复杂组件层叠上下文嵌套会导致弹层被父级裁剪或遮挡。

### 3) 结果与影响

- 对性能/稳定性/可维护性的影响：规则清晰时弹层行为稳定；规则混乱时线上遮挡问题频发且难定位。

## 相似概念对比

| 维度 | `absolute` | `fixed` | `sticky` |
|---|---|---|---|
| 参照系 | 最近定位祖先 | 视口 | 滚动容器与阈值 |
| 是否占位 | 否 | 否 | 初始占位 |
| 滚动行为 | 随容器 | 固定在视口 | 达阈值后吸附 |
| 典型场景 | 局部浮层 | 全局悬浮按钮 | 吸顶导航 |

## 代码

```css
.modal {
  position: fixed;
  inset: 0;
  z-index: 1000;
}
```

### 示例说明

- 这段代码解决什么问题：确保全局弹层相对视口固定覆盖。
- 关键语句或参数含义：`inset: 0` 快速填满视口，`z-index` 控制层级。
- 常见错误写法：只调高子元素 `z-index`，却忽略父级层叠上下文限制。

## 真实场景示例

- 场景描述：弹窗在某些页面被顶部导航遮挡。
- 问题表现：同一组件在不同容器里层级表现不一致。
- 处理方案：梳理层叠上下文，统一弹层挂载层与 `z-index` 设计。
- 验证方式：页面矩阵回归、层级可视化检查、交互覆盖测试。

## 常见误区

- 误区：`z-index` 越大一定在最上层。
  - 正解：比较发生在同一层叠上下文内。
- 误区：`fixed` 永远相对视口。
  - 正解：某些包含变换上下文的祖先会改变其参考行为。

## 关联知识点

- 依赖：经典布局与 Flex。
- 影响：弹层、吸顶、拖拽交互稳定性。
- 对比：浮动、定位、普通流。
- 组合：重排重绘性能治理。

## 相关面试题

1. z-index属性在什么情况下会失效
   - 标准答案：常见失效原因包括元素未形成可比较层叠关系、父级层叠上下文限制、定位条件不满足。机制上 `z-index` 只在同一层叠上下文内比较。场景上要先查上下文再调层级值。
   - 考察点：是否理解“失效”本质是比较域不同。

2. 元素的层叠顺序
   - 标准答案：结论是层叠顺序由文档流、定位、层叠上下文和 `z-index` 共同决定。机制上先分上下文再比较层级。场景上复杂弹层需统一层级规范。
   - 考察点：是否能讲清层叠比较流程。

3. position的属性有哪些，区别是什么
   - 标准答案：`static` 默认流、`relative` 相对自身偏移、`absolute` 相对定位祖先、`fixed` 相对视口、`sticky` 介于相对与固定之间。机制上参照系和占位行为是核心差异。场景上按交互需求选型。
   - 考察点：是否能完整比较五种定位。

4. display、float、position的关系
   - 标准答案：结论是三者共同影响元素格式化和布局参与方式。机制上 `position: absolute/fixed` 会脱离普通流，`float` 也会改变文档流关系。场景上应优先用现代布局，减少历史技巧叠加。
   - 考察点：是否具备统一心智而非碎片记忆。

5. absolute与fixed共同点与不同点
   - 标准答案：共同点是都脱离普通文档流。不同点是 `absolute` 参照最近定位祖先，`fixed` 通常参照视口。场景上局部浮层常用 `absolute`，全局悬浮常用 `fixed`。
   - 考察点：是否清楚参照系差异。

6. 对 sticky 定位的理解
   - 标准答案：结论是 `sticky` 在阈值前像相对定位，达到阈值后像固定定位。机制上依赖滚动容器和偏移阈值。场景上常用于吸顶目录和表头。
   - 考察点：是否知道 sticky 生效条件。
