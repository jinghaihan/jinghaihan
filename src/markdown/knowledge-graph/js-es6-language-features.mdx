## 概念定义

- 本质定义：ES6 语法与语言特性是现代 JavaScript 的基础表达集合。
- 解决问题：解决变量作用域、函数表达、结构拆解、模块组织等工程问题。
- 边界说明：不覆盖原型链继承和异步调度细节。

## 核心机制拆解

### 1) 机制/流程

1. 用 `let/const` 取代 `var` 管理变量生命周期。
2. 用解构和扩展语法简化数据读写。
3. 用箭头函数统一短函数表达。
4. 用模板字符串与 `rest` 参数提升可读性。

### 2) 关键条件与触发点

- 条件 A：箭头函数不绑定自身 `this`，也不能作为构造函数。
- 条件 B：`const` 约束的是绑定不可变，不代表对象内容不可改。

### 3) 结果与影响

- 对可维护性的影响：语法约束更清晰，代码表达更一致。
- 对稳定性的影响：减少变量提升和上下文混乱问题。

## 相似概念对比

| 维度 | 箭头函数 | 普通函数 | 结论 |
|---|---|---|---|
| this 绑定 | 词法绑定外层 | 调用时绑定 | 场景不同 |
| `new` 调用 | 不支持 | 支持 | 构造场景用普通函数 |
| `arguments` | 无自身 `arguments` | 有 | 参数处理方式不同 |
| 适用场景 | 回调与短函数 | 构造、方法、需动态 this | 不应强行统一 |

## 代码

```ts
const user = { name: 'A', age: 18 }
const { name, ...rest } = user

const sum = (...nums: number[]) => nums.reduce((acc, n) => acc + n, 0)

function Person(this: any, n: string) {
  this.name = n
}

console.log(name, rest, sum(1, 2, 3))
```

### 示例说明

- 展示了解构、`rest` 参数和普通函数构造能力。
- 箭头函数适合聚合计算，不适合 `new` 场景。
- 常见错误写法是用箭头函数定义对象方法并期望动态 `this`。

## 相关面试题

1. let、const、var 的区别
- 标准答案：`var` 是函数作用域且存在提升；`let/const` 是块级作用域并有暂时性死区。`const` 要求绑定不可重新赋值，但对象内部属性仍可修改。
- 考察点：作用域和提升规则。

2. const 对象的属性可以修改吗
- 标准答案：可以。`const` 只保证变量绑定地址不变，不保证引用对象的内容冻结。若需彻底不可变应配合深冻结策略。
- 考察点：是否区分绑定与值。

3. 如果 new 一个箭头函数会怎么样
- 标准答案：会抛出类型错误，因为箭头函数没有 `prototype`，也没有构造器语义。
- 考察点：是否理解箭头函数设计边界。

4. 箭头函数与普通函数的区别
- 标准答案：主要差异是 `this`、`arguments`、`new` 和 `prototype`。箭头函数更适合纯函数回调，普通函数更适合方法和构造场景。
- 考察点：是否能给出场景化结论。

5. 扩展运算符的作用及使用场景
- 标准答案：用于展开数组/对象、合并结构和函数传参，常用于不可变更新。对象扩展是浅拷贝，嵌套对象仍共享引用。
- 考察点：是否理解浅拷贝边界。

6. 什么是ES6？它的主要特性有哪些？
- 标准答案：结论是 ES6 是 JavaScript 的一次里程碑级标准升级。机制上它引入块级作用域、模块化、类、Promise、解构等能力，统一了语言工程化基础。场景上现代前端项目几乎都以 ES6+ 作为默认语法基线。
- 考察点：是否能系统列举核心特性而非零散背诵。

7. let、const和var的区别是什么？
- 标准答案：结论是 `let/const` 是块级作用域并有暂时性死区，`var` 是函数作用域且存在变量提升。机制上 `const` 仅限制绑定不可重新赋值。场景上工程代码优先 `const`，需要可变时用 `let`。
- 考察点：是否能把作用域、提升、可变性三点一次讲清。

8. 什么是箭头函数？它与普通函数有什么区别？
- 标准答案：结论是箭头函数是更简洁的函数表达，并采用词法 `this`。机制上它没有 `prototype`、不能 `new`、没有自身 `arguments`。场景上回调和函数组合适合箭头函数，构造和对象方法多用普通函数。
- 考察点：是否理解语法简化背后的语义差异。

9. 什么是模板字符串？它有什么优势？
- 标准答案：结论是模板字符串用反引号支持多行文本和表达式插值。机制上 `${expr}` 在运行时求值并拼接结果。场景上动态文案、SQL/HTML 片段拼装可读性明显优于字符串加号拼接。
- 考察点：是否能说明它改善了哪些具体开发痛点。

10. 解构赋值是什么？请举例说明数组和对象的解构。
- 标准答案：结论是解构赋值是按结构提取数据并赋给变量。机制上数组按位置匹配，对象按属性名匹配并支持重命名与默认值。场景上函数参数解构和接口返回值拆包可显著减少样板代码。
- 考察点：是否掌握数组与对象解构的不同匹配规则。

11. 什么是扩展运算符（...）？它有哪些用途？
- 标准答案：结论是扩展运算符用于展开可迭代结构和对象属性。机制上数组展开可用于拼接与传参，对象展开常用于不可变更新。场景上 Redux/React 状态更新是高频应用，但要注意浅拷贝边界。
- 考察点：是否能解释“便利性”背后的数据复制语义。

12. 什么是剩余参数（rest parameters）？
- 标准答案：结论是剩余参数把不定长参数收集为真实数组。机制上语法为 `(...args)`，且必须放在形参列表末尾。场景上工具函数和函数式组合中替代 `arguments` 更直观且类型友好。
- 考察点：是否能区分 rest 和 spread 在位置与语义上的差异。

13. ES6中的默认参数是如何工作的？
- 标准答案：结论是默认参数在实参为 `undefined` 时生效。机制上默认值表达式在调用时求值，可引用前序参数。场景上接口函数可减少手动判空与参数归一化代码。
- 考察点：是否理解默认参数触发条件和求值时机。

14. 什么是Promise？它解决了什么问题？
- 标准答案：结论是 Promise 是异步结果的状态容器。机制上通过 `pending/fulfilled/rejected` 状态机和链式 then/catch 组织异步流程。场景上它解决回调地狱和错误传播分散问题。
- 考察点：是否能把状态机模型和工程收益对应起来。

15. ES6的类（Class）是如何工作的？
- 标准答案：结论是 Class 是基于原型链的语法糖。机制上方法定义在原型上，`extends/super` 构建继承链。场景上它提升面向对象代码可读性，但本质仍是 JavaScript 原型系统。
- 考察点：是否理解“语法糖”而非新对象模型。

16. ES6模块化（import/export）是如何工作的？
- 标准答案：结论是 ES Module 提供静态依赖声明和按需导出导入。机制上编译阶段可做依赖图分析和 Tree Shaking。场景上现代构建工具都以 ESM 为核心，利于代码拆分与优化。
- 考察点：是否能从“静态结构”解释其工程价值。

17. Symbol数据类型有什么特点和用途？
- 标准答案：结论是 Symbol 是唯一且不可冲突的原始值。机制上常用于对象私有标识、避免属性命名冲突，并支持内置迭代协议等元编程入口。场景上库开发和对象扩展时可降低全局污染风险。
- 考察点：是否理解 Symbol 的“唯一性”在工程中的实际意义。

18. Map和Set数据结构有什么特点？
- 标准答案：结论是 Map 是键值集合，Set 是值唯一集合。机制上二者都保持插入顺序并提供高效查找接口。场景上对象键非字符串、去重统计、缓存索引场景优先考虑 Map/Set。
- 考察点：是否能与 Object/Array 做场景化对比。

19. 什么是生成器函数（Generator）？
- 标准答案：结论是 Generator 是可暂停、可恢复执行的函数。机制上通过 `function*` 与 `yield` 分段产出值，调用返回迭代器。场景上可用于惰性序列、复杂流程控制和协程式任务编排。
- 考察点：是否理解“可中断执行”这一核心语义。

20. 什么是迭代器（Iterator）和可迭代对象？
- 标准答案：结论是 Iterator 是带 `next()` 的遍历协议，可迭代对象实现 `Symbol.iterator` 并返回迭代器。机制上 `for...of`、展开语法都依赖该协议。场景上自定义数据结构实现迭代协议后可无缝接入语言能力。
- 考察点：是否能讲清协议接口和调用方关系。

21. for...of和for...in的区别是什么？
- 标准答案：结论是 `for...of` 遍历可迭代对象的值，`for...in` 遍历对象可枚举键。机制上后者会遍历原型链可枚举属性，前者遵循迭代协议。场景上遍历数组优先 `for...of`，遍历对象键再考虑 `for...in` 配合 `hasOwn`。
- 考察点：是否能避免把两者混用于数组导致隐患。

22. 什么是Proxy？它有什么用途？
- 标准答案：结论是 Proxy 是对象操作拦截器。机制上可代理 `get/set/delete/has` 等底层行为，实现访问控制和行为增强。场景上 Vue 3 响应式、权限校验、日志埋点都常用 Proxy。
- 考察点：是否理解 Proxy 是“行为层拦截”而非数据拷贝。

23. 什么是Reflect？它与Proxy有什么关系？
- 标准答案：结论是 Reflect 提供与 Proxy trap 对应的标准反射方法。机制上在 Proxy 拦截中可用 Reflect 保持默认语义并减少手写边界 bug。场景上实现透明代理时通常 `Proxy + Reflect` 成对出现。
- 考察点：是否能说明 Reflect 在代理实现中的工程价值。

24. 什么是WeakMap和WeakSet？它们与Map、Set有什么区别？
- 标准答案：结论是 WeakMap/WeakSet 的键或成员是弱引用对象，且不可遍历。机制上当对象无其他强引用时可被回收，容器项自动失效。场景上缓存 DOM 节点附加数据、私有元数据管理可避免内存泄漏。
- 考察点：是否理解“弱引用 + 不可遍历”这两个关键限制。

25. 什么是暂时性死区（Temporal Dead Zone）？
- 标准答案：结论是 TDZ 指变量在声明前可见但不可访问的区间。机制上 `let/const/class` 在块作用域开始即绑定，直到声明语句执行前访问会抛错。场景上它能尽早暴露变量使用顺序错误。
- 考察点：是否能区分“提升存在”与“可访问”是两件事。

26. ES6中的对象字面量有哪些增强？
- 标准答案：结论是 ES6 增强包括属性简写、方法简写、计算属性名等。机制上让对象定义更贴近变量和表达式上下文。场景上配置对象和工厂函数返回值可明显减少冗余代码。
- 考察点：是否能举出至少三类对象字面量增强语法。

27. 什么是数组的新方法？请列举几个ES6新增的数组方法。
- 标准答案：结论是 ES6 为数组增加了 `find/findIndex/from/of/fill/copyWithin` 等常用方法。机制上这些方法覆盖查找、构建和批量写入等场景。场景上可替代大量手写循环并提升语义表达。
- 考察点：是否能按功能分类列举而非零散背 API。

28. 什么是字符串的新方法？ES6为字符串添加了哪些方法？
- 标准答案：结论是 ES6 常用新增有 `includes/startsWith/endsWith/repeat` 等。机制上这些 API 直接覆盖高频匹配与重复拼接需求。场景上相比正则和 index 判断更直观且可读性更高。
- 考察点：是否能结合实际文本处理场景回答。

29. 什么是参数解构？请举例说明。
- 标准答案：结论是参数解构是在函数形参位置直接拆解对象或数组。机制上可同时声明默认值并约束输入结构。场景上配置对象参数和 API 入参归一化时能减少函数体内拆包代码。
- 考察点：是否理解参数解构与默认值的协同写法。

30. 什么是尾调用优化？ES6如何支持尾调用优化？
- 标准答案：结论是尾调用优化是在函数尾位置调用另一个函数时复用当前栈帧。机制上可降低深递归的栈占用风险，但实现依赖引擎支持与严格模式条件。场景上理论上可优化递归，实践中仍需关注兼容性。
- 考察点：是否能区分语言规范支持与引擎落地现状。

31. 什么是标签模板？它有什么用途？
- 标准答案：结论是标签模板是把模板字符串交给自定义函数处理的机制。机制上标签函数会收到字符串片段数组和插值参数，可实现转义、国际化、DSL 解析。场景上 SQL/HTML 安全拼接与样式模板库常用该能力。
- 考察点：是否理解标签模板不是“另一种字符串拼接”。

32. ES6中的正则表达式有哪些新特性？
- 标准答案：结论是常用增强包括 `u`、`y`、`s` 标志与更完善的 Unicode 支持。机制上这些特性提升了多语言文本匹配准确性和状态控制能力。场景上处理 emoji、国际化文本或流式匹配时收益明显。
- 考察点：是否能给出具体标志位的语义差异。

33. 什么是二进制和八进制字面量？
- 标准答案：结论是 ES6 支持 `0b` 二进制和 `0o` 八进制字面量。机制上解析时直接按对应进制转换数值。场景上位运算、协议字段和权限掩码表达更直观。
- 考察点：是否能正确写出语法前缀并说明使用场景。

34. ES6中的Number对象有哪些新方法？
- 标准答案：结论是常用新增有 `Number.isNaN`、`Number.isFinite`、`Number.isInteger`、`Number.parseInt` 等。机制上它们避免了全局方法的隐式类型转换陷阱。场景上输入校验和数值边界判断更安全可控。
- 考察点：是否理解新方法对类型安全的改进点。

35. 什么是Object.assign()？它有什么特点和用途？
- 标准答案：结论是 `Object.assign` 用于把源对象可枚举自有属性浅拷贝到目标对象。机制上后写属性覆盖先写属性，且只做一层复制。场景上对象合并、默认配置覆盖和不可变更新是高频用法。
- 考察点：是否能明确指出其“浅拷贝”边界与覆盖规则。
