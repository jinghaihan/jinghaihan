## 代码

```ts
import { nextTick, ref } from 'vue'

const expanded = ref(false)

async function toggleAndMeasure(el: HTMLElement) {
  expanded.value = !expanded.value
  await nextTick()
  console.log('height after patch:', el.offsetHeight)
}
```

### 示例说明

- 这段代码解决什么问题：在 DOM 提交后读取最新布局，避免读到旧值。
- 关键语句或参数含义：`nextTick` 保证回调在本轮渲染补丁提交后执行。
- 常见错误写法：状态刚改完就同步读 DOM，得到过期尺寸。

## 相关面试题

1. 对 keep-alive 的理解，它是如何实现的，具体缓存的是什么？
- 标准答案：结论是 `keep-alive` 会缓存组件实例而不是缓存 DOM 字符串。机制上切换时触发 `activated/deactivated`，避免重复创建销毁。场景上列表详情切换、标签页容器常用，但缓存过多会占内存。
- 考察点：是否能给出可执行步骤、关键 API 和常见踩坑点。

2. $nextTick 原理及作用
- 标准答案：结论是 `nextTick` 用于在本轮 DOM 更新完成后执行回调。机制上 Vue 会把更新合并到异步队列，`nextTick` 在队列 flush 后触发。场景上读取最新 DOM 尺寸、滚动定位时必须用它。
- 考察点：是否能按阶段讲清核心流程、关键数据结构和边界条件。

3. Vue data 中某一个属性的值发生改变后，视图会立即同步执行重新渲染吗？
- 标准答案：结论是通常不会立即同步重渲染，而是批量异步更新。机制上同一事件循环内的多次状态变更会合并，减少重复渲染。场景上这能提升性能，但依赖即时 DOM 时要配合 `nextTick`。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

4. 说一下 Vue 的生命周期
- 标准答案：结论是常见生命周期包括 `beforeCreate/created/beforeMount/mounted/beforeUpdate/updated/beforeUnmount/unmounted`。机制上它们对应实例初始化、挂载、更新、卸载四阶段。场景上数据请求常放 `mounted` 或路由钩子，清理副作用放卸载阶段。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

5. created 和 mounted 的区别
- 标准答案：结论是 `created` 时数据和方法可用但无真实 DOM，`mounted` 时 DOM 已挂载。机制上二者处于挂载流程不同阶段。场景上访问 DOM 或第三方控件初始化应放 `mounted`。
- 考察点：是否能从定义、触发时机、性能影响和适用场景四个维度做对比。

6. 一般在哪个生命周期请求异步数据
- 标准答案：结论是常见做法是 `mounted`、路由守卫或 `setup` 内触发请求。机制上要结合是否依赖 DOM 与是否需要首屏预取。场景上 SSR 项目优先服务端预取，纯客户端项目在 `mounted` 请求最直观。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

7. keep-alive 中的生命周期哪些
- 标准答案：结论是被缓存组件会额外触发 `activated/deactivated`。机制上进入缓存页面走 `activated`，离开但不销毁走 `deactivated`。场景上轮询、订阅在 `deactivated` 暂停，在 `activated` 恢复。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

8. Vue 的响应式更新是同步还是异步的？
- 标准答案：结论是默认异步批量更新。机制上同一事件循环内的多次状态变更会进入调度队列合并刷新，再在合适时机提交 DOM。场景上连续赋值只触发一次视图提交，需要读取最新 DOM 时应配合 `nextTick`。
- 考察点：是否理解“状态修改时刻”和“DOM 提交时刻”的区别。

9. Vue 3 的生命周期钩子有哪些？与 Vue 2 相比有什么变化？
- 标准答案：结论是 Vue 3 仍按创建、挂载、更新、卸载四阶段组织，但组合式 API 通过 `onMounted/onUpdated/onUnmounted` 等函数注册。机制上钩子语义更一致，废弃了部分旧名称并强化 setup 入口。场景上迁移项目时需做钩子映射和副作用清理策略调整。
- 考察点：是否能从迁移视角讲清钩子变更点。
