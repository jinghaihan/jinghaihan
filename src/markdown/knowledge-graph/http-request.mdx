## 知识体系概览

计算机网络体系解决“请求如何可靠、快速、安全地从浏览器到服务端再返回”的问题，是渲染主线里网络阶段的核心。

## 能力边界

- 覆盖范围：DNS（Domain Name System）解析、传输层建连、HTTP（HyperText Transfer Protocol）报文与版本、状态码决策、同源与跨域、实时通道。
- 非覆盖范围：DOM（Document Object Model）构建、CSSOM（CSS Object Model）计算、布局与绘制，这些属于浏览器原理体系。

## 核心子主题

1. DNS 解析流程：从域名到目标地址的查询路径、缓存命中与 `dns-prefetch` / `preconnect` 预热策略。
2. DNS 结果与缓存策略：TTL（Time To Live）、多级缓存和地址选择策略。
3. TCP（Transmission Control Protocol）/TLS（Transport Layer Security）建连：连接建立、加密协商与证书校验。
4. HTTP 方法语义：不同方法的语义边界、幂等性与重试策略。
5. HTTP 报文与头字段：请求行、响应行、请求头、响应头和缓存协商字段。
6. HTTP 版本演进：HTTP/1.0、HTTP/1.1、HTTP/2、HTTP/3 的能力差异。
7. 从 URL（Uniform Resource Locator）输入到页面响应：浏览器发起导航时的全链路过程。
8. 状态码决策链：2xx / 3xx / 4xx / 5xx 的处理分支。
9. 同源与跨域策略：CORS（Cross-Origin Resource Sharing）预检、响应头配置和安全边界。
10. 实时通道（WebSocket/SSE）：持续连接与服务端推送场景。

## 推荐学习路径

- 入门顺序：DNS 解析流程 -> TCP/TLS 建连 -> HTTP 方法语义 -> HTTP 报文与头字段 -> 状态码决策链 -> HTTP 版本演进 -> 同源与跨域策略 -> 实时通道。
- 面试优先顺序：从 URL 输入到页面响应 -> HTTP 方法语义 -> HTTP 版本演进 -> TCP/TLS 建连 -> 状态码决策链 -> 同源与跨域策略。
- 项目落地顺序：先保证正确性（方法语义、状态码）-> 再保证稳定性（DNS、建连、跨域）-> 最后做性能优化（版本能力、缓存、实时通道）。

## 常见混淆边界

- 传输层连接复用 vs 应用层多路复用：前者是连接数量问题，后者是同连接并发传输问题。
- 强缓存 vs 协商缓存：前者本地直接复用，后者需要请求服务端确认。
- 301 / 302 vs 307 / 308：后两者更强调保留原请求方法语义。
- CORS 错误 vs 服务端业务错误：前者是浏览器安全策略拦截，后者是接口业务处理失败。

## 相关知识岛

- 浏览器原理体系：网络响应进入 `html-parser` 后开始解析与渲染阶段。
- 性能优化体系：缓存策略、协议版本与资源加载策略直接影响核心指标。
- HTML 体系：脚本加载策略、资源标签配置会改变网络请求行为。
