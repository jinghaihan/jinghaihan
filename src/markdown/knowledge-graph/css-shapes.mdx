## 概念定义

- 本质定义：CSS 图形实战是利用边框、圆角、渐变和变换组合实现常见几何图形。
- 解决问题：减少对静态切图依赖，提升图形可维护性与响应式能力。
- 边界说明：复杂动态绘图仍应使用 Canvas 或 SVG。

## 核心机制拆解

### 1) 机制/流程

1. 明确图形类型与可缩放需求。
2. 选择实现手段：`border`、`border-radius`、`clip-path`、`transform`。
3. 按容器约束做自适应比例控制。

### 2) 关键条件与触发点

- 条件 A：图形语义不清时，后续维护者很难理解样式目的。
- 条件 B：图形过于复杂时，纯 CSS 可读性与可维护性会下降。

### 3) 结果与影响

- 对性能/稳定性/可维护性的影响：简单图形用 CSS 能减少资源请求；复杂图形硬写 CSS 会增加维护成本。

## 相似概念对比

| 维度 | 纯 CSS 图形 | 图片资源图形 | 结论 |
|---|---|---|---|
| 请求成本 | 无额外请求 | 有资源请求 | 小图标可优先 CSS |
| 可缩放性 | 高 | 取决于资源分辨率 | 响应式场景 CSS 更灵活 |
| 维护性 | 简单图形好维护 | 复杂图形更直观 | 按复杂度选型 |
| 适用场景 | 徽标、角标、几何装饰 | 写实图形、复杂插画 | 可混合使用 |

## 代码

```css
.triangle {
  width: 0;
  height: 0;
  border-left: 8px solid transparent;
  border-right: 8px solid transparent;
  border-bottom: 12px solid #0ea5e9;
}
```

### 示例说明

- 这段代码解决什么问题：用边框法生成三角形。
- 关键语句或参数含义：利用宽高为 0 时边框交点构造图形。
- 常见错误写法：未将左右边框设透明，导致形状失真。

## 真实场景示例

- 场景描述：下拉菜单需要角标三角形并随主题换色。
- 问题表现：使用切图后主题切换成本高。
- 处理方案：改成 CSS 图形，颜色跟随主题变量。
- 验证方式：主题切换回归、缩放适配回归。

## 常见误区

- 误区：所有图形都应该纯 CSS 实现。
- 正解：复杂图形应优先 SVG 或图片资源。
- 误区：CSS 图形不支持响应式。
- 正解：通过相对单位和容器约束可实现响应式缩放。

## 关联知识点

- 依赖：盒模型与文本排版。
- 影响：资源引入与图像策略。
- 对比：SVG 图形方案。
- 组合：发丝线与边框细节处理。

## 相关面试题

1. 实现一个三角形
- 标准答案：结论是可用宽高为 0 + 边框法实现。机制上通过三边透明、一边着色构造三角形。场景上常用于气泡提示角。
- 考察点：是否掌握边框法原理。
```css
.triangle {
  width: 0;
  height: 0;
  border-left: 10px solid transparent;
  border-right: 10px solid transparent;
  border-bottom: 16px solid #22c55e;
}
```

2. 实现一个扇形
- 标准答案：可用圆形容器 + `overflow` 裁切，或 `conic-gradient` 实现。机制上本质是圆形区域裁剪。场景上进度环、饼图片段常见。
- 考察点：是否能给出可落地方案。
```css
.sector {
  width: 96px;
  aspect-ratio: 1 / 1;
  border-radius: 50%;
  background: conic-gradient(#0ea5e9 0deg 90deg, #e2e8f0 90deg 360deg);
}
```

3. 实现一个圆和半圆
- 标准答案：圆可用等宽高 + `border-radius: 50%`。半圆可通过裁切或单侧圆角实现。机制上关键是圆角比例控制。场景上头像、状态点、进度胶囊常用。
- 考察点：是否理解圆角形成规则。
```css
.circle {
  width: 80px;
  aspect-ratio: 1 / 1;
  border-radius: 50%;
  background: #60a5fa;
}

.half-circle {
  width: 80px;
  height: 40px;
  border-radius: 40px 40px 0 0;
  background: #14b8a6;
}
```

4. 实现一个宽高自适应的正方形
- 标准答案：可用 `aspect-ratio: 1 / 1` 或历史 `padding-top: 100%` 技巧。机制上利用宽高比例约束实现自适应。场景上宫格卡片常用。
- 考察点：是否知道现代与兼容写法。
```css
.square {
  width: min(40vw, 220px);
  aspect-ratio: 1 / 1;
  background: #f59e0b;
}
```

5. 画一个梯形
- 标准答案：可用边框法或 `transform: skew` 实现。机制上通过斜切与边框裁剪形成上窄下宽结构。场景上标签条、步骤条常见。
- 考察点：是否掌握至少一种稳定方案。
