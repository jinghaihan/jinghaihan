## 概念定义

- 本质定义：回流、重绘与渲染优化关注样式和结构变化如何影响布局计算与绘制成本。
- 解决问题：解决页面卡顿、掉帧、交互延迟和滚动不流畅问题。
- 边界说明：不解决网络请求慢和资源下载慢问题。

## 核心机制拆解

### 1) 机制/流程

1. 样式或结构发生变化后，浏览器判断是否需要重新布局。
2. 需要布局时触发回流，计算元素几何信息。
3. 几何信息稳定后执行重绘，把像素写入图层。
4. 图层合成后展示到屏幕。
5. 优化目标是减少回流频率和缩小影响范围。

### 2) 关键条件与触发点

- 条件 A：修改尺寸、位置、文档流结构等会触发回流。
- 条件 B：只修改颜色、阴影等视觉属性通常只触发重绘。

### 3) 结果与影响

- 对性能的影响：高频回流会显著放大主线程压力，导致交互卡顿。
- 对可维护性的影响：批量读写分离和动画约束能降低性能回归风险。

## 相似概念对比

| 维度 | 回流 | 重绘 | 结论 |
|---|---|---|---|
| 触发条件 | 几何信息变化 | 外观变化但几何不变 | 回流成本更高 |
| 影响范围 | 可能波及上下文布局 | 主要影响当前绘制区域 | 回流更易扩散 |
| 性能开销 | 高 | 中 | 优先减少回流 |
| 关系 | 回流通常伴随重绘 | 重绘不一定触发回流 | 单向包含关系 |

## 代码

```ts
const box = document.getElementById('box') as HTMLElement

// 先读后写，避免读写交错触发多次布局
const width = box.offsetWidth
const nextWidth = width + 20

requestAnimationFrame(() => {
  box.style.transform = `translateX(${nextWidth}px)`
  box.style.willChange = 'transform'
})

const fragment = document.createDocumentFragment()
for (let i = 0; i < 50; i++) {
  const item = document.createElement('li')
  item.textContent = `item-${i}`
  fragment.appendChild(item)
}
document.querySelector('#list')?.appendChild(fragment)
```

### 示例说明

- 示例包含三类优化：读写分离、`requestAnimationFrame` 调度、`DocumentFragment` 批量插入。
- `transform` 动画通常比直接改 `top/left` 更不容易触发布局。
- 常见错误写法是在循环里边读布局边改样式，造成强制同步布局。

## 真实场景示例

- 场景描述：后台列表页滚动时会动态更新行高和位置。
- 问题表现：滚动掉帧，输入框输入延迟明显。
- 处理方案：把频繁布局属性改为合成属性动画，批量 DOM 变更改用 `DocumentFragment`。
- 验证方式：观察长任务数量、帧率和交互延迟指标是否改善。

## 常见误区

- 误区：只要减少 DOM 数量就不会卡。
- 正解：关键在于更新方式和触发频率，少量 DOM 也可能高频回流。
- 误区：动画都应该用 `setInterval`。
- 正解：渲染相关动画优先用 `requestAnimationFrame`，更贴合浏览器刷新节奏。

## 关联知识点

- 依赖：浏览器渲染阶段知识。
- 影响：节流与防抖策略、发布门禁指标。
- 组合：图片优化策略和懒加载策略。

## 相关面试题

1. 回流与重绘的概念及触发条件
- 标准答案：回流是布局几何信息重新计算，常由尺寸、位置、结构变化触发；重绘是外观重新绘制，常由颜色、阴影等变化触发。回流通常会带来重绘，重绘不一定导致回流。
- 考察点：是否能准确区分触发条件和成本差异。

2. 如何避免回流与重绘？
- 标准答案：核心做法是减少高频样式写入、读写分离、批量更新 DOM、优先使用不影响布局的属性、把复杂更新下沉到低影响区域。关键机制是“减少触发次数 + 缩小影响范围”。
- 考察点：是否能给出可执行的工程措施。

3. 如何优化动画？
- 标准答案：优先使用 `transform`、`opacity` 这类合成层友好属性，动画调度使用 `requestAnimationFrame`，并避免在动画过程中查询布局信息。这样可减少布局和绘制压力，降低掉帧风险。
- 考察点：是否理解动画属性选择与渲染阶段关系。

4. documentFragment 是什么？用它跟直接操作 DOM 的区别是什么？
- 标准答案：`DocumentFragment` 是内存中的轻量节点容器，批量拼装后一次插入真实 DOM，可减少中间状态反复触发布局与绘制。直接逐条插入 DOM 更直观，但在批量场景下开销更高。
- 考察点：是否理解批量更新与渲染成本关系。
