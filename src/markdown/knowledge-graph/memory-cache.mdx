## 概念定位

内存缓存检查是浏览器缓存链路中最快的一层。它在请求触发后优先尝试从进程内存直接复用响应副本，目标是极低延迟返回。

## 工作机制

1. 浏览器按请求键匹配可复用响应（URL、方法、Vary 维度等）。
2. 判断响应是否仍在可用生命周期内。
3. 命中且新鲜时直接返回副本，不进入磁盘和网络。
4. 未命中或失效则继续下探到磁盘缓存或后续链路。

## 关键知识点

- **近零延迟**：省去磁盘 IO（Input/Output）和网络往返。
- **生命周期短**：受页面、进程、内存回收策略影响。
- **可用性受策略约束**：并非所有资源都可驻留内存缓存。
- **与 BFCache 区别**：内存缓存是资源级，BFCache 是页面快照级。

## 对页面性能的影响

- 复访或重复请求场景下可显著降低等待。
- 内存压力波动会导致命中率不稳定。
- 大资源滥用可能挤占可用内存，影响整体性能稳定性。

## 观测与排障

### 浏览器侧

- Network 关注资源来源是否显示 memory cache。
- 对比同会话首次与二次请求耗时。
- 结合 Performance 看主线程是否仍被后续解析阻塞。

### 排障步骤

1. 先确认响应头是否允许合理缓存。
2. 观察是否存在频繁进程重启或页面重建。
3. 检查资源体积是否过大导致缓存淘汰。
4. 回归看重复访问路径下的请求耗时分布。

## 常见误区

- “命中内存缓存就没有性能问题”：不准确，后续解析和渲染仍可能慢。
- “清除磁盘缓存就等于清内存缓存”：不总是等价。
- “内存缓存命中率越高越好”：不绝对，要平衡内存占用和稳定性。

## 题库深挖补充

### 高频对比与边界（基于题库）

- 内存缓存命中速度最快，但生命周期通常与标签页或进程存活强相关。
- 内存缓存容量受浏览器内存压力影响，可能被主动回收，不保证持久性。
- 同 URL 命中内存缓存不代表跨标签页可复用，这与浏览器实现有关。
- Cache-Control 的 no-store 会禁止缓存落地，private/public 会影响共享缓存策略。
- 面试常把“强缓存”与“内存缓存”混为一谈，实际前者是策略，后者是存储层。
- 内存缓存是提升回访速度的关键，但版本一致性仍依赖指纹和入口策略。

### 线上排障剧本

1. 先确认是策略不允许缓存，还是缓存被淘汰。
2. 检查请求头与响应头中的 Cache-Control/Pragma/Expires。
3. 对比同标签页复访和跨标签页复访差异。
4. 检查是否因内存压力导致缓存提前回收。
5. 发布问题排查时，不要只看命中率，要看版本一致性。

### 高频追问补充（附答案）

1. 内存缓存为什么有时“忽然没了”？

浏览器在内存紧张时会回收缓存，命中并非强保证。

2. 强缓存命中就一定在内存里吗？

不一定，可能命中磁盘缓存，也可能命中内存缓存，取决于浏览器策略。

3. 内存缓存和 Service Worker 缓存冲突吗？

两者层级和控制逻辑不同，排障时需要区分命中来源。

### 术语速查与易错点

- Memory Cache：进程内缓存，命中快但不持久。
- no-store：禁止缓存落地。
- private：仅私有缓存可存。
- immutable：内容不变可长期复用。
- Revalidation：协商校验缓存新鲜度。

常见易错点：

- 把“命中内存缓存”当成跨会话保证。
- 只看命中率，不看版本一致性。
- 误把策略问题当成浏览器 bug。

## 深度延展（案例对照）

### 内存缓存命中错觉案例

- 场景：页面二次访问很快，但新开标签页又变慢。
- 诊断：只命中当前进程内存缓存，跨标签页未复用。
- 处理：优化磁盘缓存与协商缓存策略，降低跨会话波动。

### 排查补充

1. 同标签页刷新、跨标签页打开分别测一次。
2. 观察 from memory cache / from disk cache 分布。
3. 对关键资源做指纹和缓存头校验。

## 进阶排障清单（补充）

### 现场处置顺序

1. 区分同标签页复访、跨标签页复访、冷启动三种路径。
2. 核对 from memory cache / from disk cache / 304 三类命中。
3. 检查响应头是否允许缓存复用。
4. 检查是否因内存压力触发回收。
5. 验证发布后资源版本是否仍一致。

### 面试追问强化

1. 为什么同资源在不同访问方式下命中层不同？

浏览器会按会话状态、资源类型和内存压力动态选择缓存层。

2. 内存缓存命中和用户体验一定正相关吗？

不一定，若入口或关键脚本未命中，整体体验仍可能慢。

3. 如何防止缓存优化引发版本错配？

入口短缓存、资源指纹化、发布原子切换三者同时到位。

## 速查补充（第四轮）

### 快速判定问答

1. 什么时候先怀疑缓存层而不是权威记录本身？

当不同客户端结果分叉明显且权威查询稳定时，优先检查本地与递归缓存状态。

2. DNS 类问题为什么要强调样本固定？

因为网络、地域、时间窗口会影响解析路径，不固定样本容易得到互相矛盾结论。

## 面试补充

### 30 秒说明

内存缓存是请求链路里最快的复用层。命中后可以跳过磁盘和网络，显著降低等待。但它受生命周期和内存压力影响，不能替代完整缓存策略设计。

### 高频追问

1. 为什么同页面里二次请求经常极快？

因为命中了进程内的响应副本，省去磁盘与网络阶段。

2. 什么时候会突然失去内存缓存收益？

页面重建、进程回收、内存压力或缓存策略变化都会导致命中率下降。

3. 如何把内存缓存收益稳定下来？

核心是资源分层与缓存头治理，避免大资源污染缓存空间，并保持请求键稳定。
