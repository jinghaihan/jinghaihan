## 概念定义

- 本质定义：盒模型与文本排版定义元素尺寸计算和文本渲染规则。
- 解决问题：避免尺寸错乱、行高异常、溢出处理不一致。
- 边界说明：它不负责定位层叠顺序和跨设备适配策略。

## 核心机制拆解

### 1) 机制/流程

1. 先确定盒模型（`content-box` 或 `border-box`）。
2. 再计算 `content`、`padding`、`border`、`margin` 组成的最终尺寸。
3. 文本排版按 `font-size`、`line-height`、`white-space` 与溢出规则渲染。

### 2) 关键条件与触发点

- 条件 A：混用盒模型会导致同宽组件显示不一致。
- 条件 B：`line-height` 继承设置不当会导致多层文本错位。

### 3) 结果与影响

- 对性能/稳定性/可维护性的影响：规则一致时组件尺寸稳定；规则不一致时 UI（User Interface）抖动和排障成本上升。

## 相似概念对比

| 维度 | `content-box` | `border-box` | 结论 |
|---|---|---|---|
| `width` 含义 | 仅内容区 | 含 `padding` 和 `border` | 现代项目常统一 `border-box` |
| 布局心智负担 | 计算成本更高 | 更贴近视觉尺寸 | 大规模组件更易维护 |
| 典型场景 | 历史兼容 | 组件化项目默认 | 推荐统一策略 |
| 风险 | 尺寸误差 | 与第三方样式冲突需注意 | 全局策略要固定 |

## 代码

```css
* { box-sizing: border-box; }

.single-line {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}
```

### 示例说明

- 这段代码解决什么问题：统一盒模型并实现单行文本省略。
- 关键语句或参数含义：`box-sizing: border-box` 降低尺寸计算复杂度。
- 常见错误写法：仅写 `text-overflow: ellipsis` 而缺少 `overflow` 与 `white-space`。

## 真实场景示例

- 场景描述：表格列宽在不同页面出现截断和换行不一致。
- 问题表现：同样宽度下文案有的换行有的不换行。
- 处理方案：统一盒模型，明确文本溢出规则，标准化 `line-height`。
- 验证方式：多数据量快照对比、视觉回归、可读性验收。

## 常见误区

- 误区：`line-height` 只影响文字上下间距。
- 正解：它直接影响行盒高度和垂直对齐表现。
- 误区：`margin` 和 `padding` 只是“外边距和内边距”字面差异。
- 正解：二者会影响背景绘制范围与占位行为。

## 关联知识点

- 依赖：display 与可见性。
- 影响：布局系统、重排重绘成本。
- 对比：响应式单位与媒体查询。
- 组合：图像和替换元素尺寸策略。

## 相关面试题

1. 对盒模型的理解
- 标准答案：结论是盒模型由 `content`、`padding`、`border`、`margin` 组成。机制上标准盒模型 `width` 仅含内容区，`border-box` 含 `padding` 和 `border`。场景上组件化项目通常统一 `border-box`。
- 考察点：是否能准确解释两种盒模型计算差异。

2. 替换元素的概念及计算规则
- 标准答案：结论是替换元素内容由外部资源决定，如 `img`、`input`。机制上尺寸计算遵循固有尺寸、HTML 属性尺寸、CSS 尺寸优先级。场景上只设宽或高时会按固有比例缩放。
- 考察点：是否理解替换元素尺寸优先级。

3. margin 和 padding 的使用场景
- 标准答案：结论是需要元素外部留白用 `margin`，需要内部留白且背景延展用 `padding`。机制上二者作用于不同盒层。场景上按钮点击区扩大通常用 `padding`。
- 考察点：是否能从布局与视觉同时解释差异。

4. 对line-height 的理解及其赋值方式
- 标准答案：结论是 `line-height` 定义行盒高度，不只控制文字“看起来的间距”。机制上无单位数值会按比例继承，`px` 是固定值。场景上推荐用无单位值提升组件继承稳定性。
- 考察点：是否理解继承与实际计算差异。

5. 单行、多行文本溢出隐藏
- 标准答案：单行需 `overflow: hidden` + `text-overflow: ellipsis` + `white-space: nowrap`。多行常用 `display: -webkit-box` + `-webkit-line-clamp` + `-webkit-box-orient: vertical`。机制上多行省略依赖私有实现。场景上需评估浏览器兼容并准备降级样式。
- 考察点：是否掌握可落地配置与兼容边界。
