## 概念定义

- 本质定义：响应式单位与媒体查询用于让页面在不同尺寸和像素密度设备下保持可用与可读。
- 解决问题：统一多设备下的尺寸、排版和清晰度策略。
- 边界说明：它不替代具体布局实现，布局细节在“经典布局与 Flex”节点展开。

## 核心机制拆解

### 1) 机制/流程

1. 选单位：`px`、`%`、`em`、`rem`、`vw`、`vh`。
2. 设基准：通常以根字体和断点作为全局标尺。
3. 用媒体查询在关键断点调整布局与字号。
4. 对高像素密度设备提供合适图像资源（`@2x` / `@3x`）。

### 2) 关键条件与触发点

- 条件 A：根字体策略不稳定会导致 `rem` 体系抖动。
- 条件 B：只做宽度适配忽略像素密度时，图片会发虚。

### 3) 结果与影响

- 对性能/稳定性/可维护性的影响：策略统一时多端表现一致；策略分裂时维护成本和视觉偏差显著增加。

## 相似概念对比

| 维度 | `em` | `rem` | 结论 |
|---|---|---|---|
| 参照对象 | 父元素字体大小 | 根元素字体大小 | `rem` 更利于全局一致 |
| 级联影响 | 易连锁放大 | 可控性更高 | 大型项目多用 `rem` |
| 适用场景 | 局部相对缩放 | 全局响应式缩放 | 可组合使用 |
| 风险 | 深层嵌套难控 | 根字体变动影响全局 | 需统一规范 |

## 代码

```css
html { font-size: 16px; }
@media (max-width: 768px) {
  html { font-size: 14px; }
}
.card { padding: 1rem; }
```

### 示例说明

- 这段代码解决什么问题：通过调整根字体实现整体比例缩放。
- 关键语句或参数含义：`1rem` 会跟随 `html` 字体变化。
- 常见错误写法：断点很多但缺乏统一设计基准，导致样式碎片化。

## 真实场景示例

- 场景描述：同一页面在手机和平板上文本密度差异过大。
- 问题表现：小屏挤压严重，大屏留白过多。
- 处理方案：统一 `rem` 基准 + 关键断点媒体查询 + 高清资源策略。
- 验证方式：多设备截图比对、字号可读性评估、LCP（Largest Contentful Paint）图像质量检查。

## 常见误区

- 误区：响应式只需要媒体查询。
  - 正解：单位体系、断点策略、图片密度策略要一起设计。
- 误区：所有场景都该用 `rem`。
  - 正解：固定像素边框、细线场景仍常用 `px`。

## 关联知识点

- 依赖：盒模型与文本排版。
- 影响：布局系统、图像策略。
- 对比：发丝线与 1px 问题。
- 组合：工程化与设计令牌。

## 相关面试题

1. 常见的CSS布局单位
   - 标准答案：常见单位有 `px`、`%`、`em`、`rem`、`vw`、`vh`。机制上它们的参照系不同，`px` 固定，`rem` 相对根元素，`vw/vh` 相对视口。场景上移动端常组合 `rem + 媒体查询`。
   - 考察点：是否理解单位参照系差异。

2. px、em、rem的区别及使用场景
   - 标准答案：`px` 固定，`em` 相对父级，`rem` 相对根元素。机制上 `em` 易受层级级联影响，`rem` 更利于全局控制。场景上多端适配常用 `rem`，局部微调可用 `em`。
   - 考察点：是否能结合场景给出取舍。

3. 什么是物理像素，逻辑像素和像素密度，为什么在移动端开发时需要用到@3x, @2x这种图片？
   - 标准答案：结论是逻辑像素用于布局，物理像素由设备硬件决定，像素密度描述两者比值。机制上高密度屏需要更高分辨率资源避免模糊。场景上常用 `@2x` / `@3x` 配合媒体条件或 `srcset` 提供候选图。
   - 考察点：是否理解高清屏清晰度问题根因。

4. 对媒体查询的理解？
   - 标准答案：结论是媒体查询用于在特定设备条件下启用对应样式。机制上浏览器按媒体特征表达式结果决定规则是否生效。场景上断点设计应服务内容布局而不是只对齐设备型号。
   - 考察点：是否掌握断点策略而非死记语法。

5. 如何根据设计稿进行移动端适配？
   - 标准答案：结论是先建立设计基准，再确定单位体系和断点策略。机制上通过 `rem` 或 `vw` 缩放结合媒体查询实现比例适配。场景上要同时考虑触控区域、字体最小可读值和图像清晰度。
   - 考察点：是否具备从设计稿到代码的完整落地思路。

6. 响应式设计的概念及基本原理
   - 标准答案：结论是同一套内容通过弹性布局和媒体查询适配不同设备。机制上核心是流式尺寸、断点调整与资源按需切换。场景上应优先保证核心内容可达，再增强复杂交互。
   - 考察点：是否理解响应式不是“缩放页面”，而是“重排内容”。
