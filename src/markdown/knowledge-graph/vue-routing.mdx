## 代码

```ts
import { createRouter, createWebHistory } from 'vue-router'

const router = createRouter({
  history: createWebHistory(),
  routes: [
    { path: '/', component: () => import('./pages/Home.vue') },
    { path: '/admin', component: () => import('./pages/Admin.vue') },
  ],
})

router.beforeEach((to) => {
  const isLogin = Boolean(localStorage.getItem('token'))
  if (to.path === '/admin' && !isLogin) return '/'
})
```

### 示例说明

- 这段代码解决什么问题：演示路由懒加载和登录态导航守卫。
- 关键语句或参数含义：`beforeEach` 可中断或重定向导航流程。
- 常见错误写法：守卫里异步分支不返回明确结果，导致导航状态不一致。

## 相关面试题

1. 路由的 hash 和 history 模式的区别
- 标准答案：结论是 hash 依赖 `#` 片段、history 依赖 History API。机制上 hash 不会触发服务端 404，history 需要服务端兜底到同一入口。场景上静态托管常先用 hash，正规生产站点更常用 history。
- 考察点：是否能从定义、触发时机、性能影响和适用场景四个维度做对比。

2. 如何获取页面的 hash 变化
- 标准答案：结论是监听 `window.onhashchange` 或 `addEventListener("hashchange")`。机制上 URL 片段变化会触发该事件。场景上手写路由或埋点场景可直接监听，框架项目优先用路由库钩子。
- 考察点：是否能给出可执行步骤、关键 API 和常见踩坑点。

3. $route 和 $router 的区别
- 标准答案：结论是 `$route` 是当前路由信息对象，`$router` 是路由实例。机制上前者偏“读”，后者偏“操作导航”。场景上取参数看 `$route`，跳转和守卫注册用 `$router`。
- 考察点：是否能从定义、触发时机、性能影响和适用场景四个维度做对比。

4. 如何定义动态路由？如何获取传过来的动态参数？
- 标准答案：结论是通过路径参数如 `/user/:id` 定义动态路由。机制上路由匹配后把参数放到 `params`，查询参数放 `query`。场景上资源详情页用 `params`，筛选条件更适合 `query`。
- 考察点：是否能给出可执行步骤、关键 API 和常见踩坑点。

5. Vue-router 路由钩子在生命周期的体现
- 标准答案：结论是路由钩子优先于组件渲染生命周期。机制上常见顺序是全局守卫 -> 路由独享守卫 -> 组件守卫 -> 组件挂载。场景上鉴权与重定向放路由守卫，页面初始化逻辑放组件生命周期。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

6. params 和 query 的区别
- 标准答案：结论是 `params` 绑定路径、`query` 绑定查询串。机制上 `params` 更强调资源标识，`query` 更强调筛选条件。场景上分享链接和刷新保留状态时，`query` 可读性更高。
- 考察点：是否能从定义、触发时机、性能影响和适用场景四个维度做对比。

7. Vue-router 导航守卫有哪些
- 标准答案：结论是主要有全局守卫（`beforeEach/afterEach`）、路由独享守卫、组件内守卫。机制上它们串联成导航决策链，可中断、重定向或放行。场景上登录鉴权、权限校验和离开确认是典型应用。
- 考察点：是否能完整列举关键项，并说明每一项的适用场景。

8. 对前端路由的理解
- 标准答案：结论是前端路由是在不整页刷新的前提下完成 URL 与视图映射。机制上监听 URL 变化并渲染对应组件。场景上 SPA 依赖它做状态化导航，多页站点则更多依赖服务端路由。
- 考察点：是否能给出明确结论，并补充机制和业务场景。
