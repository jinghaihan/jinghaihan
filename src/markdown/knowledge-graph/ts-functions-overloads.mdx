## 概念定义

- 本质定义：函数类型与重载关注如何约束参数、返回值、`this` 上下文和多种调用签名。
- 解决问题：让函数“能怎么被调用、调用后会返回什么”在签名层面说清楚。
- 边界说明：本节点聚焦函数边界，不展开对象建模和模块系统。

## 核心机制拆解

### 1) 机制/流程

1. 先给函数参数和返回值写清类型。
2. 当同一函数需要支持多种调用方式时，用重载签名表达不同输入输出关系。
3. 涉及回调或对象方法时，再明确 `this` 类型，避免运行时绑定混乱。
4. 异步函数统一通过 `Promise` 返回结果，类型约束也跟着进入异步链。

### 2) 关键条件与触发点

- 条件 A：只写实现签名不写重载签名时，调用方经常拿不到准确推断。
- 条件 B：忽略 `this` 类型时，回调和对象方法最容易出现上下文错误。
- 条件 C：异步函数返回值类型不明确时，后续 `await` 链路会越来越模糊。

### 3) 结果与影响

- 对性能/稳定性/可维护性的影响：函数边界越清楚，公共工具和组合函数越好复用；边界越模糊，错误就越容易沿调用链扩散。

## 相似概念对比

| 维度 | 单一签名函数 | 重载函数 | 结论 |
|---|---|---|---|
| 调用方式 | 一种主要输入输出关系 | 多种明确调用关系 | 复杂公共函数更适合重载 |
| 推断效果 | 简单直接 | 更精细 | 但实现要更谨慎 |
| 维护成本 | 低 | 更高 | 不是所有函数都该上重载 |
| 适用场景 | 业务函数、局部工具函数 | 公共库函数、兼容多入参 API | 看调用复杂度决定 |

## 代码

```ts
function format(value: number): string
function format(value: Date): string
function format(value: number | Date): string {
  return typeof value === 'number'
    ? value.toFixed(2)
    : value.toISOString()
}

const tool = {
  label: 'search',
  handle(this: { label: string }) {
    return this.label
  },
}
```

### 示例说明

- 这段代码解决什么问题：同一个函数支持多种入参，并显式声明对象方法里的 `this` 类型。
- 关键语句或参数含义：前两行是重载签名；最后一行实现签名必须兼容所有重载；`this: { label: string }` 用来约束调用上下文。
- 常见错误写法：只写联合入参的实现签名，导致调用方无法根据不同入参拿到准确返回值推断。

## 真实场景示例

- 场景描述：公共格式化函数既要接时间戳，也要接 `Date`，结果调用方每次都要自己判断返回值行为。
- 问题表现：函数内部能跑，但调用方体验很差，自动提示不准，后续重构也容易误用。
- 处理方案：把不同入参的调用关系提升到重载签名，同时给回调里的 `this` 和异步返回值补清楚类型。
- 验证方式：看调用点推断是否准确、重载实现是否仍可维护，以及异步链返回值是否稳定。

## 常见误区

- 误区：函数重载就是多写几次同名函数实现。
- 正解：TypeScript 只有一份实现，前面写的是多个重载签名。
- 误区：箭头函数天然解决所有 `this` 问题。
- 正解：箭头函数继承外层 `this`，但对象方法和库回调仍然需要明确边界。
- 误区：异步函数返回值写不写都一样，反正最后都 `await`。
- 正解：返回值类型会沿调用链传播，越早写清楚越稳。

## 关联知识点

- 依赖：基础类型与类型推断、对象建模与接口。
- 影响：泛型工具函数、公共库 API、框架回调签名。
- 对比：单一签名和重载签名。
- 组合：`this` 约束、异步函数、函数式工具库。

## 相关面试题

1. 解释 TypeScript 中的`this` 和`=>` (箭头函数)。
- 标准答案：普通函数的 `this` 取决于调用方式，而箭头函数不会创建自己的 `this`，会继承外层上下文。机制上 TypeScript 允许你显式声明函数的 `this` 类型，提前约束调用方式。场景上事件回调、对象方法和类实例方法里，这个区别特别关键。
- 考察点：是否把“语法差异”讲到上下文绑定和类型约束层。

2. TypeScript 如何处理函数重载？
- 标准答案：通过多个重载签名描述不同的调用方式，最后再写一份兼容所有情况的实现签名。机制上调用方看到的是前面的重载签名，真正运行的是最后一份实现。场景上公共工具函数、兼容多种入参的库 API 很适合用重载，但业务函数不要滥用。
- 考察点：是否理解“多签名 + 单实现”的规则。

3. 如何在 TypeScript 中使用`async/await`？
- 标准答案：写法上和 JavaScript 一样，但要明确异步函数返回的是 `Promise<T>`。机制上 `await` 会把 `Promise` 解析后的类型继续传下去。场景上请求函数、异步数据加载和服务封装都应该把最终解析值的类型写清楚。
- 考察点：是否知道异步类型会沿函数边界传播。
