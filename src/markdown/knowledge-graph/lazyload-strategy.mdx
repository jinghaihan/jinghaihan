## 概念定义

- 本质定义：懒加载与按需加载是在资源尚未需要时延后请求，等进入可视区或触发条件满足后再加载。
- 解决问题：解决长列表、大量图片、低频模块造成的首屏负担问题。
- 边界说明：不解决图片本身体积过大或网络链路抖动问题。

## 核心机制拆解

### 1) 机制/流程

1. 初始渲染时仅加载占位资源或骨架。
2. 把真实资源地址放在 `data-*` 或模块动态导入入口。
3. 监听可视区或用户行为触发点。
4. 触发后将真实地址赋值给 `src` 或执行动态导入。
5. 资源加载完成后替换占位并取消监听。

### 2) 关键条件与触发点

- 条件 A：页面存在长列表、图片墙或低频功能模块时，懒加载收益高。
- 条件 B：触发阈值设置过晚会导致用户看到空白，过早会损失节流效果。

### 3) 结果与影响

- 对性能的影响：可减少首屏请求数和下载体积。
- 对稳定性的影响：需要处理失败重试和占位兜底，避免白块或闪烁。

## 相似概念对比

| 维度 | 懒加载 | 一次性全量加载 | 结论 |
|---|---|---|---|
| 触发时机 | 进入可视区或触发行为后 | 页面初始加载阶段 | 懒加载更省首屏资源 |
| 首屏成本 | 低 | 高 | 体积差异明显 |
| 交互稳定性 | 需处理占位与加载状态 | 逻辑简单但成本高 | 按场景取舍 |
| 适用场景 | 长列表、低频模块 | 小页面、资源很少 | 不应一刀切 |

## 代码

```ts
const observer = new IntersectionObserver((entries) => {
  for (const entry of entries) {
    if (!entry.isIntersecting) continue
    const img = entry.target as HTMLImageElement
    const realSrc = img.dataset.src
    if (realSrc) img.src = realSrc
    observer.unobserve(img)
  }
}, { rootMargin: '100px 0px' })

document.querySelectorAll<HTMLImageElement>('img[data-src]').forEach((img) => {
  observer.observe(img)
})
```

### 示例说明

- 这段代码用可视区观察实现图片按需加载，避免滚动事件高频计算。
- `rootMargin` 可以提前预加载，减少用户滚到目标位置时的空白。
- 常见错误写法是每次滚动都全量遍历所有图片，导致主线程负担增加。

## 真实场景示例

- 场景描述：商品列表页首屏只展示 8 张图，但页面总图数超过 200。
- 问题表现：首次加载时间长，弱网下首屏等待明显。
- 处理方案：首屏内图片正常加载，首屏外图片改懒加载并加占位图。
- 验证方式：观察首屏资源体积、首屏时间和滚动加载成功率。

## 常见误区

- 误区：懒加载就是把 `src` 设为空。
- 正解：关键是触发机制、占位策略和失败兜底，不是单一属性替换。
- 误区：只要上懒加载就一定更快。
- 正解：如果阈值或监听策略错误，会引入滚动卡顿和图片闪烁。

## 关联知识点

- 依赖：资源预提示优化、图片优化策略。
- 影响：回流、重绘与渲染优化。
- 组合：Webpack 构建与体积优化。

## 相关面试题

1. 懒加载的概念
- 标准答案：懒加载是把非首屏资源的下载时机延后到“即将使用”或“已经可见”时，再发起请求。它的核心收益是减少首屏无效加载和带宽占用，常用于长列表和图片密集页面。
- 考察点：是否理解懒加载的目标是“按需时机”，不是“永不加载”。

2. 懒加载的特点
- 标准答案：主要特点是减少无效请求、降低首屏压力、提升首屏体验，但会增加加载状态管理复杂度。若占位与触发策略做得差，用户会感知白屏或闪烁。
- 考察点：是否能同时说明收益和副作用。

3. 懒加载的实现原理
- 标准答案：先保存真实资源地址，再在元素进入可视区时把真实地址赋值给 `src` 或触发动态导入。机制关键是“可视区判断 + 触发赋值 + 卸载监听”，而不是简单监听滚动。
- 考察点：是否能说清触发条件和执行链路。
```ts
const observer = new IntersectionObserver((entries) => {
  entries.forEach((entry) => {
    if (!entry.isIntersecting) return
    const img = entry.target as HTMLImageElement
    const src = img.dataset.src
    if (src) img.src = src
    observer.unobserve(img)
  })
}, { rootMargin: '120px 0px' })

document.querySelectorAll<HTMLImageElement>('img[data-src]').forEach((img) => {
  observer.observe(img)
})
```
- 追问 1：为什么推荐 `IntersectionObserver`？
- 标准答案 1：因为它由浏览器调度可见性变化，避免手写滚动计算导致高频回调和布局查询开销。
