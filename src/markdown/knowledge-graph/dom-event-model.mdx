## 知识体系概览

浏览器原理体系解决“资源如何被加载、解析、渲染、交互与回收”的问题，是前端性能、稳定性和安全性的底层基础。

## 能力边界

- 覆盖范围：浏览器多进程模型、渲染线程协作、渲染流水线、缓存命中路径、事件派发机制、本地存储边界与安全风险。
- 非覆盖范围：后端接口语义、业务领域模型、框架组件设计细节。

## 核心子主题

1. 浏览器架构与内核：理解浏览器组成和渲染内核职责。
2. 浏览器进程模型：明确浏览器进程、渲染进程、网络进程分工。
3. 渲染进程线程模型：理解主线程、渲染线程、网络回调协作。
4. 浏览器安全机制：覆盖 XSS、CSRF、同源与网络劫持防护。
5. 刷新与缓存判定：区分普通刷新、强制刷新、地址栏回车。
6. Service Worker 拦截链：离线缓存、请求接管与更新策略。
7. 渲染流水线：HTML 解析、样式计算、布局、绘制、合成。
8. 解析阻塞治理：脚本与样式阻塞、关键渲染路径优化。
9. 存储与事件机制：Cookie、LocalStorage、SessionStorage、IndexedDB 与事件派发。
10. 垃圾回收与内存泄漏：理解回收机制和常见泄漏场景。

## 推荐学习路径

- 入门顺序：浏览器架构与内核 -> 浏览器进程模型 -> 渲染流水线 -> 刷新与缓存判定 -> 解析阻塞治理 -> 存储与安全边界 -> 事件派发机制 -> 垃圾回收与内存泄漏。
- 面试优先顺序：渲染流水线 -> 解析阻塞治理 -> 浏览器缓存 -> 浏览器安全机制 -> 事件机制 -> 垃圾回收。
- 项目落地顺序：先建立性能观测和缓存命中路径 -> 再治理阻塞与回流重绘 -> 最后完善安全与内存稳定性。

## 常见混淆边界

- 渲染进程 vs 浏览器进程：前者负责页面内容执行和渲染，后者负责窗口管理和进程协调。
- 回流 vs 重绘：前者涉及几何信息变化，后者仅涉及视觉样式变化。
- 强缓存 vs 协商缓存：前者本地直接复用，后者仍需向服务端校验。
- 事件委托 vs 事件冒泡：前者是监听策略，后者是事件传播机制。

## 相关知识岛

- 计算机网络体系：缓存策略、状态码路径与浏览器缓存命中强关联。
- HTML 体系：文档结构与资源引入方式直接影响解析和阻塞。
- JavaScript 体系：事件循环、宏任务与微任务决定主线程调度节奏。
- 性能优化体系：渲染阶段指标和瓶颈治理依赖浏览器原理。
