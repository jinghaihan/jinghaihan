## 概念定义

- 本质定义：手写数据处理题是对数组、字符串、数字、对象转换能力的综合验证。
- 解决问题：解决集合变换、格式转换和结构重组等高频工程需求。
- 边界说明：不处理复杂图算法和系统设计问题。

## 核心机制拆解

### 1) 机制/流程

1. 明确输入结构和输出结构。
2. 选择遍历策略与中间状态结构。
3. 控制时间复杂度与空间复杂度。
4. 补齐空值、重复值与极端输入边界。

### 2) 关键条件与触发点

- 条件 A：数组去重、扁平化、树转换题通常考察哈希和递归。
- 条件 B：大数运算题要求字符串按位运算，不能直接依赖 Number。

### 3) 结果与影响

- 对面试表现的影响：可快速体现编码稳定性与边界意识。
- 对工程实践的影响：常直接沉淀为业务工具函数。

## 相似概念对比

| 维度 | map | forEach | 结论 |
|---|---|---|---|
| 返回值 | 新数组 | `undefined` | `map` 更适合转换 |
| 语义 | 一进一出映射 | 侧重副作用执行 | 语义应匹配场景 |
| 可链式 | 可以 | 不可以 | 组合能力不同 |
| 常见误用 | 只做副作用 | 期待返回新数组 | 需按意图使用 |

## 相关面试题

1. 实现数组的扁平化
   - 标准答案：可以用递归、栈或原生 `flat`，面试手写常用 `reduce + 递归`。关键点是支持任意层嵌套和空数组边界，而不是只拍平一层。
   - 考察点：递归思维与边界处理。
```ts
function flatDeep(arr: unknown[]): unknown[] {
  return arr.reduce<unknown[]>((acc, cur) => {
    if (Array.isArray(cur)) return acc.concat(flatDeep(cur))
    acc.push(cur)
    return acc
  }, [])
}
```

2. 实现数组去重
   - 标准答案：基础场景可直接 `Set` 去重，复杂对象数组要先定义唯一键再去重。只说“用 Set”不够，必须补上对象去重的判定规则。
   - 考察点：是否知道不同数据结构的去重策略。
```ts
function unique<T>(arr: T[]): T[] {
  return [...new Set(arr)]
}
```

3. 实现数组的 map 方法
   - 标准答案：遍历原数组，回调结果写入新数组并返回；实现时要保留 `thisArg` 和稀疏数组语义。`map` 的本质是一进一出映射，不应只做副作用。
   - 考察点：是否理解原生方法语义。
```ts
function myMap<T, R>(arr: T[], cb: (item: T, index: number, source: T[]) => R, thisArg?: unknown): R[] {
  const result: R[] = new Array(arr.length)
  for (let i = 0; i < arr.length; i += 1) {
    if (!(i in arr)) continue
    result[i] = cb.call(thisArg, arr[i], i, arr)
  }
  return result
}
```

4. 实现字符串的 repeat 方法
   - 标准答案：可用循环或二进制快速幂拼接，核心是处理非整数和负数异常。线上高频场景是生成占位串和日志前缀。
   - 考察点：是否关注非法输入边界。
```ts
function myRepeat(str: string, count: number) {
  if (!Number.isInteger(count) || count < 0)
    throw new RangeError('Invalid count value')
  let times = count
  let base = str
  let result = ''
  while (times > 0) {
    if (times & 1) result += base
    base += base
    times >>= 1
  }
  return result
}
```

5. 将数字每千分位用逗号隔开
   - 标准答案：可用正则或分组拼接实现，必须处理负数、小数和空值。只处理正整数的答案在业务里会频繁翻车。
   - 考察点：是否考虑格式边界。
```ts
function thousandFormat(input: string | number) {
  const [intPart, decimalPart = ''] = String(input).split('.')
  const sign = intPart.startsWith('-') ? '-' : ''
  const intAbs = sign ? intPart.slice(1) : intPart
  const withComma = intAbs.replace(/\B(?=(\d{3})+(?!\d))/g, ',')
  return decimalPart ? `${sign}${withComma}.${decimalPart}` : `${sign}${withComma}`
}
```

6. 实现非负大整数相加和相乘
   - 标准答案：结论是字符串按位运算。加法用双指针加进位，乘法用双层循环把每位乘积累计到结果数组。这样在超长数值下不会触发 Number 精度丢失。
   - 考察点：是否能给出精度安全实现。
```ts
function addBigInt(a: string, b: string) {
  let i = a.length - 1
  let j = b.length - 1
  let carry = 0
  let result = ''
  while (i >= 0 || j >= 0 || carry) {
    const x = i >= 0 ? Number(a[i]) : 0
    const y = j >= 0 ? Number(b[j]) : 0
    const sum = x + y + carry
    result = (sum % 10) + result
    carry = Math.floor(sum / 10)
    i -= 1
    j -= 1
  }
  return result
}

function multiplyBigInt(a: string, b: string) {
  if (a === '0' || b === '0') return '0'
  const result = new Array(a.length + b.length).fill(0)

  for (let i = a.length - 1; i >= 0; i -= 1) {
    for (let j = b.length - 1; j >= 0; j -= 1) {
      const mul = Number(a[i]) * Number(b[j])
      const p1 = i + j
      const p2 = i + j + 1
      const sum = mul + result[p2]
      result[p2] = sum % 10
      result[p1] += Math.floor(sum / 10)
    }
  }

  while (result[0] === 0) result.shift()
  return result.join('')
}
```

7. 解析 URL Params 为对象
   - 标准答案：按 `&` 和 `=` 拆分并 `decodeURIComponent`，重复 key 转数组。要覆盖 `?` 前缀、空值和中文编码。
   - 考察点：是否具备鲁棒解析思维。
```ts
function parseQuery(search: string): Record<string, string | string[]> {
  const query = search.startsWith('?') ? search.slice(1) : search
  if (!query) return {}

  return query.split('&').reduce<Record<string, string | string[]>>((acc, item) => {
    if (!item) return acc
    const [rawKey, rawVal = ''] = item.split('=')
    const key = decodeURIComponent(rawKey)
    const value = decodeURIComponent(rawVal)

    if (acc[key] === undefined) acc[key] = value
    else if (Array.isArray(acc[key])) acc[key].push(value)
    else acc[key] = [acc[key] as string, value]

    return acc
  }, {})
}
```
