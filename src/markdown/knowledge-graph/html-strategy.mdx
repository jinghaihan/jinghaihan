## 知识体系概览

HTML 体系解决“文档如何被浏览器正确解析、组织、加载并呈现”的问题，是前端渲染链路的结构入口。

## 能力边界

- 覆盖范围：文档声明与 head 配置、语义结构、加载阻塞、脚本时序、HTML5 能力、媒体图形、iframe 通信、Worker、离线存储、兼容策略。
- 非覆盖范围：CSS 计算与布局算法细节、JavaScript 语言机制、网络传输协议实现。

## 核心子主题

1. 文档声明与 head 配置：Doctype、`head`、`meta`、编码与解析模式。
2. 语义化与可访问性：语义标签、标题层级、表单关联、标签分类。
3. 加载与阻塞模型：`src` 和 `href` 语义差异与阻塞链路。
4. script 加载策略（defer / async）：下载并行、执行时机、顺序约束。
5. HTML5 能力总览：语义标签、表单增强、存储、拖拽与图形能力。
6. 媒体与图形：`img` `srcset`、Canvas、SVG 的选型。
7. iframe 嵌入与跨域通信：同源边界、`postMessage` 协议与安全约束。
8. Web Worker：主线程减负与消息通信。
9. 离线存储与 Service Worker：历史 `.appcache` 与现代离线策略。
10. 渐进增强与优雅降级：跨能力层级的兼容设计方法。

## 推荐学习路径

- 入门顺序：文档声明与 head 配置 -> 语义化与可访问性 -> 加载与阻塞模型 -> script 加载策略（defer / async） -> HTML5 能力总览 -> 媒体与图形 -> iframe 嵌入与跨域通信 -> Web Worker -> 离线存储与 Service Worker -> 渐进增强与优雅降级
- 面试优先顺序：script 加载策略（defer / async） -> 加载与阻塞模型 -> 文档声明与 head 配置 -> iframe 嵌入与跨域通信 -> 语义化与可访问性 -> 离线存储与 Service Worker -> 媒体与图形 -> Web Worker -> HTML5 能力总览 -> 渐进增强与优雅降级
- 项目落地顺序：先治理阻塞和脚本时序 -> 再补齐语义和文档声明与 head 配置 -> 再做媒体图形与 iframe 通信 -> 再做离线能力 -> 最后沉淀兼容策略

## 常见混淆边界

- `src` 和 `href`：`src` 偏资源嵌入，`href` 偏关系引用。
- `defer` 和 `async`：前者保序，后者不保序。
- Canvas 和 SVG：前者像素绘制，后者对象化矢量。
- `.appcache` 和 `Service Worker`：前者历史机制，后者现代可编程缓存。

## 相关知识岛

- 浏览器原理体系：HTML 输出会进入解析、渲染和缓存链路。
- CSS 体系：语义结构和资源加载顺序会影响样式计算与首屏展示。
- JavaScript 体系：脚本执行时序和 Worker 模型与 HTML 加载阶段耦合。
- 性能优化体系：关键资源组织、图像策略和离线策略直接影响性能指标。
