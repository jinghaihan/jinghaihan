## 知识体系概览

HTML 体系解决“页面结构如何被声明、浏览器如何理解这些结构、资源如何被发现并按顺序进入渲染链路”的问题，是前端所有可见内容的入口层。

## 能力边界

- 覆盖范围：文档声明、`head` 元信息、语义化结构、表单与原生控件、资源引用、脚本加载策略、HTML5 原生能力、媒体图形、`iframe`、Worker、离线资源管理、兼容设计。
- 非覆盖范围：CSS 的布局和绘制细节、JavaScript 的执行模型、HTTP（HyperText Transfer Protocol）协议语义与缓存策略。

## 核心子主题

1. 文档声明与 head 配置：Doctype、`head`、`meta`、编码与解析模式。
2. 语义化与可访问性：语义标签、标题层级、文本语义和基础结构标签。
3. 表单与原生交互控件：表单结构、原生校验、焦点顺序和控件语义。
4. 加载与阻塞模型：`src` 和 `href` 差异、链接关系和首屏阻塞链路。
5. script 加载策略（defer / async）：下载并行、执行时机和顺序约束。
6. HTML5 能力总览：数据属性、拖拽、模板、插槽和原生增强能力。
7. 媒体与图形：图片、音视频、Canvas、SVG 和响应式媒体策略。
8. iframe 嵌入与跨域通信：隔离上下文、权限收敛和消息通信。
9. Web Worker：把计算任务移出主线程。
10. 离线存储与 Service Worker：历史 `.appcache`、缓存策略和离线资源加载。
11. 渐进增强与优雅降级：能力检测、回退路径和 `noscript` 兜底。

## 推荐学习路径

- 入门顺序：文档声明与 head 配置 -> 语义化与可访问性 -> 表单与原生交互控件 -> 加载与阻塞模型 -> script 加载策略（defer / async） -> 媒体与图形 -> HTML5 能力总览 -> iframe 嵌入与跨域通信 -> Web Worker -> 离线存储与 Service Worker -> 渐进增强与优雅降级
- 面试优先顺序：script 加载策略（defer / async） -> 加载与阻塞模型 -> 文档声明与 head 配置 -> 语义化与可访问性 -> 表单与原生交互控件 -> iframe 嵌入与跨域通信 -> 离线存储与 Service Worker -> 媒体与图形 -> HTML5 能力总览 -> Web Worker -> 渐进增强与优雅降级
- 项目落地顺序：先保证文档声明、`head` 和基础语义正确 -> 再治理脚本阻塞和表单可用性 -> 再做媒体、`iframe` 和原生增强能力 -> 再接离线能力 -> 最后补齐兼容策略

## 常见混淆边界

- `src` 和 `href`：`src` 偏资源嵌入，`href` 偏关系引用。
- `defer` 和 `async`：前者保序，后者不保序。
- `title` 和 `h1`：前者是文档标题，后者是正文主标题。
- Canvas 和 SVG：前者像素绘制，后者对象化矢量。
- `.appcache` 和 `Service Worker`：前者历史机制，后者现代可编程缓存。

## 相关知识岛

- 浏览器原理体系：HTML 输出会进入解析、渲染和缓存链路。
- CSS 体系：语义结构和资源加载顺序会影响样式计算与首屏展示。
- JavaScript 体系：脚本执行时序、Worker 模型和 DOM 操作都与 HTML 入口耦合。
- 性能优化体系：关键资源组织、图像策略和离线策略直接影响性能指标。
