## 概念定位

HTML 策略节点关注文档组织、关键内容前置和资源嵌入策略，决定解析起跑线。

它影响首屏结构可见时间和后续样式脚本阻塞程度。

## 工作机制

1. 先输出稳定语义骨架和关键首屏内容。
2. 控制首屏 HTML 体积，避免过大阻塞传输。
3. 将关键样式和脚本策略化放置（内联/延后/异步）。
4. 为媒体与第三方资源设置懒加载和占位。
5. 保证文档结构可访问且便于解析。

## 关键知识点

### 必须掌握

- HTML 结构决定解析顺序和关键资源发现时机。
- 关键内容前置可缩短首屏感知时间。
- 过度内联会拉大 HTML 体积。
- 第三方脚本应降级优先级。
- 语义结构有助于 SEO 与可访问性。
- 模板渲染需控制首屏冗余节点。

### 选型与取舍

- 更多关键内容前置可提感知速度，但初始体积增加。
- 极简 HTML 传输快，但可能导致首屏骨架不足。
- 激进延后脚本可提首屏，但交互就绪可能变慢。

## 对页面性能与稳定性的影响

- 节点稳定时，Knowledge Graph 主线 的网络与渲染链路更可预测，性能治理收益更稳定。
- 节点失稳时，问题会向后续阶段扩散，常表现为首屏变慢、交互卡顿或版本一致性风险。
- 优化动作应结合上下游联动验证，避免局部优化把成本转移到其他节点。

## 观测与排障

### 核心指标

- HTML 文档大小
- 首屏可见时间
- 关键资源发现时延
- 首屏结构化可访问评分

### 排障顺序

1. 先看 HTML 体积和首包传输耗时。
2. 检查关键内容是否足够前置。
3. 检查第三方脚本位置和执行时机。
4. 检查图片/iframe 是否懒加载。
5. 对比优化前后首屏路径。

### 常见故障场景

- 首包很大导致解析启动晚。
- 关键文本出现太晚。
- 第三方脚本阻塞首屏。
- 首屏结构缺失导致用户感知差。

## 常见误区

- 只看 JS 优化，忽视 HTML 组织。
- 为图省事把大量内容一次性塞入首包。
- 关键内容依赖后置脚本注入。

## 题库深挖补充

### 高频对比与边界（基于题库）

- HTML 解析、CSS 计算、渲染树构建是连续链路，任何一段阻塞都会延后首屏可见。
- async/defer/module 的选择会改变脚本执行时机与阻塞模式。
- 关键 CSS 提取能提速，但内联过多会抬高 HTML 体积。
- render tree 不包含所有 DOM 节点，display:none 等节点不会参与渲染树。
- 语义结构不仅影响 SEO，也影响后续样式和脚本维护成本。
- 面试常问“为什么首屏慢”，根因往往是关键路径资源组织不合理。

### 线上排障剧本

1. 先画出首屏关键资源链（HTML、CSS、关键脚本、字体）。
2. 检查阻塞脚本和阻塞样式是否可延后或拆分。
3. 检查首屏 DOM 规模与无效节点比例。
4. 检查样式计算热点是否由复杂选择器触发。
5. 回归冷缓存、弱网和中低端设备。

### 高频追问补充（附答案）

1. defer 一定优于 async 吗？

不一定。defer 更可控，async 更激进，取决于依赖关系。

2. 为什么说 HTML 结构会影响性能？

结构决定资源发现和解析路径，也决定渲染树规模。

3. render tree 和 DOM tree 的关系？

render tree 基于 DOM + CSSOM 生成，只包含参与渲染的节点。

## 进阶追问（补充）

1. 如果网络指标正常但用户仍感知慢，优先看哪一层？

优先看主线程任务和渲染阶段（Layout/Paint/Composite）是否拥塞，网络快不代表交互快。

2. 缓存策略和发布策略为什么必须一起设计？

缓存决定资源复用路径，发布决定资源版本切换；两者脱节会出现版本错配。

3. 如何验证一次链路优化不是偶然收益？

至少做对照组、分位数对比，并覆盖冷缓存与弱网样本。

## 案例追问加深（第三轮）

### 真实故障片段

- 故障表现：同一版本在不同地区首屏耗时差异明显，且刷新方式不同结果不同。
- 排查结论：网络链路、缓存路径和渲染调度叠加导致体验分化，单点优化无法彻底解决。
- 治理动作：建立链路分层观察面板，按导航类型、缓存命中层和渲染阶段联合分析。

### 追问补充

1. 为什么链路问题经常不是单点根因？

Knowledge Graph 本身是多阶段因果链，一个阶段的波动会在后续阶段被放大。

2. 如何快速缩小排障范围？

先固定触发方式和样本维度，再按网络、缓存、渲染三层逐步排除。

## 面试补充

### 30 秒说明

HTML 策略是渲染链路的起跑姿势。结构和体积设计得当，可显著改善首屏与解析效率。

### 高频追问

1. 为什么 HTML 体积会影响性能？

体积越大，传输和解析启动越慢。

2. 关键内容前置有什么价值？

用户可更早看到有效信息，感知速度提升。

3. 第三方脚本怎么放置更稳妥？

尽量异步或延后，避免阻塞关键路径。

4. HTML 策略和 SEO 有什么关系？

语义结构清晰更利于抓取和内容理解。
