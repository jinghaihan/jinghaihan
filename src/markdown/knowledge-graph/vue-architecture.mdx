## 知识体系概览

Vue 体系解决组件化开发中的状态组织、渲染更新、路由流转与发布治理问题。

## 能力边界

- 覆盖范围：响应式、模板编译、组件通信、路由、状态、调度、性能、测试发布。
- 非覆盖范围：浏览器底层协议实现与后端系统设计。

## 核心子主题

1. 响应式核心：解释数据变化到视图更新。
2. 模板编译：解释指令和模板执行路径。
3. 通信与路由：解释页面流转和组件协作。
4. 状态与性能：解释共享状态和优化治理。

## 推荐学习路径

- 入门顺序：响应式 -> 通信 -> 路由 -> 状态 -> 调度 -> 性能与发布。
- 面试优先顺序：响应式/生命周期 -> 通信/路由 -> 状态管理 -> 性能优化。
- 项目落地顺序：先边界治理，再性能治理，最后规范闭环。

## 常见混淆边界

- computed vs watch：前者是派生值，后者是副作用。
- v-if vs v-show：前者控制创建销毁，后者控制显示隐藏。

## 相关知识岛

- JavaScript 与手写体系：Vue 核心机制依赖 JS 运行时。
- 浏览器原理体系：渲染时序影响组件更新表现。
- 工程化体系：构建与发布策略直接影响 Vue 线上稳定性。
