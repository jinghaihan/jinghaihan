## 概念定义

- 本质定义：数据库访问与缓存策略关注 Node.js 服务如何读写持久化数据，以及如何用缓存减少重复查询和热点压力。
- 解决问题：解释“为什么数据库连通不等于接口就稳定”，也解释“为什么上缓存之后还会有脏数据和穿透问题”。
- 边界说明：本节点聚焦 Node.js 侧的数据接入和缓存模式，不展开数据库索引设计或分库分表。

## 核心机制拆解

### 1) 机制/流程

1. 服务启动时初始化连接池或 ORM 客户端，避免每个请求都重新建连。
2. 读请求优先尝试缓存，未命中再查数据库并回填缓存。
3. 写请求更新数据库后，再删除或更新缓存，维持一致性。
4. 不同数据按访问频率和一致性要求选择内存缓存、Redis 或 HTTP 缓存。

### 2) 关键条件与触发点

- 条件 A：数据库连接必须池化，否则高并发下建连成本会很高。
- 条件 B：缓存命中率高不代表绝对正确，失效策略和回填时机决定一致性风险。
- 条件 C：本地内存缓存只能覆盖单进程，跨实例共享通常要靠 Redis 一类外部缓存。

### 3) 结果与影响

- 对性能/稳定性/可维护性的影响：连接池和缓存策略配合得当时，接口 RT 和数据库压力都会下降；配不好时，缓存雪崩、脏数据和击穿问题会一起出现。

## 相似概念对比

| 维度 | 内存缓存 | Redis 缓存 | HTTP 缓存 | 结论 |
|---|---|---|---|
| 作用范围 | 单进程内 | 多实例共享 | 客户端和代理层 | 层级不同 |
| 访问速度 | 最快 | 很快 | 取决于链路命中 | 命中层级越前越省 |
| 一致性治理 | 最难跨实例同步 | 可集中治理 | 依赖缓存头 | 要按场景组合 |
| 适用场景 | 配置、低量热点 | 会话、热点数据、排行榜 | 静态资源、可协商接口 | 组合使用更常见 |

## 代码

```js
const mysql = require('mysql2/promise')
const redis = require('redis')

const pool = mysql.createPool({
  host: 'localhost',
  user: 'root',
  database: 'demo',
  connectionLimit: 10,
})

const client = redis.createClient()

async function getUser(id) {
  const cacheKey = `user:${id}`
  const cached = await client.get(cacheKey)
  if (cached)
    return JSON.parse(cached)

  const [rows] = await pool.execute('SELECT * FROM users WHERE id = ?', [id])
  const user = rows[0]
  await client.setEx(cacheKey, 300, JSON.stringify(user))
  return user
}
```

### 示例说明

- 这段代码解决什么问题：展示连接池加缓存旁路模式的最小实现。
- 关键语句或参数含义：`createPool()` 用于复用数据库连接；`setEx()` 负责带过期时间写入缓存；缓存未命中时回源数据库。
- 常见错误写法：每次请求新建数据库连接、写库后不清缓存、把永不过期缓存当默认方案。

## 真实场景示例

- 场景描述：用户详情接口在大促期间成为热点，数据库 RT 快速抬高。
- 问题表现：服务直接查库，热点用户被重复读取，数据库连接数逼近上限。
- 处理方案：引入连接池和 Redis 缓存，按缓存旁路模式回填，并对写接口补缓存失效逻辑。
- 验证方式：看数据库连接数、缓存命中率、热点接口 RT 和脏数据投诉是否下降。

## 常见误区

- 误区：用了 ORM 就不用关注数据库连接。
- 正解：ORM 仍然依赖底层连接池和查询策略，连接治理照样重要。
- 误区：缓存命中率高就没有一致性问题。
- 正解：写后失效、过期时间和回填竞争都会影响数据正确性。
- 误区：本地内存缓存可以解决所有性能问题。
- 正解：多实例场景下，本地缓存天生存在实例间不一致问题。

## 关联知识点

- 依赖：服务端请求链、状态码策略和部署拓扑。
- 影响：接口 RT、数据库压力和热点稳定性。
- 对比：本地缓存、Redis 缓存、HTTP 缓存。
- 组合：连接池、缓存旁路、消息队列和失效治理。

## 相关面试题

1. 如何在 Node.js 中连接和操作数据库？
- 标准答案：Node.js 通常通过驱动、连接池或 ORM 连接数据库，比如用 `mysql2`、`pg`、Mongoose 等。核心不只是“连上库”，而是复用连接、控制查询方式和处理错误。场景上连接池是高并发接口的基础，不然建连成本会把吞吐先吃掉。
- 考察点：是否理解数据库接入不仅是语法问题，还包括连接治理。

2. Node.js 中如何实现缓存？
- 标准答案：常见缓存包括单进程内存缓存、Redis 这类外部缓存和 HTTP 缓存。机制上最常见的是缓存旁路模式，先查缓存，未命中再查数据库并回填；写请求则同步更新或删除缓存。场景上热点详情、配置和排行榜最常见，但必须一起设计失效策略，否则会出现脏数据。
- 考察点：是否知道缓存的层级、模式和一致性代价。
