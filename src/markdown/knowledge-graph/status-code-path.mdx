## 概念定义

- 本质定义：状态码决策链是客户端和服务端基于响应状态码进行后续动作分支的规则集合。
- 解决问题：统一处理成功、重定向、客户端错误、服务端错误和缓存协商结果。
- 边界说明：它不决定底层连接是否建立成功，也不负责渲染阶段性能。

## 核心机制拆解

### 1) 机制/流程

1. 客户端收到响应后先读取状态码类别。
2. 2xx 分支进入正常消费流程。
3. 3xx 分支按 `Location` 和方法语义执行跳转或缓存复用。
4. 4xx / 5xx 分支进入错误处理、重试或降级逻辑。
5. 文档请求成功后进入 `html-parser`。

### 2) 关键条件与触发点

- 条件 A：是否是条件请求，决定 `304` 是否可命中。
- 条件 B：重定向状态码是否保留原方法语义，决定是否产生副作用风险。

### 3) 结果与影响

- 对性能 / 稳定性 / 可维护性的影响：状态码处理一致性能减少错误重试和重复跳转；错误分类清晰有助于快速定位客户端与服务端责任。

## 相似概念对比

| 维度 | 302 | 307 | 结论 |
|---|---|---|---|
| 定义 | 临时重定向，历史上常被客户端改写方法 | 临时重定向，强调保留原方法 | 面试常考方法是否会被改写 |
| 触发时机 | 业务临时跳转 | 需要临时跳转且保持方法语义 | 写操作接口更适合用 307/308 |
| 性能影响 | 额外往返一次 | 同样有额外往返 | 核心不是快慢，而是语义安全 |
| 适用场景 | 页面跳转、活动页 | 表单提交、接口写操作跳转 | 选错可能造成重复副作用 |

## 代码

```bash
curl -I -H 'If-None-Match: "v1"' https://www.example.com/app.js
```

```nginx
location /old-path {
  return 307 https://www.example.com/new-path;
}
```

### 示例说明

- 这段代码 / 配置解决什么问题：第一段验证协商缓存是否返回 `304`，第二段展示保留方法语义的临时重定向。
- 关键参数含义：`If-None-Match` 触发基于实体标签的条件请求。
- 常见错误配置：写操作接口返回 `302`，客户端改写成 GET 导致语义偏差。

## 真实场景示例

- 场景描述：用户在活动页提交表单后出现重复提交。
- 问题表现：抓包发现服务端返回 `302`，客户端跳转后方法被改写。
- 处理方案：将关键跳转改为 `307/308`，并在后端增加幂等保护。
- 验证方式：回归提交链路并检查是否仍有重复写入。

## 常见误区

- 误区：`304` 越多越好。
- 正解：`304` 表示协商缓存有效，但如果资源长期不更新且策略僵化，也会带来内容时效问题。
- 误区：3xx 都是一样的跳转。
- 正解：不同状态码对方法保留和缓存语义不同，影响实际业务行为。

## 关联知识点

- 依赖：依赖 `http-message-headers` 提供条件请求和重定向字段。
- 影响：文档类成功响应会流向 `html-parser`。
- 对比：`status-code-path` 决定“收到后怎么做”，`http-method-semantics` 决定“发送前怎么定义语义”。
- 组合：常与 `cache-policy`、`request-lifecycle` 结合定位加载异常。

## 相关面试题

1. HTTP 状态码 304 是多好还是少好
- 标准答案：`304` 本质是协商缓存命中信号，不是错误；合理的 `304` 可以减少传输成本，但是否“好”要结合资源更新频率和缓存策略看。
- 考察点：是否能避免把状态码做简单二元判断。
- 追问 1：为什么返回 `304` 仍有请求成本？
- 标准答案 1：协商缓存仍需一次请求往返，只是省掉响应体传输。
- 追问 2：什么情况下会故意降低 `304` 比例？
- 标准答案 2：高频更新内容需要更快拿到新版本时，会调整缓存策略减少旧副本复用。

2. 2XX (Success 成功状态码)
- 标准答案：2xx 代表请求被成功处理，常见有 `200`、`204`、`206`；其中 `206` 常用于范围请求和断点续传。
- 考察点：是否能区分不同成功码在业务链路中的意义。

3. HTTP 常见的状态码有哪些？
- 标准答案：常见分类包括 `2xx`（如 `200`、`204`、`206`）、`3xx`（如 `301`、`302`、`304`、`307`）、`4xx`（如 `400`、`401`、`403`、`404`、`429`）和 `5xx`（如 `500`、`502`、`503`）。回答时要结合场景说明：例如 `304` 属于缓存协商命中，`429` 常用于限流，`503` 多见于过载或维护窗口。
- 考察点：是否能给出分组清单并解释关键状态码的业务语义。
