## 概念定义

- 本质定义：该节点聚焦 组件间数据和事件协议稳定传递。

- 边界说明：不替代其他节点的专属机制域。

4. 用回归验证结果是否稳定。

### 2) 关键条件与触发点

## 相似概念对比

| 维度 | props/emits | provide/inject | 结论 |
|---|---|---|---|
| 定义 | 关注单点能力 | 关注对照能力 | 职责边界不同 |
| 触发时机 | 偏局部触发 | 偏链路触发 | 需要按场景选择 |
| 性能影响 | 局部可见 | 全链路可见 | 不可只看单点 |
| 适用场景 | 快速落地 | 长期治理 | 建议组合使用 |

## 代码

```vue
<!-- Parent.vue -->
<script setup lang="ts">
import Child from './Child.vue'
import { ref } from 'vue'

const name = ref('Tom')
function updateName(next: string) {
  name.value = next
}
</script>

<template>
  <Child :name="name" @update:name="updateName" />
</template>
```

```vue
<!-- Child.vue -->
<script setup lang="ts">
const props = defineProps<{ name: string }>()
const emit = defineEmits<{ 'update:name': [value: string] }>()
</script>

<template>
  <button @click="emit('update:name', props.name + '!')">{{ props.name }}</button>
</template>
```

### 示例说明

- 这段代码解决什么问题：展示父传子与子传父的主通道（`props + emits`）。
- 关键语句或参数含义：`defineEmits` 明确事件契约，避免隐式通信。
- 常见错误写法：子组件直接修改 `props`，破坏单向数据流。

## 真实场景示例

- 场景描述：核心页面迭代中出现跨组件状态与渲染时序问题。
- 问题表现：功能可用但偶发抖动、错渲染或发布回退。
- 处理方案：先收敛边界，再做最小修复，最后补自动化验证。
- 验证方式：通过关键路径回归、性能指标和线上监控确认生效。

## 相关面试题

1. 子组件可以直接改变父组件的数据吗？
- 标准答案：结论是不建议直接改父数据。机制上这会破坏单向数据流，导致状态来源不可追踪。场景上应通过 `emit` 通知父组件更新，或把共享状态放到状态管理层。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

2. props / $emit
- 标准答案：结论是父传子用 `props`，子传父用 `$emit`。机制上 `props` 只读，子组件通过事件把意图上抛，由父组件改源状态。场景上这是最稳定的父子通信协议。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

3. EventBus 事件总线（$emit / $on）
- 标准答案：结论是 EventBus 适合临时跨组件事件广播，但不适合长期核心状态。机制上本质是发布订阅，需手动 `on/off` 防止监听泄漏。场景上一次性提示、埋点广播可用，业务状态建议用 Pinia/Redux。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

4. 依赖注入（provide / inject）
- 标准答案：结论是 `provide/inject` 适合跨层级传递“稳定上下文”。机制上祖先通过 `provide` 暴露值，后代 `inject` 读取，避免层层透传 `props`。场景上主题、国际化、表单上下文最常用。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

5. ref / $refs
- 标准答案：结论是 `ref/$refs` 用于访问组件实例或 DOM。机制上它在渲染完成后才可用，更新后通常要配合 `nextTick`。场景上聚焦输入框、测量元素尺寸、调用子组件暴露方法时使用。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

6. $parent / $children
- 标准答案：结论是 `$parent/$children` 耦合高，不推荐作为主通信方式。机制上它依赖组件层级结构，重构后容易失效。场景上历史项目可临时兼容，新代码优先 `props/emit`、`provide/inject` 或状态管理。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

7. $attrs / $listeners
- 标准答案：结论是 `$attrs/$listeners`（Vue 3 中事件并入 attrs）用于透传未声明属性和事件。机制上可构建高阶包装组件而不破坏外部 API。场景上二次封装 UI 组件、表单控件透传最常见。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

8. Vue-router 跳转和 location.href 有什么区别
- 标准答案：结论是 Vue Router 跳转是前端路由更新，`location.href` 是整页刷新。机制上前者复用 SPA 运行时并保留应用状态，后者会重新请求文档。场景上站内导航优先 Router，跨站或强制刷新才用 `location.href`。
- 考察点：是否能从定义、触发时机、性能影响和适用场景四个维度做对比。
