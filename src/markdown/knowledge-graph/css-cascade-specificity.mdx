## 概念定义

- 本质定义：层叠与优先级是浏览器决定“同一元素最终应用哪条样式”的规则系统。
- 解决问题：避免样式覆盖冲突和不可预测结果。
- 边界说明：它只决定命中和覆盖顺序，不直接决定布局算法。

## 核心机制拆解

### 1) 机制/流程

1. 浏览器先匹配选择器，筛出候选规则。
2. 按来源、重要性（`!important`）、优先级权重、声明顺序比较。
3. 继承属性在无直接声明时参与最终计算。

### 2) 关键条件与触发点

- 条件 A：高权重选择器叠加过多时，样式难以维护。
- 条件 B：滥用 `!important` 会破坏可预测性。

### 3) 结果与影响

- 对性能/稳定性/可维护性的影响：规则简洁时覆盖关系稳定；规则混乱时回归成本急剧上升。

## 相似概念对比

| 维度 | 伪类 | 伪元素 | 结论 |
|---|---|---|---|
| 作用对象 | 真实元素状态 | 虚拟片段 | 职责不同 |
| 语法 | `:hover` | `::before` | CSS3 推荐双冒号伪元素 |
| 是否新增节点 | 否 | 否（渲染层虚拟） | 都不改 DOM（Document Object Model） |
| 典型场景 | 交互态样式 | 装饰性内容 | 可组合使用 |

## 代码

```css
#app .card:hover { color: #0ea5e9; }
.card::before { content: 'NEW'; }
```

### 示例说明

- 这段代码解决什么问题：同时演示状态样式与伪元素内容插入。
- 关键语句或参数含义：`#app .card:hover` 权重大于普通类选择器；`::before` 创建前置虚拟内容。
- 常见错误写法：把布局关键内容写在伪元素里，导致语义和可访问性下降。

## 真实场景示例

- 场景描述：组件样式在不同页面被反复覆盖，出现“改一处崩多处”。
- 问题表现：同一组件在 A 页面正常，在 B 页面颜色错乱。
- 处理方案：降低选择器嵌套深度，建立层级命名规范，清理 `!important`。
- 验证方式：组件快照对比、跨页面回归、样式冲突数统计。

## 常见误区

- 误区：优先级只看选择器权重。
- 正解：还要看来源顺序、`!important` 和声明先后。
- 误区：继承属性优先级很高。
- 正解：继承值优先级最低，直接声明优先。

## 关联知识点

- 依赖：HTML 结构语义。
- 影响：工程化命名规范和组件复用稳定性。
- 对比：层叠规则与 BFC（Block Formatting Context）布局隔离。
- 组合：预处理器和设计令牌体系。

## 相关面试题

1. CSS选择器及其优先级
- 标准答案：结论是优先级比较遵循“`!important` > 内联 > id > 类/属性/伪类 > 标签/伪元素 > 通配符”。机制上同级再比较后写规则。场景上应减少高权重选择器，避免覆盖失控。
- 考察点：是否能准确说明权重顺序与覆盖条件。

2. CSS中可继承与不可继承属性有哪些
- 标准答案：可继承常见有 `font-*`、`color`、`line-height`、`text-align`；不可继承常见有 `display`、`position`、`margin`、`padding`、`border`、`background`。机制上继承仅在子元素未声明时生效。场景上可用继承减少重复声明。
- 考察点：是否能按“文本类可继承、盒模型类不可继承”快速分类。

3. 伪元素和伪类的区别和作用？
- 标准答案：结论是伪类描述元素状态，伪元素描述元素内部虚拟片段。机制上二者都不新增真实 DOM 节点。场景上伪类常用于交互态，伪元素常用于装饰内容。
- 考察点：是否能区分状态与结构片段。

4. ::before 和 :after 的双冒号和单冒号有什么区别？
- 标准答案：结论是双冒号是 CSS3 规范写法，用于伪元素；单冒号是历史兼容写法。机制上现代浏览器通常兼容两种写法。场景上推荐统一使用双冒号以提升可读性和规范一致性。
- 考察点：是否理解规范演进与兼容策略。
