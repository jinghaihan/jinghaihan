## 概念定位

布局（Layout）节点负责计算元素几何位置和尺寸，是渲染链路中的高成本阶段。

布局频繁触发会直接导致交互卡顿和掉帧。

## 工作机制

1. 浏览器根据渲染树和约束规则计算盒模型。
2. 确定每个节点的尺寸、位置与流式关系。
3. 处理 flex/grid/position 等布局模型。
4. 布局结果输出给绘制阶段。
5. 后续样式和结构变更可能再次触发布局。

## 关键知识点

### 必须掌握

- 布局成本取决于受影响节点范围。
- 强制同步布局会把成本集中到交互路径。
- 复杂嵌套和未知尺寸内容会提高布局难度。
- 布局抖动会直接影响 CLS。
- 读取布局信息要避免与写样式交错。
- 容器约束稳定可减少重排连锁。

### 选型与取舍

- 更灵活动态布局可提升体验，但计算成本更高。
- 固定约束布局性能更稳，但适配弹性下降。
- 局部隔离可降成本，但实现复杂度提升。

## 对页面性能与稳定性的影响

- 节点稳定时，Knowledge Graph 主线 的网络与渲染链路更可预测，性能治理收益更稳定。
- 节点失稳时，问题会向后续阶段扩散，常表现为首屏变慢、交互卡顿或版本一致性风险。
- 优化动作应结合上下游联动验证，避免局部优化把成本转移到其他节点。

## 观测与排障

### 核心指标

- Layout 总耗时
- 单帧布局耗时峰值
- 布局触发次数
- CLS 分位数

### 排障顺序

1. 先找布局热点函数和触发源。
2. 检查是否存在读写交错导致强制布局。
3. 检查受影响节点范围是否过大。
4. 优化布局模型和容器约束。
5. 回归交互关键路径。

### 常见故障场景

- 输入时页面明显卡顿。
- 滚动过程中布局频繁触发。
- 异步内容插入导致页面跳动。
- 复杂表单场景延迟高。

## 常见误区

- 把布局问题误判成网络问题。
- 忽略布局触发范围。
- 高频交互中大量读取布局信息。

## 题库深挖补充

### 高频对比与边界（基于题库）

- Layout 负责几何计算，Paint 负责像素绘制，Composite 负责图层合成。
- 回流（Reflow）通常比重绘（Repaint）成本高，合成通常成本更低但不是免费。
- top/left 动画常触发布局与绘制，transform/opacity 更可能走合成路径。
- 强制同步布局（Forced Synchronous Layout）来自“写样式后立刻读布局”。
- will-change 能提速也能增内存，滥用会导致图层膨胀。
- 面试常问“为什么动画卡”，核心看属性选择与任务调度。

### 线上排障剧本

1. 在性能面板拆分 Scripting/Layout/Paint/Composite 占比。
2. 定位触发高频布局或重绘的 DOM 操作。
3. 调整动画属性到合成友好路径并验证视觉一致性。
4. 检查图层数量、GPU 内存和掉帧峰值。
5. 在真实设备上回归而非只看本机。

### 高频追问补充（附答案）

1. transform 为什么常更流畅？

因为更容易在合成阶段完成，减少布局和绘制开销。

2. 什么场景会触发强制同步布局？

连续读写布局属性交错，浏览器被迫立即计算几何信息。

3. 合成层越多越好吗？

不是，过多图层会增加内存和合成管理成本。

## 进阶追问（补充）

1. 如果网络指标正常但用户仍感知慢，优先看哪一层？

优先看主线程任务和渲染阶段（Layout/Paint/Composite）是否拥塞，网络快不代表交互快。

2. 缓存策略和发布策略为什么必须一起设计？

缓存决定资源复用路径，发布决定资源版本切换；两者脱节会出现版本错配。

3. 如何验证一次链路优化不是偶然收益？

至少做对照组、分位数对比，并覆盖冷缓存与弱网样本。

## 案例追问加深（第三轮）

### 真实故障片段

- 故障表现：同一版本在不同地区首屏耗时差异明显，且刷新方式不同结果不同。
- 排查结论：网络链路、缓存路径和渲染调度叠加导致体验分化，单点优化无法彻底解决。
- 治理动作：建立链路分层观察面板，按导航类型、缓存命中层和渲染阶段联合分析。

### 追问补充

1. 为什么链路问题经常不是单点根因？

Knowledge Graph 本身是多阶段因果链，一个阶段的波动会在后续阶段被放大。

2. 如何快速缩小排障范围？

先固定触发方式和样本维度，再按网络、缓存、渲染三层逐步排除。

## 面试补充

### 30 秒说明

布局节点是渲染性能的重灾区，核心治理思路是减少触发频率、收敛影响范围、避免强制同步布局。

### 高频追问

1. 为什么布局比绘制更容易卡？

布局常涉及全局几何计算，影响范围大。

2. 如何快速判断是否布局瓶颈？

看性能面板中 Layout 时间占比和触发频率。

3. 强制同步布局怎么产生？

写样式后立刻读布局信息会迫使浏览器立即计算。

4. CLS 和布局有什么关系？

布局不稳定会导致可见内容位移，直接抬高 CLS。
