## 概念定义

- 本质定义：请求模式关注前端如何发请求、处理响应、控制超时和错误。
- 解决问题：解决重复请求、竞态覆盖、错误处理分散和 API 选择混乱问题。
- 边界说明：不处理底层传输协议实现。

## 核心机制拆解

### 1) 机制/流程

1. 统一请求入口，封装超时、取消和错误转换。
2. 区分读取与写入请求的重试策略。
3. 通过拦截器或中间层管理鉴权和日志。
4. 统一响应结构后分发给业务层。

### 2) 关键条件与触发点

- 条件 A：页面快速切换时需取消过期请求，防止旧响应覆盖新状态。
- 条件 B：弱网场景下需设置超时和可恢复提示。

### 3) 结果与影响

- 对稳定性的影响：统一请求层能显著降低线上偶发错误。
- 对可维护性的影响：API 选择标准明确后，团队协作成本更低。

## 相似概念对比

| 维度 | ajax | axios | fetch |
|---|---|---|---|
| 本质 | 通常指 XHR 异步请求模式 | 基于 XHR 的封装库 | 浏览器原生 Promise API |
| 拦截能力 | 需手写 | 内置拦截器 | 需手写包装 |
| 错误处理 | 事件回调风格 | 统一错误对象 | `4xx/5xx` 默认不抛错 |
| 适用场景 | 兼容旧项目 | 中大型项目统一治理 | 轻量或自定义封装 |

## 代码

```ts
async function requestJSON(url: string, timeout = 8000) {
  const controller = new AbortController()
  const timer = setTimeout(() => controller.abort(), timeout)
  try {
    const res = await fetch(url, { signal: controller.signal })
    if (!res.ok) throw new Error(`HTTP ${res.status}`)
    return await res.json()
  }
  finally {
    clearTimeout(timer)
  }
}
```

### 示例说明

- 示例体现了 `fetch` 封装中的超时和状态码判断。
- `fetch` 仅在网络失败时 reject，业务需要手动检查 `res.ok`。
- 常见错误写法是直接 `res.json()` 而不处理 `4xx/5xx`。

## 相关面试题

1. 对 AJAX 的理解，实现一个 AJAX 请求
   - 标准答案：AJAX 是异步更新页面数据的请求模式，早期常基于 XHR。实现时至少要处理 `open/send`、状态码判断、超时与错误回调。
   - 考察点：是否理解请求最小闭环。
```ts
function ajaxGet(url: string, timeout = 8000) {
  return new Promise<string>((resolve, reject) => {
    const xhr = new XMLHttpRequest()
    xhr.open('GET', url, true)
    xhr.timeout = timeout

    xhr.onreadystatechange = () => {
      if (xhr.readyState !== XMLHttpRequest.DONE) return
      if (xhr.status >= 200 && xhr.status < 300) resolve(xhr.responseText)
      else reject(new Error(`HTTP ${xhr.status}`))
    }
    xhr.onerror = () => reject(new Error('network error'))
    xhr.ontimeout = () => reject(new Error('request timeout'))

    xhr.send()
  })
}
```

2. ajax、axios、fetch 的区别
   - 标准答案：ajax 指模式，axios 是封装库，fetch 是原生 API。axios 拦截和统一处理能力更强，fetch 更轻量但需自行补齐错误和超时治理。
   - 考察点：是否能给出工程选型理由。

3. JavaScript 脚本延迟加载的方式有哪些？
   - 标准答案：常见有 `defer`、`async`、动态创建 `script`、按需加载模块。选择关键是是否依赖解析顺序和是否阻塞文档解析。
   - 考察点：是否理解加载时序与执行时机。

4. 实现 AJAX 请求
   - 标准答案：手写可基于 XHR 封装 Promise，处理超时、中止、状态码和异常分支。生产代码通常进一步封装重试、鉴权和统一错误映射。
   - 考察点：是否能从“能发请求”升级到“可治理请求层”。
```ts
interface AjaxOptions {
  url: string
  method?: 'GET' | 'POST'
  body?: Document | XMLHttpRequestBodyInit | null
  timeout?: number
  headers?: Record<string, string>
}

function ajaxRequest<T = unknown>({ url, method = 'GET', body = null, timeout = 8000, headers = {} }: AjaxOptions) {
  return new Promise<T>((resolve, reject) => {
    const xhr = new XMLHttpRequest()
    xhr.open(method, url, true)
    xhr.timeout = timeout
    xhr.responseType = 'json'

    Object.entries(headers).forEach(([k, v]) => xhr.setRequestHeader(k, v))

    xhr.onload = () => {
      if (xhr.status >= 200 && xhr.status < 300) resolve(xhr.response as T)
      else reject(new Error(`HTTP ${xhr.status}`))
    }
    xhr.onerror = () => reject(new Error('network error'))
    xhr.ontimeout = () => reject(new Error('request timeout'))

    xhr.send(body)
  })
}
```
