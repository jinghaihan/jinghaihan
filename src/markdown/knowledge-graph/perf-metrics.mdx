## 知识体系概览

性能优化体系解决“页面为什么慢、如何稳定变快、如何防止回退”这三类问题，位于网络链路、渲染链路和工程发布链路的交汇处。

## 能力边界

- 覆盖范围：边缘分发、缓存策略、资源提示、懒加载、图片策略、渲染优化、事件频控、构建优化、发布门禁。
- 非覆盖范围：框架内部实现细节、业务接口设计、后端数据库调优。

## 核心子主题

1. CDN 与边缘分发：通过就近访问和边缘缓存降低首访时延。
2. 缓存策略优化：统一浏览器与边缘缓存规则，平衡速度和一致性。
3. 资源预提示优化：提前建立连接与请求关键资源，缩短等待时间。
4. 懒加载与按需加载：只在需要时加载资源，减少无效下载。
5. 图片优化策略：按场景选择格式、尺寸和加载策略。
6. 回流、重绘与渲染优化：减少不必要的布局与绘制开销。
7. 节流与防抖策略：控制高频事件触发，稳定交互性能。
8. Webpack 构建与体积优化：降低打包耗时和产物体积。
9. 发布回归与性能门禁：把优化结果固化到发布流程，防止回退。

## 推荐学习路径

- 入门顺序：CDN 与边缘分发 -> 缓存策略优化 -> 图片优化策略 -> 回流、重绘与渲染优化。
- 面试优先顺序：回流、重绘与渲染优化 -> Webpack 构建与体积优化 -> 节流与防抖策略 -> CDN 与边缘分发。
- 项目落地顺序：缓存策略优化 -> 图片优化策略 -> Webpack 构建与体积优化 -> 发布回归与性能门禁。

## 常见混淆边界

- 预加载 vs 懒加载：预加载是提前获取，懒加载是延后获取。
- 回流 vs 重绘：回流会改变布局并触发重绘，重绘只改外观不改布局。

## 相关知识岛

- 计算机网络体系：DNS、连接建立和缓存协商直接影响加载时延。
- 浏览器原理体系：布局、绘制、合成决定渲染链路上限。
- 前端工程化体系：构建产物、分包和发布流程影响性能稳定性。
