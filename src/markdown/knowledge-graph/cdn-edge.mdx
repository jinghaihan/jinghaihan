## 概念定位

CDN 边缘节点负责就近分发与缓存命中，是降低网络时延和源站压力的关键环节。

CDN 策略决定首访速度和发布一致性。

## 工作机制

1. 通过 DNS/GSLB（Global Server Load Balancing）将请求引导到合适边缘节点。
2. 边缘节点按缓存头判断命中与回源。
3. 未命中时向源站拉取并回填边缘缓存。
4. 根据区域和运营商差异做节点调度。
5. 发布时通过预热与失效保障版本一致。

## 关键知识点

### 必须掌握

- 命中率要结合版本一致性一起评估。
- 回源路径异常会放大首字节延迟。
- 缓存键设计（Vary）直接影响命中效果。
- 多地区发布要验证边缘一致性。
- CDN 错误页和回退策略要可控。
- 边缘缓存策略必须和构建指纹策略协同。

### 选型与取舍

- 长缓存提升命中，但失效控制难度更高。
- 频繁失效一致性好，但回源成本增加。
- 多层边缘策略可提速，但排障复杂度提升。

## 对页面性能与稳定性的影响

- 节点稳定时，Knowledge Graph 主线 的网络与渲染链路更可预测，性能治理收益更稳定。
- 节点失稳时，问题会向后续阶段扩散，常表现为首屏变慢、交互卡顿或版本一致性风险。
- 优化动作应结合上下游联动验证，避免局部优化把成本转移到其他节点。

## 观测与排障

### 核心指标

- CDN 命中率
- 回源率
- 边缘节点错误率
- 多区域 TTFB 分位数

### 排障顺序

1. 先确认异常是否集中在特定地区或运营商。
2. 检查边缘缓存命中和回源链路。
3. 检查缓存键与请求头差异。
4. 检查发布后的失效与预热状态。
5. 多节点抽样验证恢复情况。

### 常见故障场景

- 部分地区访问慢。
- 发布后有人新有人旧。
- 边缘命中低导致源站压力突增。
- 节点异常导致资源 5xx。

## 常见误区

- 只看全局命中率不看区域分布。
- 发布时忽略边缘失效策略。
- 缓存键设计过于粗糙。

## 题库深挖补充

### 高频对比与边界（基于题库）

- CDN（Content Delivery Network）核心目标是“就近访问 + 回源收敛”。
- GSLB（Global Server Load Balancing）负责把请求导向更优边缘节点。
- 边缘命中率高不等于用户体验好，还要看版本一致性与回源稳定性。
- stale-while-revalidate / stale-if-error 策略可提升可用性，但要防陈旧数据长期驻留。
- preconnect/dns-prefetch 是客户端侧资源提示，与 CDN 调度协同但不是同一层机制。
- 面试常问“CDN 为什么有时比源站还慢”，常见是节点异常或回源链路问题。

### 线上排障剧本

1. 按地区和运营商分桶看 TTFB 差异。
2. 检查边缘命中、回源、回源失败比例。
3. 检查缓存键（Vary）是否导致碎片化命中。
4. 检查发布后预热与失效是否完成。
5. 边缘异常时准备回源兜底与限流策略。

### 高频追问补充（附答案）

1. 为什么同一资源在不同地区体验差很多？

边缘节点健康、运营商路由和回源路径都可能不同。

2. 命中率高但仍有人报旧版本，怎么解释？

命中可能是旧入口或错误缓存键，命中率和一致性要一起看。

3. preconnect 和 CDN 是替代关系吗？

不是。preconnect 是客户端提前建连，CDN 是分发架构层。

## 进阶追问（补充）

1. 如果网络指标正常但用户仍感知慢，优先看哪一层？

优先看主线程任务和渲染阶段（Layout/Paint/Composite）是否拥塞，网络快不代表交互快。

2. 缓存策略和发布策略为什么必须一起设计？

缓存决定资源复用路径，发布决定资源版本切换；两者脱节会出现版本错配。

3. 如何验证一次链路优化不是偶然收益？

至少做对照组、分位数对比，并覆盖冷缓存与弱网样本。

## 案例追问加深（第三轮）

### 真实故障片段

- 故障表现：同一版本在不同地区首屏耗时差异明显，且刷新方式不同结果不同。
- 排查结论：网络链路、缓存路径和渲染调度叠加导致体验分化，单点优化无法彻底解决。
- 治理动作：建立链路分层观察面板，按导航类型、缓存命中层和渲染阶段联合分析。

### 追问补充

1. 为什么链路问题经常不是单点根因？

Knowledge Graph 本身是多阶段因果链，一个阶段的波动会在后续阶段被放大。

2. 如何快速缩小排障范围？

先固定触发方式和样本维度，再按网络、缓存、渲染三层逐步排除。

## 面试补充

### 30 秒说明

CDN 节点治理核心是“命中、回源、一致性”三者平衡。只追命中率会隐藏版本风险。

### 高频追问

1. GSLB 在 CDN 中的作用是什么？

按地理和网络条件把用户导向更优边缘节点。

2. 命中率高为什么还会出问题？

可能命中的是旧版本或错误缓存键。

3. 回源率突升通常意味着什么？

边缘缓存失效、缓存键变化或节点异常。

4. 发布后如何保证边缘一致？

预热关键资源并校验失效完成情况。
