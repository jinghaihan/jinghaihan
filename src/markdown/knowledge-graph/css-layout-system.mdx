## 概念定义

- 本质定义：经典布局与 Flex 关注页面结构如何在已知约束下稳定排布。
- 解决问题：快速实现两栏、三栏、居中和网格类布局。
- 边界说明：浮动清理和 BFC 细节在“浮动、clear 与 BFC”节点展开。

## 核心机制拆解

### 1) 机制/流程

1. 明确约束：固定宽、弹性宽、等高、居中、换行。
2. 选择模型：普通流、浮动、定位、Flex、Grid。
3. 优先用现代布局（Flex / Grid），兼容场景再补传统方案。

### 2) 关键条件与触发点

- 条件 A：布局方案和业务约束不匹配时，后续补丁会快速增多。
- 条件 B：只追求“实现出来”而忽视可维护性，会导致重构成本上升。

### 3) 结果与影响

- 对性能/稳定性/可维护性的影响：布局方案正确时结构稳定且易扩展；方案错误时常见溢出、错位和兼容修补。

## 相似概念对比

| 维度 | Flex | 绝对定位布局 | 结论 |
|---|---|---|---|
| 适应性 | 强 | 弱 | 现代响应式优先 Flex |
| 代码复杂度 | 中低 | 中高 | 动态内容场景 Flex 更稳 |
| 对文档流影响 | 保留流式关系 | 易脱离文档流 | 维护难度不同 |
| 典型场景 | 一维分布与对齐 | 精确覆盖定位 | 按约束选型 |

## 代码

```css
.container {
  display: flex;
}
.left {
  width: 240px;
}
.right {
  flex: 1;
}
```

### 示例说明

- 这段代码解决什么问题：实现左侧固定、右侧自适应两栏。
- 关键语句或参数含义：`flex: 1` 表示剩余空间分配因子为 1。
- 常见错误写法：父容器忘记 `display: flex`，导致 `flex: 1` 不生效。

## 真实场景示例

- 场景描述：管理后台需要同一套组件适配桌面大屏和笔记本窄屏。
- 问题表现：旧浮动布局在窄屏出现换行错位。
- 处理方案：核心容器迁移到 Flex，局部复杂网格使用 Grid。
- 验证方式：多分辨率回归、缩放测试、断点下交互可用性检查。

## 常见误区

- 误区：`flex: 1` 等于 `width: 100%`。
  - 正解：`flex: 1` 是弹性分配，不是固定宽度。
- 误区：三栏布局只能靠浮动实现。
  - 正解：Flex、Grid、定位都可实现，优先可维护方案。

## 关联知识点

- 依赖：响应式单位与媒体查询。
- 影响：定位与层叠上下文、渲染抖动治理。
- 对比：浮动方案与现代布局方案。
- 组合：发丝线和移动端适配策略。

## 相关面试题

1. 两栏布局的实现
   - 标准答案：可用浮动 + `margin`、BFC、Flex、定位等方案。机制上本质是“固定列 + 弹性列”的空间分配。场景上现代项目优先 Flex 实现。
   - 考察点：是否能给出多方案并说明取舍。
```css
.layout {
  display: flex;
}
.left {
  width: 240px;
}
.right {
  flex: 1;
}
```

2. 三栏布局的实现
   - 标准答案：可用绝对定位、浮动、Flex、Grid 实现。机制上核心是左右固定宽度、中间自适应。场景上内容动态变化时优先 Flex / Grid。
   - 考察点：是否掌握方案多样性和适用边界。
```css
.layout {
  display: grid;
  grid-template-columns: 200px 1fr 220px;
  gap: 16px;
}
```

3. 水平垂直居中的实现
   - 标准答案：常见方案有 Flex 居中、绝对定位 + `transform`、Grid 居中。机制上居中本质是主轴和交叉轴对齐控制。场景上未知宽高元素优先 Flex。
   - 考察点：是否能覆盖“已知尺寸/未知尺寸”两类情况。
```css
.parent {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

4. 对Flex布局的理解及其使用场景
   - 标准答案：结论是 Flex 适合一维布局的分布和对齐。机制上通过主轴、交叉轴和弹性因子控制空间分配。场景上导航栏、表单行、卡片列表是高频场景。
   - 考察点：是否理解 Flex 核心心智模型。

5. flex:1 表示什么
   - 标准答案：结论是 `flex: 1` 等价于 `flex-grow: 1; flex-shrink: 1; flex-basis: 0%`。机制上会参与剩余空间分配。场景上常用于自适应主内容区。
   - 考察点：是否知道简写展开含义。

6. 实现“品” 字布局
   - 标准答案：可用 Flex + 换行或 Grid 模板实现。机制上核心是行列分组和跨度控制。场景上推荐 Grid，表达更直观。
   - 考察点：是否能把题目映射到对应布局模型。
```css
.pin {
  display: grid;
  grid-template-columns: repeat(2, 120px);
  grid-template-areas:
    "top top"
    "left right";
  gap: 8px;
}
.pin > :nth-child(1) { grid-area: top; justify-self: center; }
.pin > :nth-child(2) { grid-area: left; }
.pin > :nth-child(3) { grid-area: right; }
```

7. 实现九宫格布局
   - 标准答案：可用 Grid 三行三列、Flex 换行配合固定项宽，或定位方案。机制上关键是容器分轨与子项对齐。场景上现代项目优先 Grid。
   - 考察点：是否具备工程可维护视角。
```css
.grid9 {
  display: grid;
  grid-template-columns: repeat(3, minmax(0, 1fr));
  gap: 8px;
}
.grid9 > div {
  aspect-ratio: 1 / 1;
}
```
