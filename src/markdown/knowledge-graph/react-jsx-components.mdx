## 概念定位

JSX 与组件抽象节点关注组件职责、组合方式和复用边界，目标是可读、可测、可演进。

组件抽象过粗或过细都会放大维护成本。

## 工作机制

1. 按业务语义拆组件，而不是按 DOM 片段机械拆分。
2. 优先组合（composition）而不是继承。
3. 提取可复用逻辑到自定义 Hook，保持视图组件轻量。
4. 约束组件输入输出契约，减少隐式依赖。
5. 为基础组件建立稳定 API，避免频繁破坏兼容。

## 关键知识点

### 必须掌握

- JSX 本质是语法糖，核心仍是函数调用与元素对象。
- 高阶组件、Render Props、Hook 是不同复用范式。
- 组件边界应与业务语义一致。
- 避免“万能组件”承载过多分支。
- 受控组件与非受控组件要按场景选择。
- 组件文档和示例是抽象质量的重要保障。

### 选型与取舍

- 抽象越强复用越高，但理解成本上升。
- 组件越小复用越灵活，但组合复杂度更高。
- 统一基础组件可提效，但迭代节奏需兼顾业务。

## 对页面性能与稳定性的影响

- 节点稳定时，React 工作流 的链路可预测性会明显提升，发布和排障成本更可控。
- 节点失稳时，问题会向上下游扩散，常表现为性能回退、交互异常或交付风险叠加。
- 该节点优化应和上下游协同验证，避免局部优化把成本转移给后续节点。

## 观测与排障

### 核心指标

- 组件复用率
- 组件平均复杂度
- 组件 API 破坏性变更次数
- 组件相关回归缺陷数

### 排障顺序

1. 先看问题是职责混杂还是复用边界错误。
2. 检查 props 是否过多且语义不清。
3. 识别是否有重复逻辑可抽成 Hook。
4. 评估组件拆分后的测试可维护性。
5. 沉淀组件设计约定。

### 常见故障场景

- 组件 props 数量失控。
- 同类逻辑在多个页面重复实现。
- 基础组件升级引发大量兼容问题。
- 组件职责混杂导致测试困难。

## 常见误区

- 为了抽象而抽象。
- 忽略组件 API 稳定性。
- 把业务逻辑塞进基础组件。

## 题库深挖补充

### 高频对比与边界（基于题库）

- React 核心是状态驱动渲染，不是命令式改 DOM。
- JSX 是语法糖，组件抽象质量取决于职责边界而非语法本身。
- state 与 props 分工不清会导致重复状态和渲染扩散。
- 测试与发布要围绕关键路径，而不是只追覆盖率数字。
- 团队治理要把 Hook 规则、副作用边界写进可执行规范。
- 面试常问“为什么 React 不断引入新范式”，核心是抽象与调度能力演进。

### 线上排障剧本

1. 先确认问题是状态边界错误还是组件抽象错误。
2. 检查是否存在重复状态来源。
3. 检查组件输入输出契约是否稳定。
4. 回归关键页面交互和异常路径。
5. 把结论沉淀到评审规范与测试用例。

### 高频追问补充（附答案）

1. 为什么单向数据流能提升可维护性？

更新路径可追踪，状态源更明确，排障成本更低。

2. 组件抽象越细越好吗？

不一定，过细会增加组合复杂度与通信成本。

3. React 项目治理最容易忽略什么？

副作用边界和状态生命周期规则。

## 进阶追问（补充）

1. React 优化中最容易造成反效果的动作是什么？

盲目加 memo/useCallback，增加复杂度却没有命中真正热点。

2. 为什么 effect 问题总在复杂页面里更明显？

组件层级深、状态来源多、异步并发高，会放大依赖与清理缺陷。

3. 如何判断状态管理选型是否需要升级？

当更新粒度无法收敛、排障不可追踪、跨页面一致性反复出问题时。

## 案例追问加深（第三轮）

### 真实故障片段

- 故障表现：交互偶发卡顿，且问题难以稳定复现。
- 排查结论：Hook 依赖、状态订阅粒度和组件边界共同导致重渲染扩散。
- 治理动作：先收敛状态边界，再治理 effect 依赖，最后做热点组件优化。

### 追问补充

1. 为什么 React 性能问题常和状态设计绑定？

状态边界决定更新扇出，扇出范围决定渲染成本上限。

2. 如何避免“优化 API 堆叠但收益很小”？

先定位热点和根因，再选择最小有效动作，而不是先上 memo 类技巧。

## 面试补充

### 30 秒说明

React 组件抽象的目标不是代码变少，而是职责清晰、复用可控、长期维护成本可预测。

### 高频追问

1. 组合优于继承的原因是什么？

组合更灵活，能按需拼装能力且耦合更低。

2. 什么时候用自定义 Hook？

当多个组件共享同一状态逻辑或副作用逻辑时。

3. 如何避免组件变成万能组件？

明确职责边界，拆分业务特例，保持 API 聚焦。

4. Render Props 与 Hook 的取舍？

Hook 可读性通常更好；Render Props 适合显式控制渲染输出。
