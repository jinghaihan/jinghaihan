## 知识体系概览

Node.js 体系解决“如何用 JavaScript 构建服务端、CLI 和中间层能力”的问题，是前端向全栈协作延伸时最常见的运行时入口。

## 能力边界

- 覆盖范围：Node.js 运行时定位、模块系统、事件循环、Buffer、Stream、文件 I/O、Express、中间件、安全、缓存、进程治理、测试、部署与监控。
- 非覆盖范围：数据库建模、Linux 运维细节、容器编排和后端分布式理论，这些只在 Node.js 场景下做接口级说明。

## 核心子主题

1. 运行时定位与 npm 生态：明确 Node.js 是什么、适合什么，以及 `package.json` 如何组织项目。
2. 事件循环与异步编程：理解非阻塞 I/O、回调、Promise、`async/await` 和事件驱动模型。
3. 模块系统与加载缓存：掌握 CommonJS、模块解析和缓存行为。
4. Buffer、Stream 与文件 I/O：理解二进制数据、流式处理和文件读写策略。
5. Express 与服务端通信：掌握中间件、RESTful API、CORS 和 WebSocket。
6. 错误处理与服务安全：建立错误边界、认证授权、校验与安全防线。
7. 数据库访问与缓存策略：把数据库、缓存和一致性治理接起来。
8. 进程模型、并发与性能治理：区分 Cluster、Worker Threads、垃圾回收和性能优化手段。
9. 日志、测试与交付监控：把日志、定时任务、测试、部署和监控纳入同一条交付链路。

## 推荐学习路径

- 入门顺序：运行时定位与 npm 生态 -> 事件循环与异步编程 -> 模块系统与加载缓存 -> Buffer、Stream 与文件 I/O -> Express 与服务端通信 -> 错误处理与服务安全 -> 数据库访问与缓存策略 -> 进程模型、并发与性能治理 -> 日志、测试与交付监控
- 面试优先顺序：事件循环与异步编程 -> 模块系统与加载缓存 -> Express 与服务端通信 -> 进程模型、并发与性能治理 -> 运行时定位与 npm 生态 -> 错误处理与服务安全 -> Buffer、Stream 与文件 I/O -> 数据库访问与缓存策略 -> 日志、测试与交付监控
- 项目落地顺序：先搭好运行时和依赖管理 -> 再打通模块、异步和 I/O -> 再建设 Web 服务与安全边界 -> 最后补缓存、性能、监控和交付闭环

## 常见混淆边界

- Node.js vs 浏览器 JavaScript：前者运行在服务端运行时，强调 I/O 和进程治理；后者运行在浏览器环境，强调页面交互与渲染。
- Cluster vs Worker Threads：前者是多进程扩展吞吐，后者是多线程分担 CPU 密集型任务。
- Buffer vs Stream：前者是一次性持有一段二进制数据，后者是分块传输与消费机制。
- JWT vs Session：前者偏无状态令牌，后者偏服务端会话存储。

## 相关知识岛

- JavaScript 体系：Node.js 的语言基础、闭包和事件循环都建立在 JavaScript 规则之上。
- 计算机网络体系：RESTful API、CORS、缓存头和 WebSocket 依赖网络协议语义。
- 前端工程化体系：构建脚本、包管理、发布流程和 Node.js 强绑定。
- 性能优化体系：服务端缓存、压缩和监控指标最终会反馈到前端体验。
