## 知识体系概览

React 体系解决声明式组件开发中的状态流转、副作用治理、路由组织和并发渲染问题。

## 能力边界

- 覆盖范围：组件抽象、state/props、Hooks、生命周期、路由、状态管理、并发、性能、发布。
- 非覆盖范围：浏览器协议细节和后端架构设计。

## 核心子主题

1. JSX 与组件抽象：定义组件职责边界。
2. 状态与 Hook：定义状态更新和副作用规则。
3. 路由与状态管理：定义页面和共享状态组织。
4. 并发与性能：定义调度和优化策略。

## 推荐学习路径

- 入门顺序：组件 -> state/props -> Hooks -> Effect -> 路由 -> 状态管理 -> 并发与性能。
- 面试优先顺序：setState/state/props -> Hooks/effect -> Fiber/diff/key -> Redux/Router。
- 项目落地顺序：先状态边界，再副作用治理，再性能和发布门禁。

## 常见混淆边界

- state vs props：前者组件内可变，后者父级输入只读。
- useEffect vs useLayoutEffect：执行时机不同，阻塞风险不同。

## 相关知识岛

- JavaScript 与手写体系：闭包和事件循环直接影响 React 行为。
- 浏览器原理体系：渲染时序与交互流畅度强相关。
- 工程化体系：构建发布与 React 性能治理联动。
