## 知识体系概览

小程序体系解决“如何在微信容器环境里组织页面、数据和交互”的问题，重点不在浏览器标准本身，而在平台约束下的运行机制和工程取舍。

## 能力边界

- 覆盖范围：应用和页面生命周期、双线程架构、`setData` 更新机制、路由栈、组件化、事件系统、性能优化、存储和网络请求。
- 非覆盖范围：微信开放平台运营规则、后端云开发细节和浏览器标准兼容性实现。

## 核心子主题

1. 应用与页面生命周期：掌握 App 和 Page 两层时机，避免初始化和销毁写错位置。
2. 双线程架构与 H5 差异：理解逻辑层、渲染层和 JSBridge 通信成本。
3. 数据绑定与 `setData`：掌握单向数据流、更新粒度和性能边界。
4. 路由栈与页面跳转：区分 `navigateTo`、`redirectTo`、`switchTab` 和 `reLaunch`。
5. 组件化与事件系统：掌握 `properties`、`triggerEvent`、事件冒泡和组件通信。
6. 性能优化、存储与网络请求：把渲染、缓存、请求和资源体积治理串起来。

## 推荐学习路径

- 入门顺序：应用与页面生命周期 -> 双线程架构与 H5 差异 -> 数据绑定与 `setData` -> 路由栈与页面跳转 -> 组件化与事件系统 -> 性能优化、存储与网络请求
- 面试优先顺序：双线程架构与 H5 差异 -> 数据绑定与 `setData` -> 应用与页面生命周期 -> 路由栈与页面跳转 -> 性能优化、存储与网络请求 -> 组件化与事件系统
- 项目落地顺序：先明确生命周期和路由入口 -> 再控制数据更新与组件通信 -> 最后做性能、请求和缓存治理

## 常见混淆边界

- 小程序 vs H5：前者运行在微信容器里，API 能力和分发方式不同；后者运行在浏览器里，受浏览器标准和兼容性约束。
- `data` 直改 vs `setData`：前者只改逻辑层对象，后者才会同步到视图层。
- `navigateTo` vs `redirectTo`：前者保留当前页进栈，后者直接替换当前页。
- `bindtap` vs `catchtap`：前者会冒泡，后者会阻止继续冒泡。

## 相关知识岛

- JavaScript 体系：小程序逻辑层仍建立在 JavaScript 基础之上。
- HTML 体系：虽然不是标准浏览器 DOM，但模板、事件和组件思维仍可类比页面开发。
- 性能优化体系：`setData` 粒度、包体积、缓存和请求治理会直接影响体验。
- Vue 体系：组件化、单向数据流和事件通信思路可以互相迁移。
