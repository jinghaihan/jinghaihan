## 概念定位

React 性能优化节点关注重渲染控制、任务调度、列表渲染和资源加载协同，目标是稳定改善真实用户体验。

它是 React 工作流中最贴近用户体感和线上指标的节点。

## 工作机制

1. 通过 memo/useMemo/useCallback 控制不必要更新。
2. 按组件边界拆分更新范围，降低渲染扇出。
3. 长列表采用虚拟化和分片渲染。
4. 高频交互使用 transition 或任务切片降低阻塞。
5. 建立性能预算与持续监控，防止版本回退。

## 关键知识点

### 必须掌握

- memo 只在输入稳定且比较成本可控时有收益。
- useCallback 不是默认优化，需要结合子组件依赖。
- 过早优化会增加复杂度，应先定位热点。
- 重渲染次数和单次渲染成本都要看。
- 性能优化需兼顾可读性与可维护性。
- 真实设备与弱网场景验证不可省略。

### 选型与取舍

- 强记忆化降低重渲染，但会增加心智负担。
- 组件拆分提升局部更新效率，但通信复杂度上升。
- 激进懒加载可减首包，但切换时可能抖动。

## 对页面性能与稳定性的影响

- 节点稳定时，React 工作流 的链路可预测性会明显提升，发布和排障成本更可控。
- 节点失稳时，问题会向上下游扩散，常表现为性能回退、交互异常或交付风险叠加。
- 该节点优化应和上下游协同验证，避免局部优化把成本转移给后续节点。

## 观测与排障

### 核心指标

- INP 分位数
- 关键组件渲染耗时
- 长任务数量
- 路由切换体感耗时

### 排障顺序

1. 先用 Profiler 找重渲染热点。
2. 确认热点来源是 props 变化、状态变化还是上下文变化。
3. 评估 memo 化收益与维护成本。
4. 对列表和高频交互做专项优化。
5. 上线后持续监控并防回退。

### 常见故障场景

- 输入时明显卡顿。
- 列表滚动掉帧。
- 一次状态更新触发大面积刷新。
- 优化后代码复杂度激增。

## 常见误区

- 把 memo 当万能按钮。
- 不看 profiler 直接改代码。
- 只看实验室指标不看线上分位数。

## 深入拆解

### 优化决策顺序

1. 先看 Profiler 找热点。
2. 再收敛状态边界和渲染范围。
3. 再考虑 memo/useMemo/useCallback。
4. 最后做微观优化。

### memo 失效常见原因

- props 每次都创建新引用。
- 父组件频繁传入匿名函数。
- 上下文更新粒度过粗。

### 大列表策略

- 视窗虚拟化。
- item 组件稳定 key。
- 滚动事件节流和批量更新。

## 题库深挖补充

### 高频对比与边界（基于题库）

- useEffect 关键是依赖完整和清理完整，省依赖是高风险短路。
- Context 适合低频上下文，不适合高频大状态共享。
- 路由与数据获取策略要处理预取命中率和取消竞态。
- 性能优化应先看 Profiler 热点，再决定 memo/useMemo/useCallback。
- 状态管理选型要区分本地 UI 状态与服务端状态。
- 面试常问“Fiber/并发有什么价值”，实战落点是更好的交互优先级调度。

### 线上排障剧本

1. 先确认瓶颈是副作用重跑、状态扩散还是渲染热点。
2. 检查 effect 依赖和异步取消机制。
3. 检查 selector 粒度与订阅范围。
4. 检查路由切换期间数据请求竞态。
5. 用对照组验证优化前后 INP 和重渲染分布。

### 高频追问补充（附答案）

1. 为什么 useEffect 容易出事故？

依赖漏写、清理缺失和异步竞态会叠加放大问题。

2. React.memo 为什么不是万能优化？

它也有比较与维护成本，需命中稳定输入场景才有收益。

3. 状态管理何时该拆 store？

当更新频率和业务边界差异明显时，应拆分避免更新扩散。

## 进阶深挖（第二轮）

### Profiler 结果解读

- Render 次数高但单次低：优先收敛更新范围。
- Render 次数不高但单次重：优先组件拆分与计算前移。
- Commit 阶段重：优先检查 DOM 规模与布局影响。

### 优化执行顺序

1. 先做数据与状态边界收敛。
2. 再做组件拆分和列表治理。
3. 最后才做 memo 细粒度优化。

### 追问扩展

1. 为什么有时去掉 memo 反而更快？

若比较成本高于重渲染成本，memo 会成为负担。

2. 如何证明优化不是“局部幻觉”？

看关键路径的线上分位数变化，并确认没有引入错误率或内存副作用。

## 面试补充

### 30 秒说明

React 性能优化要围绕真实瓶颈和用户体感，不是堆叠 API。能持续防回退才算完成治理。

### 高频追问

1. 什么时候 React.memo 最有价值？

子组件渲染昂贵且 props 大多数时间稳定时。

2. useCallback 为什么会被滥用？

它本身也有维护和比较成本，不是默认越多越好。

3. 优化列表首选动作是什么？

先做虚拟化和渲染范围收敛。

4. 如何验证优化有效？

对照 profiler 和线上分位数指标双重验证。
