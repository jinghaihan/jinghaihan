## 代码

```tsx
import { BrowserRouter, Link, Route, Routes } from 'react-router-dom'
import { lazy, Suspense } from 'react'

const Home = lazy(() => import('./pages/Home'))
const Admin = lazy(() => import('./pages/Admin'))

export default function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/admin">Admin</Link>
      </nav>
      <Suspense fallback={<span>loading...</span>}>
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/admin" element={<Admin />} />
        </Routes>
      </Suspense>
    </BrowserRouter>
  )
}
```

### 示例说明

- 这段代码解决什么问题：路由切换和页面懒加载协同，减少首屏包体积。
- 关键语句或参数含义：`lazy + Suspense` 让路由组件按需加载。
- 常见错误写法：全部路由组件同步打包，导致首次加载过重。

## 相关面试题

1. React-Router 的实现原理是什么？
- 标准答案：结论是核心是“URL 变化驱动组件树变化”。机制上监听 history/hash 变化，匹配路由表后渲染目标组件。场景上多级路由需要把公共布局提升到父路由以避免重复卸载。
- 考察点：是否能按阶段讲清核心流程、关键数据结构和边界条件。

2. 如何配置 React-Router 实现路由切换
- 标准答案：结论是定义路由表并在应用入口挂载 Router 容器。机制上通过 `Routes/Route` 或配置式路由把路径映射到组件。场景上懒加载路由组件可显著降低首屏包体积。
- 考察点：是否能给出可执行步骤、关键 API 和常见踩坑点。

3. React-Router 怎么设置重定向？
- 标准答案：结论是使用 `Navigate`（v6）或历史版本重定向组件。机制上在匹配到旧路径时返回重定向节点并替换历史记录。场景上登录态校验、旧地址迁移都依赖重定向策略。
- 考察点：是否能给出可执行步骤、关键 API 和常见踩坑点。

4. react-router 里的 Link 标签和 a 标签的区别
- 标准答案：结论是 `Link` 走客户端路由，`a` 默认触发整页跳转。机制上 `Link` 拦截点击并调用 history API，不刷新文档。场景上站内导航用 `Link`，外链或下载才用 `a`。
- 考察点：是否能从定义、触发时机、性能影响和适用场景四个维度做对比。

5. React-Router 如何获取 URL 的参数和历史对象？
- 标准答案：结论是参数通常通过 `useParams/useSearchParams` 获取，导航对象通过 `useNavigate`。机制上路由上下文在匹配后注入这些 Hook。场景上路径参数用于资源标识，查询参数用于筛选状态。
- 考察点：是否能给出可执行步骤、关键 API 和常见踩坑点。

6. React-Router 4 怎样在路由变化时重新渲染同一个组件？
- 标准答案：结论是可监听路由参数变化并在 effect 中响应，或给组件绑定随路径变化的 key 强制重挂载。机制上同一路由组件实例默认会复用。场景上详情页切换 id 时常用参数监听而非整页重建。
- 考察点：是否能给出明确结论，并补充机制和业务场景。

7. React-Router 的路由有几种模式？
- 标准答案：结论是常见有 BrowserRouter（history）和 HashRouter（hash）。机制上前者依赖服务端回退，后者不依赖服务端配置。场景上企业内网或静态托管可先 hash，正式站点优先 history。
- 考察点：是否能完整列举关键项，并说明每一项的适用场景。

8. React-Router 4 的 Switch 有什么用？
- 标准答案：结论是历史版本 `Switch` 用于只渲染首个命中路由（v6 对应 `Routes`）。机制上避免多个前缀路由同时命中。场景上 `/users` 与 `/users/:id` 这种层级路由必须控制匹配顺序。
- 考察点：是否能给出明确结论，并补充机制和业务场景。
