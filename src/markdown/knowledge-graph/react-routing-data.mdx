## 概念定位

路由与数据获取策略节点关注页面切换时的数据加载、预取、错误边界和流式渲染策略。

它直接影响导航体验、首屏时间和错误恢复能力。

## 工作机制

1. 按路由拆分代码和数据加载，避免全量拉取。
2. 路由切换前可预取关键数据，降低白屏时间。
3. 为每段路由设置错误边界和加载态策略。
4. 切换时取消无效请求，防止竞态覆盖。
5. 服务端渲染或流式渲染场景要设计同构数据协议。

## 关键知识点

### 必须掌握

- 导航体验由代码、数据和渲染三者共同决定。
- 预取要有命中概率评估，避免浪费流量。
- 错误边界应按业务模块切分，避免整页崩溃。
- 路由参数变化要触发正确的数据刷新策略。
- Skeleton 与 loading 设计要防布局抖动。
- 回退导航需考虑缓存与恢复策略。

### 选型与取舍

- 激进预取提升命中体验，但后台成本增加。
- 严格串行加载稳定，但切换延迟更高。
- 粗粒度错误边界简单，但故障影响面更大。

## 对页面性能与稳定性的影响

- 节点稳定时，React 工作流 的链路可预测性会明显提升，发布和排障成本更可控。
- 节点失稳时，问题会向上下游扩散，常表现为性能回退、交互异常或交付风险叠加。
- 该节点优化应和上下游协同验证，避免局部优化把成本转移给后续节点。

## 观测与排障

### 核心指标

- 路由切换耗时
- 导航白屏时长
- 路由请求取消率
- 路由错误恢复成功率

### 排障顺序

1. 先定位是代码加载慢还是数据加载慢。
2. 检查预取命中与浪费比例。
3. 检查切换期间请求是否取消。
4. 检查错误边界是否隔离有效。
5. 回归前进/后退/刷新路径。

### 常见故障场景

- 切路由后长时间空白。
- 返回页面状态丢失。
- 错误导致整页崩溃。
- 弱网下切换频繁失败。

## 常见误区

- 只优化代码分包，不优化数据加载。
- 错误边界只放最外层。
- 忽略导航回退路径。

## 深入拆解

### 路由数据加载模式

- 路由进入前加载：首屏稳定，切换可能稍慢。
- 路由进入后加载：首屏快，需精细 loading 设计。
- 混合模式：关键数据前置，非关键数据后置。

### 错误边界层次

1. 应用级兜底（防全局崩溃）。
2. 路由级边界（局部隔离）。
3. 组件级边界（细粒度恢复）。

### 导航体验优化

- 预取高概率下一跳数据。
- 页面切换期间保留骨架，防止闪屏。

## 题库深挖补充

### 高频对比与边界（基于题库）

- useEffect 关键是依赖完整和清理完整，省依赖是高风险短路。
- Context 适合低频上下文，不适合高频大状态共享。
- 路由与数据获取策略要处理预取命中率和取消竞态。
- 性能优化应先看 Profiler 热点，再决定 memo/useMemo/useCallback。
- 状态管理选型要区分本地 UI 状态与服务端状态。
- 面试常问“Fiber/并发有什么价值”，实战落点是更好的交互优先级调度。

### 线上排障剧本

1. 先确认瓶颈是副作用重跑、状态扩散还是渲染热点。
2. 检查 effect 依赖和异步取消机制。
3. 检查 selector 粒度与订阅范围。
4. 检查路由切换期间数据请求竞态。
5. 用对照组验证优化前后 INP 和重渲染分布。

### 高频追问补充（附答案）

1. 为什么 useEffect 容易出事故？

依赖漏写、清理缺失和异步竞态会叠加放大问题。

2. React.memo 为什么不是万能优化？

它也有比较与维护成本，需命中稳定输入场景才有收益。

3. 状态管理何时该拆 store？

当更新频率和业务边界差异明显时，应拆分避免更新扩散。

## 案例追问加深（第三轮）

### 真实故障片段

- 故障表现：交互偶发卡顿，且问题难以稳定复现。
- 排查结论：Hook 依赖、状态订阅粒度和组件边界共同导致重渲染扩散。
- 治理动作：先收敛状态边界，再治理 effect 依赖，最后做热点组件优化。

### 追问补充

1. 为什么 React 性能问题常和状态设计绑定？

状态边界决定更新扇出，扇出范围决定渲染成本上限。

2. 如何避免“优化 API 堆叠但收益很小”？

先定位热点和根因，再选择最小有效动作，而不是先上 memo 类技巧。

## 面试补充

### 30 秒说明

React 路由数据策略的关键是让导航链路可预测：加载快、错误可隔离、状态可恢复。

### 高频追问

1. 路由切换卡顿一般先查什么？

先区分代码分包加载慢还是数据请求慢。

2. 预取策略怎么避免浪费？

按用户行为概率触发，并监控命中率。

3. 为什么要分层错误边界？

可把故障限制在局部模块，避免整页雪崩。

4. 如何处理路由切换竞态？

取消旧请求并用最新导航上下文校验写入。
