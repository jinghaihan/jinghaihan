## 概念定义

- 本质定义：媒体与图形节点关注图片资源策略和图形渲染选型。
- 解决问题：在清晰度、带宽、渲染开销之间做平衡。
- 边界说明：`iframe` 通信边界在独立节点“iframe 嵌入与跨域通信”中展开。

## 核心机制拆解

### 1) 机制/流程

1. 图片场景优先用 `srcset` + `sizes` 做分辨率和尺寸自适应。
2. 图形场景按更新频率和交互复杂度选择 Canvas 或 SVG。
3. 通过清晰度、资源体积和渲染耗时做回归验证。

### 2) 关键条件与触发点

- 条件 A：同图单资源覆盖所有设备，常导致高分屏糊图或低端机浪费带宽。
- 条件 B：复杂图形选型错误，会导致渲染开销和维护成本持续上升。

### 3) 结果与影响

- 对性能/稳定性/可维护性的影响：选型正确时可同时兼顾体验与成本；选型错误时会出现卡顿、糊图和调优困难。

## 相似概念对比

| 维度 | Canvas | SVG | 结论 |
|---|---|---|---|
| 渲染模型 | 像素级绘制 | 对象化矢量节点 | 模型不同 |
| 更新成本 | 重绘成本与帧率相关 | DOM（Document Object Model）节点操作成本相关 | 场景不同 |
| 交互支持 | 需自行命中检测 | 原生节点事件支持较好 | 交互复杂度不同 |
| 适用场景 | 高频重绘、粒子、游戏 | 图表、流程图、图标缩放 | 按更新模式选型 |

## 代码

```html
<img
  src="/images/avatar-320.jpg"
  srcset="/images/avatar-320.jpg 320w, /images/avatar-640.jpg 640w, /images/avatar-960.jpg 960w"
  sizes="(max-width: 480px) 120px, 200px"
  alt="用户头像"
/>
```

### 示例说明

- 这段代码解决什么问题：让浏览器根据视口和像素密度选择更合适的图片资源。
- 关键语句或参数含义：`srcset` 提供候选图，`sizes` 告诉浏览器不同条件下的展示宽度。
- 常见错误写法：只写 `srcset` 不写 `sizes`，导致浏览器无法按真实展示尺寸选择最优资源。

## 真实场景示例

- 场景描述：商品列表在高分屏糊图，且移动端流量成本高。
- 问题表现：同一图片在不同设备加载体积不合理，清晰度不稳定。
- 处理方案：为核心图片增加 `srcset + sizes`，图表按更新模式在 Canvas 和 SVG 之间选型。
- 验证方式：看图片平均下载体积、清晰度反馈、渲染耗时和滚动流畅度。

## 常见误区

- 误区：`srcset` 只服务高分屏。
  - 正解：它本质是按设备条件做资源选择，核心目标是“清晰度 + 带宽”平衡。
- 误区：Canvas 永远比 SVG 快。
  - 正解：两者性能取决于场景，高频重绘偏 Canvas，结构化图形偏 SVG。

## 关联知识点

- 依赖：HTML5 能力总览。
- 影响：加载与阻塞模型、渲染优化。
- 对比：Canvas 与 SVG。
- 组合：iframe 嵌入与跨域通信。

## 相关面试题

1. img 的 srcset 属性的作⽤？
   - 标准答案：结论是 `srcset` 用于给出多份候选图片，让浏览器按设备像素密度和展示宽度选择资源。机制上它通常与 `sizes` 配合使用，由浏览器自动挑选最合适文件。场景上可在保证清晰度的同时降低图片下载体积。
   - 考察点：是否理解响应式图片的核心价值是“清晰度与带宽平衡”。

2. Canvas 和 SVG 的区别
   - 标准答案：结论是 Canvas 偏像素绘制，SVG（Scalable Vector Graphics）偏对象化矢量描述。机制上 Canvas 常通过重绘更新，SVG 通过节点和属性更新。场景上高频动态效果偏 Canvas，结构化图形和缩放展示偏 SVG。
   - 考察点：是否能把选型结论和渲染机制绑定。
