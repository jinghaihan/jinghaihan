## 概念定位

资源提示节点通过 preconnect、dns-prefetch、preload、prefetch 提前布局网络与资源优先级。

正确使用可降低关键资源等待，错误使用会抢带宽。

## 工作机制

1. dns-prefetch 提前做域名解析。
2. preconnect 提前建立 TCP/TLS 连接。
3. preload 提前拉取即将使用的关键资源。
4. prefetch 在空闲时准备未来导航资源。
5. 浏览器依据优先级调度实际下载顺序。

## 关键知识点

### 必须掌握

- preload 用于当前导航关键资源，prefetch 用于未来可能资源。
- hint 过多会造成优先级竞争。
- 跨域 preload 需关注 CORS 配置。
- 预连接对跨域关键资源收益明显。
- hint 需要结合真实命中率评估。
- 资源提示应随页面结构迭代更新。

### 选型与取舍

- hint 越多命中机会越高，但网络拥塞风险上升。
- 保守 hint 风险低，但首屏潜在收益减少。
- 全站统一 hint 简单，但页面差异化不足。

## 对页面性能与稳定性的影响

- 节点稳定时，Knowledge Graph 主线 的网络与渲染链路更可预测，性能治理收益更稳定。
- 节点失稳时，问题会向后续阶段扩散，常表现为首屏变慢、交互卡顿或版本一致性风险。
- 优化动作应结合上下游联动验证，避免局部优化把成本转移到其他节点。

## 观测与排障

### 核心指标

- hint 命中率
- 关键资源等待时长
- 无效预加载比例
- 首屏渲染时间变化

### 排障顺序

1. 先确认关键资源是否被正确提示。
2. 检查提示类型是否匹配使用场景。
3. 检查是否存在高比例无效预取。
4. 对比加 hint 前后瀑布图。
5. 清理低收益 hint。

### 常见故障场景

- preload 了但资源未被使用。
- prefetch 抢占带宽影响当前页面。
- 跨域资源 hint 不生效。
- 提示过多导致关键资源反而变慢。

## 常见误区

- 把 hint 当越多越好。
- 混淆 preload 和 prefetch 语义。
- 不做命中效果回归。

## 题库深挖补充

### 高频对比与边界（基于题库）

- CDN（Content Delivery Network）核心目标是“就近访问 + 回源收敛”。
- GSLB（Global Server Load Balancing）负责把请求导向更优边缘节点。
- 边缘命中率高不等于用户体验好，还要看版本一致性与回源稳定性。
- stale-while-revalidate / stale-if-error 策略可提升可用性，但要防陈旧数据长期驻留。
- preconnect/dns-prefetch 是客户端侧资源提示，与 CDN 调度协同但不是同一层机制。
- 面试常问“CDN 为什么有时比源站还慢”，常见是节点异常或回源链路问题。

### 线上排障剧本

1. 按地区和运营商分桶看 TTFB 差异。
2. 检查边缘命中、回源、回源失败比例。
3. 检查缓存键（Vary）是否导致碎片化命中。
4. 检查发布后预热与失效是否完成。
5. 边缘异常时准备回源兜底与限流策略。

### 高频追问补充（附答案）

1. 为什么同一资源在不同地区体验差很多？

边缘节点健康、运营商路由和回源路径都可能不同。

2. 命中率高但仍有人报旧版本，怎么解释？

命中可能是旧入口或错误缓存键，命中率和一致性要一起看。

3. preconnect 和 CDN 是替代关系吗？

不是。preconnect 是客户端提前建连，CDN 是分发架构层。

## 进阶追问（补充）

1. 如果网络指标正常但用户仍感知慢，优先看哪一层？

优先看主线程任务和渲染阶段（Layout/Paint/Composite）是否拥塞，网络快不代表交互快。

2. 缓存策略和发布策略为什么必须一起设计？

缓存决定资源复用路径，发布决定资源版本切换；两者脱节会出现版本错配。

3. 如何验证一次链路优化不是偶然收益？

至少做对照组、分位数对比，并覆盖冷缓存与弱网样本。

## 案例追问加深（第三轮）

### 真实故障片段

- 故障表现：同一版本在不同地区首屏耗时差异明显，且刷新方式不同结果不同。
- 排查结论：网络链路、缓存路径和渲染调度叠加导致体验分化，单点优化无法彻底解决。
- 治理动作：建立链路分层观察面板，按导航类型、缓存命中层和渲染阶段联合分析。

### 追问补充

1. 为什么链路问题经常不是单点根因？

Knowledge Graph 本身是多阶段因果链，一个阶段的波动会在后续阶段被放大。

2. 如何快速缩小排障范围？

先固定触发方式和样本维度，再按网络、缓存、渲染三层逐步排除。

## 面试补充

### 30 秒说明

资源提示节点的核心是精确而克制：只提前真正关键且高概率使用的资源。

### 高频追问

1. preload 和 prefetch 最大区别？

preload 服务当前页面关键路径，prefetch 面向未来导航。

2. 什么时候用 preconnect？

跨域关键资源且建连成本显著时。

3. hint 为什么会伤害性能？

错误提示会抢占带宽，挤压真正关键资源。

4. hint 怎么验证收益？

看命中率、瀑布图和核心指标对照变化。
