## 概念定义

- 本质定义：该节点聚焦 CSS 特性的选择、组合和落地边界。
- 解决问题：解决“同样能实现效果，为什么线上表现差异很大”的工程问题。
- 边界说明：不替代具体框架样式体系，只回答 CSS 机制与选型。

## 核心机制拆解

### 1) 机制/流程

1. 先按目标分类：布局、动画、可访问性、响应式、工程化。
2. 再按浏览器渲染链路评估代价：样式计算、布局、绘制、合成。
3. 选择最小可维护方案，并明确兼容与降级策略。
4. 用性能指标和视觉回归验证收益。

### 2) 关键条件与触发点

- 条件 A：当页面出现抖动、卡顿或错层时，优先检查布局与层叠链路。
- 条件 B：当样式规模持续增长时，需要引入层级、变量和规范治理。

### 3) 结果与影响

- 对性能/稳定性/可维护性的影响：正确的特性选型可以同时降低渲染成本和维护成本，错误选型会放大样式耦合与线上回归。

## 相似概念对比

| 维度 | 偏视觉快速实现 | 偏机制可维护实现 | 结论 |
|---|---|---|---|
| 定义 | 先做效果再补治理 | 先做边界和约束再实现 | 长线项目更适合后者 |
| 触发时机 | 需求临时加急 | 持续迭代和多人协作 | 后者更稳 |
| 性能影响 | 易触发隐性开销 | 可预测可优化 | 机制优先 |
| 适用场景 | 活动页短周期 | 中长期业务系统 | 按周期选策略 |

## 代码

```css
.card-list {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(240px, 1fr));
  gap: 16px;
}

.card {
  transition: transform 160ms ease;
}

.card:hover {
  transform: translateY(-2px);
}
```

### 示例说明

- 这段代码解决什么问题：同时展示响应式网格和低成本动画。
- 关键语句或参数含义：`grid` 负责布局，`transform` 负责合成层友好动画。
- 常见错误写法：用 `top/left` 做高频动画导致布局抖动。

## 真实场景示例

- 场景描述：中后台页面样式快速堆积，新增需求越来越容易破坏旧页面。
- 问题表现：样式覆盖混乱、列表滚动掉帧、移动端展示不一致。
- 处理方案：按特性分类治理样式，统一层叠规则、动画策略和响应式断点。
- 验证方式：看 CLS、交互延迟、视觉回归结果和样式冲突告警数量。

## 相关面试题

1. CSS中的animation、transition、transform有什么区别？
- 标准答案：结论是 `animation` 适合关键帧序列，`transition` 适合状态切换过渡，`transform` 是具体变换属性。机制上 `transform` 常走合成阶段，`animation/transition` 负责时序控制。场景上交互动效优先 `transition + transform`，复杂分段动效再用 `animation`。
- 考察点：是否能把“时序控制”和“变换属性”区分开。

2. 怎么做移动端的样式适配？
- 标准答案：结论是要同时处理视口、单位和断点策略。机制上通过 `meta viewport`、`rem/vw`、媒体查询和弹性布局实现跨尺寸适配。场景上移动优先设计能显著降低多端返工。
- 考察点：是否能给出成体系的适配方案而非单点技巧。

3. 相邻的两个inline-block节点为什么会出现间距，该怎么解决？
- 标准答案：结论是间距来自 HTML 空白字符参与排版。机制上 `inline-block` 会像文本一样处理空格和换行。场景上可用移除空白、父级 `font-size: 0`、改用 `flex` 等方式解决。
- 考察点：是否知道问题来源于文本排版规则。

4. CSS Grid网格布局的基本概念和使用方法？
- 标准答案：结论是 Grid 是二维布局系统。机制上容器定义行列轨道和区域，子项按网格线或区域定位。场景上卡片矩阵、后台面板和复杂仪表盘布局优先考虑 Grid。
- 考察点：是否能从容器属性和子项属性两侧回答。

5. CSS3新增了哪些特性？
- 标准答案：结论是 CSS3 主要扩展了选择器、动画、变换、圆角阴影、渐变和媒体查询。机制上这些能力减少了对图片和脚本的依赖。场景上现代交互页面基本都依赖 CSS3 特性组合。
- 考察点：是否能按类别列举而不是只背两个属性名。

6. CSS实现动画的方式有哪些？
- 标准答案：结论是常见方式有 `transition`、`animation` 和脚本配合 `requestAnimationFrame`。机制上选择关键在于时序复杂度和可控性。场景上简单悬浮态用 `transition`，多阶段循环动画用 `animation`。
- 考察点：是否能按复杂度选择动画实现方式。

7. 什么是回流和重绘？如何减少回流和重绘？
- 标准答案：结论是回流是布局重算，重绘是像素重画。机制上回流通常成本更高且会连带重绘。场景上通过读写分离、批量更新、优先 `transform/opacity` 可降低渲染开销。
- 考察点：是否清楚两者触发条件与优化路径。

8. CSS选择器的优先级是怎样的？
- 标准答案：结论是优先级按 `!important`、内联样式、ID、类/属性/伪类、标签/伪元素逐级比较。机制上同权重再比较声明顺序。场景上大型项目应避免过高权重导致样式难以维护。
- 考察点：是否能准确说明权重比较规则。

9. 什么是BFC（块级格式化上下文）？
- 标准答案：结论是 BFC 是独立布局区域。机制上它可隔离浮动影响并避免外部 margin 折叠。场景上浮动清除、两栏布局和溢出控制常用创建 BFC 解决。
- 考察点：是否能给出创建条件和典型用途。

10. Flexbox布局的主要特性和使用方法？
- 标准答案：结论是 Flex 适合一维布局与弹性分配空间。机制上主轴与交叉轴配合 `justify-content/align-items/flex` 控制排列。场景上导航栏、工具条、列表项内部对齐是高频场景。
- 考察点：是否理解主轴模型和弹性因子。

11. CSS中的position属性有哪些值？
- 标准答案：结论是常用有 `static`、`relative`、`absolute`、`fixed`、`sticky`。机制上不同值决定元素是否脱离文档流和定位参照系。场景上吸顶、浮层、角标定位都依赖 position 体系。
- 考察点：是否能结合定位参照系讲清差异。

12. 如何实现元素的水平垂直居中？
- 标准答案：结论是可用 `flex`、`grid`、绝对定位 + transform 等方案。机制上不同方案依赖父容器布局模型。场景上未知尺寸元素优先 `flex/grid`，兼容旧布局再考虑绝对定位技巧。
- 考察点：是否能按场景给出稳定方案而非单一写法。

13. CSS盒模型是什么？
- 标准答案：结论是盒模型由内容、内边距、边框、外边距构成。机制上 `box-sizing` 决定宽高是否包含边框与内边距。场景上组件库普遍统一 `border-box` 以降低尺寸计算复杂度。
- 考察点：是否能说明 `content-box` 与 `border-box` 的差异。

14. CSS中的float属性及其清除浮动的方法？
- 标准答案：结论是 `float` 最初用于图文环绕，也被历史布局大量使用。机制上浮动元素脱离常规文档流会影响父容器高度计算。场景上清除浮动可用 `clearfix`、BFC 或 `overflow` 方案。
- 考察点：是否理解浮动带来的布局副作用。

15. CSS中的z-index属性如何工作？
- 标准答案：结论是 `z-index` 只在同一层叠上下文内比较层级。机制上新层叠上下文会隔离比较范围。场景上弹窗、遮罩和悬浮菜单错层问题通常不是数值不够大，而是上下文不一致。
- 考察点：是否能从层叠上下文角度解释错层问题。

16. CSS预处理器（Sass/Less）的优势是什么？
- 标准答案：结论是预处理器提升样式复用和组织能力。机制上变量、嵌套、混入和函数在编译期展开为标准 CSS。场景上中大型项目可降低重复代码，但要控制嵌套深度避免生成臃肿选择器。
- 考察点：是否能同时说明收益和滥用风险。

17. 如何实现响应式设计？
- 标准答案：结论是响应式设计依赖流式布局、弹性单位和媒体查询协作。机制上根据视口宽度和能力特征切换样式策略。场景上多终端产品应优先定义断点体系和组件级响应规则。
- 考察点：是否具备系统化响应式方案。

18. CSS中的伪类和伪元素有什么区别？
- 标准答案：结论是伪类描述元素状态，伪元素描述元素内部虚拟片段。机制上伪类常以 `:` 写状态选择，伪元素通常以 `::` 写内容片段。场景上交互态用伪类，装饰内容用伪元素。
- 考察点：是否能区分“状态”与“结构片段”。

19. 什么是CSS Sprites？有什么优缺点？
- 标准答案：结论是 CSS Sprites 把多张小图合成一张图并通过背景定位显示。机制上可减少请求数量但增加维护复杂度。场景上图标数量固定且更新频率低时可用，现代项目更多转向 SVG 精灵或图标字体方案。
- 考察点：是否能说明其在现代工程中的适用边界。

20. CSS中的单位有哪些？
- 标准答案：结论是常见有绝对单位（如 `px`）和相对单位（如 `em/rem/vw/vh/%`）。机制上相对单位依赖父级、根字体或视口计算。场景上移动端与响应式设计应优先使用相对单位组合。
- 考察点：是否能按参照系分类说明单位差异。

21. 如何实现CSS三角形？
- 标准答案：结论是常用边框法实现三角形。机制上设置零宽高元素并利用边框颜色与透明边形成可见三角。场景上气泡箭头、提示框指示器是典型应用。
- 考察点：是否理解边框法背后的几何原理。

22. CSS中的层叠规则是什么？
- 标准答案：结论是层叠规则由来源、优先级、重要性和声明顺序共同决定。机制上浏览器按层叠算法逐步决策最终样式。场景上样式冲突排查必须沿层叠链路逐项定位。
- 考察点：是否能把层叠规则讲成可排障的方法。

23. 什么是CSS-in-JS？有什么优缺点？
- 标准答案：结论是 CSS-in-JS 把样式定义放进 JavaScript 运行或编译流程。机制上可实现作用域隔离和主题动态化，但会增加构建或运行时成本。场景上组件库主题化场景收益明显，静态内容站点不一定划算。
- 考察点：是否能从工程成本角度客观评价 CSS-in-JS。

24. 如何实现CSS动画的性能优化？
- 标准答案：结论是核心是选择低成本属性并控制动画数量。机制上优先 `transform/opacity`，减少布局触发和主线程阻塞。场景上列表批量动画和滚动联动动画要重点做帧率监控。
- 考察点：是否能给出可执行的动画优化策略。

25. CSS中的@media查询有哪些常用的特性？
- 标准答案：结论是常用特性包括 `width`、`height`、`orientation`、`prefers-color-scheme`、`prefers-reduced-motion`。机制上媒体特征表达设备能力与用户偏好。场景上响应式布局和无障碍偏好适配都依赖这些查询。
- 考察点：是否能覆盖尺寸与偏好两类媒体特征。

26. 如何实现CSS的垂直居中？
- 标准答案：结论是垂直居中优先 `flex` 的 `align-items: center` 或 Grid 的 `place-items: center`。机制上它们直接在布局模型层解决对齐问题。场景上复杂嵌套布局不建议再依赖旧式 line-height 或负 margin 技巧。
- 考察点：是否能按现代布局方案回答。

27. CSS中的calc()函数有什么用途？
- 标准答案：结论是 `calc()` 用于在样式中做运行时长度计算。机制上可混合百分比与绝对单位得到精确尺寸。场景上“全宽减去侧栏宽度”“视口减去固定头部高度”是常见用法。
- 考察点：是否知道 `calc()` 的组合计算价值。

28. 什么是CSS变量（自定义属性）？
- 标准答案：结论是 CSS 变量用于在样式层定义可复用、可级联、可动态修改的值。机制上通过 `--token` 声明、`var()` 读取。场景上主题切换、设计令牌和暗色模式适配非常依赖自定义属性。
- 考察点：是否理解变量的级联作用域机制。

29. CSS中的object-fit属性有什么作用？
- 标准答案：结论是 `object-fit` 控制替换元素内容在容器内的缩放裁剪方式。机制上常用值有 `cover`、`contain`、`fill`。场景上图片卡片和视频封面可避免拉伸变形。
- 考察点：是否能解释 `cover` 与 `contain` 的差异。

30. CSS中的clip-path属性如何使用？
- 标准答案：结论是 `clip-path` 用于定义元素可见裁剪区域。机制上支持基本图形与路径语法。场景上异形头像、波浪分割和创意卡片过渡常用该属性。
- 考察点：是否理解裁剪区域与真实布局区域的区别。

31. 什么是CSS的层叠上下文（Stacking Context）？
- 标准答案：结论是层叠上下文是元素层级比较的独立环境。机制上某些属性会创建新上下文并隔离 `z-index` 比较。场景上排查弹层被遮挡时要先看上下文结构而非盲目增大数值。
- 考察点：是否能列举创建上下文的常见条件。

32. CSS中的contain属性有什么作用？
- 标准答案：结论是 `contain` 用于声明元素对子树渲染影响的隔离边界。机制上浏览器可据此减少样式、布局和绘制的外溢计算。场景上大型组件或虚拟列表容器可用来控制渲染成本。
- 考察点：是否理解 `contain` 对渲染范围裁剪的意义。

33. CSS中的aspect-ratio属性如何使用？
- 标准答案：结论是 `aspect-ratio` 用于声明盒子的宽高比约束。机制上在只有一边尺寸时可自动推导另一边。场景上媒体卡片、占位骨架和响应式缩略图可减少额外占位技巧代码。
- 考察点：是否能说明其替代 padding hack 的价值。

34. CSS中的scroll-behavior属性有什么作用？
- 标准答案：结论是 `scroll-behavior` 控制滚动跳转是否平滑。机制上 `smooth` 会让锚点跳转和脚本滚动呈现过渡动画。场景上目录导航和回到顶部交互可提升体验，但要考虑减少动画偏好用户。
- 考察点：是否能把体验提升与可访问性偏好一起回答。

35. 什么是CSS的逻辑属性（Logical Properties）？
- 标准答案：结论是逻辑属性按书写方向表达布局，不直接绑定物理方向。机制上如 `margin-inline-start` 可自动适配 LTR/RTL。场景上多语言国际化页面可显著减少方向适配分支样式。
- 考察点：是否理解逻辑方向与物理方向的差异。

36. CSS中的scroll-snap属性如何实现滚动吸附效果？
- 标准答案：结论是通过容器 `scroll-snap-type` 与子项 `scroll-snap-align` 配合实现。机制上滚动停止时浏览器自动对齐到最近吸附点。场景上轮播、卡片流和分屏阅读常使用滚动吸附增强交互。
- 考察点：是否能说明容器与子项属性的配合关系。

37. CSS中的@supports规则如何使用？
- 标准答案：结论是 `@supports` 用于做特性检测分支样式。机制上浏览器在解析时判断属性或值是否支持，再决定是否应用规则块。场景上新特性渐进增强时可避免低版本浏览器样式崩坏。
- 考察点：是否理解能力检测与降级策略的关系。

38. 什么是CSS的容器查询（Container Queries）？
- 标准答案：结论是容器查询让样式根据父容器尺寸而不是视口尺寸变化。机制上通过 `container-type` 和 `@container` 定义组件级响应规则。场景上可复用组件在不同布局槽位下能自适应而无需页面级断点干预。
- 考察点：是否能说明容器查询相对媒体查询的优势。

39. CSS中的color()函数和新的颜色空间如何使用？
- 标准答案：结论是新颜色函数支持更广色域和更精确插值表达。机制上可在支持的浏览器中使用更现代颜色空间并配合回退值。场景上品牌色管理和渐变过渡可得到更一致的视觉效果。
- 考察点：是否了解新颜色空间需要兼容策略。

40. CSS中的:has()伪类选择器如何使用？
- 标准答案：结论是 `:has()` 可按后代或相邻关系反向匹配父级元素。机制上它把“父选择器”能力引入 CSS 选择器体系。场景上表单错误高亮、卡片状态联动可减少额外脚本和标记。
- 考察点：是否理解 `:has()` 的选择器能力边界和性能影响。

41. CSS中的@layer规则如何管理样式层叠？
- 标准答案：结论是 `@layer` 用于显式定义样式层级顺序。机制上层级优先级可控，减少文件加载顺序带来的不确定覆盖。场景上设计系统、第三方库样式和业务样式混用时非常有效。
- 考察点：是否能说明 @layer 对大型项目样式治理价值。

42. CSS中的subgrid如何实现嵌套网格布局？
