## 概念定义

- 本质定义：语义化与可访问性是用正确标签表达结构和交互意图，让人和机器都能稳定理解页面。
- 解决问题：提升可读性、可维护性、可检索性和可访问体验。
- 边界说明：它不替代视觉设计与业务逻辑，但会影响后续样式和交互成本。

## 核心机制拆解

### 1) 机制/流程

1. 使用语义标签组织结构，如 `header`、`main`、`article`、`footer`。
2. 使用正确交互标签，跳转用 `a`，提交/操作用 `button`。
3. 对表单控件补齐 `label` 关联，提高可点击区域和读屏可理解性。
4. 确保标题层级和强调语义正确，如 `h1`、`strong`、`em`。
5. 通过键盘导航和读屏回归验证语义正确性。

### 2) 关键条件与触发点

- 条件 A：`label` 未正确关联表单控件，会导致可用性和可访问性下降。
- 条件 B：标题层级混乱会影响内容结构理解和检索效果。

### 3) 结果与影响

- 对性能/稳定性/可维护性的影响：语义结构清晰可减少补丁代码，降低长期维护成本。

## 相似概念对比

| 维度 | `strong` / `em` | `b` / `i` | 结论 |
|---|---|---|---|
| 定义 | 语义强调 | 视觉样式 | 语义优先用前者 |
| 触发时机 | 需要强调语义时 | 仅需样式变化时 | 不能互相替代 |
| 性能影响 | 结构语义清晰便于维护 | 易演化为样式补丁 | 语义正确维护更稳 |
| 适用场景 | 正文重点和语气强调 | 纯视觉微调 | 先语义后样式 |

## 代码

```html
<article>
  <h1>账户设置</h1>
  <label for="nickname">昵称</label>
  <input id="nickname" type="text" />
  <p>请阅读并 <strong>确认</strong> 规则。</p>
  <button type="submit">保存</button>
</article>
```

### 示例说明

- 这段代码解决什么问题：用语义标签明确结构层级，并保证表单控件可访问。
- 关键语句或参数含义：`label for` 与 `id` 建立可点击关联，`strong` 提供语义强调。
- 常见错误写法：用 `div` 模拟按钮、表单项缺失 `label`、把语义强调都写成 `b`。

## 真实场景示例

- 场景描述：表单页转化率下降，键盘用户反馈难以操作。
- 问题表现：点字段文字无法聚焦输入框，读屏播报不完整。
- 处理方案：补齐 `label`、校正标题层级、替换伪交互标签。
- 验证方式：键盘路径回归、读屏抽样、表单完成率对比。

## 常见误区

- 误区：语义化只影响 SEO（Search Engine Optimization）。
  - 正解：语义化还直接影响可访问性、组件复用和维护成本。
- 误区：ARIA（Accessible Rich Internet Applications）可以完全替代原生语义标签。
  - 正解：ARIA 是补充，优先选择原生语义标签。

## 关联知识点

- 依赖：文档声明与 head 配置。
- 影响：媒体与图形、渐进增强与优雅降级。
- 对比：语义标签与纯视觉标签。
- 组合：表单可用性、标题结构、检索友好。

## 相关面试题

1. 对 HTML 语义化的理解
   - 标准答案：结论是语义化就是“用正确标签表达正确内容结构”。机制上它让浏览器、检索系统和辅助技术更准确理解页面。场景上语义结构清晰时团队维护和组件复用成本更低。
   - 考察点：是否能把语义化讲到工程收益层。

2. 行内元素有哪些？块级元素有哪些？ 空(void)元素有那些？
   - 标准答案：
     行内元素常见有：`a`、`span`、`strong`、`em`、`b`、`i`、`img`、`input`、`label`、`select`、`textarea`。
     块级元素常见有：`div`、`p`、`h1` 到 `h6`、`ul`、`ol`、`li`、`section`、`article`、`header`、`footer`、`main`。
     空元素常见有：`br`、`hr`、`img`、`input`、`link`、`meta`、`source`、`track`、`area`、`base`、`col`、`embed`、`wbr`。
     机制上三类元素的默认布局行为和闭合规则不同。场景上分类错误会造成结构语义混乱和样式异常。
   - 考察点：是否具备标签行为分类能力和基础准确度。

3. title 与 h1 的区别、b 与 strong 的区别、i 与 em 的区别？
   - 标准答案：结论是 `title` 是文档标题，`h1` 是正文主标题；`strong` / `em` 有语义强调，`b` / `i` 更偏视觉表现。机制上前者作用于页面结构语义，后者作用于文本表达层。场景上页面通常同时需要 `title` 和 `h1`，强调语义优先用 `strong` / `em`。
   - 考察点：是否能区分结构语义和视觉语义边界。

4. label 的作用是什么？如何使用？
   - 标准答案：结论是 `label` 用于绑定表单控件并扩大点击热区。机制上点击 `label` 会把焦点转移到关联控件。场景上可以使用 `for + id` 绑定，也可以包裹控件写法。
   - 考察点：是否掌握表单可访问基础规则。
