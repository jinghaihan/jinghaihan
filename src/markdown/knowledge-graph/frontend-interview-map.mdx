## 知识体系概览

前端知识图谱用于把“页面从需求到上线再到稳定运行”的核心能力串成可复习、可落地的完整体系。

## 能力边界

- 覆盖范围：语言基础、页面结构、样式系统、网络链路、浏览器机制、性能优化、工程化、Vue 与 React 业务开发能力。
- 非覆盖范围：后端架构设计、数据库建模、移动原生开发、算法竞赛类题型。

## 核心子主题

1. JavaScript 体系：语言机制、异步模型与手写能力，是理解运行时行为的底座。
2. HTML 体系：文档结构、加载时序与语义能力，决定页面输入层质量。
3. CSS 体系：布局、层叠与渲染性能治理，决定界面稳定性与表现力。
4. 计算机网络体系：请求链路、协议语义与跨域策略，决定数据交互可靠性。
5. 浏览器原理体系：解析、渲染、缓存与安全机制，解释线上问题的根因。
6. 性能优化体系：指标、策略与验证闭环，保障体验与成本平衡。
7. 前端工程化体系：构建、发布与协作流程，支撑团队规模化交付。
8. Vue 体系：以响应式和组件协作为中心的项目开发能力。
9. React 体系：以状态建模和渲染调度为核心的项目开发能力。

## 推荐学习路径

- 入门顺序：HTML 体系 -> CSS 体系 -> JavaScript 体系 -> 计算机网络体系 -> 浏览器原理体系 -> 前端工程化体系 -> 性能优化体系 -> Vue 体系 -> React 体系。
- 面试优先顺序：JavaScript 体系 -> HTML 体系 -> CSS 体系 -> 计算机网络体系 -> 浏览器原理体系 -> 性能优化体系 -> 前端工程化体系 -> Vue 体系 -> React 体系。
- 项目落地顺序：先打稳页面与交互基础（HTML、CSS、JavaScript）-> 再治理请求与渲染链路（网络、浏览器）-> 再做工程化与性能闭环 -> 最后按业务栈深入 Vue 或 React。

## 常见混淆边界

- 框架能力 vs 浏览器原理：前者解决“如何组织业务代码”，后者解释“代码为何这样执行与渲染”。
- 性能优化 vs 工程化：前者关注运行结果和指标改进，后者关注构建交付与协作效率。
- 语义化质量 vs 视觉效果：前者服务结构可读性与可访问性，后者服务界面呈现，不可互相替代。

## 相关知识岛

- JavaScript 体系：承担语言机制与异步调度基础。
- HTML 体系：承担文档结构与资源加载入口。
- CSS 体系：承担布局与视觉表现规则。
- 浏览器原理体系：承担解析、渲染、缓存和安全的底层机制。
- 计算机网络体系：承担请求与响应链路的可靠传输。
- 性能优化体系：承担指标驱动的优化与验证。
- 前端工程化体系：承担构建发布与团队协作流程。
- Vue 体系：承担 Vue 技术栈的组件与状态设计。
- React 体系：承担 React 技术栈的组件与渲染策略。
