## 知识体系概览

CSS 体系解决“页面如何稳定展示、可维护扩展、可跨设备适配”的问题，是前端体验质量的基础层。

## 能力边界

- 覆盖范围：层叠优先级、可见性、盒模型、布局、定位与层叠、BFC、渲染性能、工程化、图形与像素适配。
- 非覆盖范围：JavaScript 运行时调度、网络协议细节、框架状态管理。

## 核心子主题

1. 层叠与优先级：选择器权重、继承、伪类与伪元素。
2. display 与可见性：布局参与、占位行为与可交互差异。
3. 盒模型与文本排版：尺寸计算、行高、溢出与替换元素。
4. 响应式单位与媒体查询：`px` / `em` / `rem` / `vw` / `vh` 与移动端适配。
5. 经典布局与 Flex：两栏、三栏、居中、九宫格与 `flex: 1`。
6. 定位与层叠上下文：`position`、`z-index`、层叠顺序与失效场景。
7. 浮动、clear 与 BFC：浮动清理、`clear` 原理、margin 重叠处理。
8. 重排、重绘与渲染抖动：`translate`、回流成本与性能抖动治理。
9. 动画与渲染时序：`transition`、`animation`、`requestAnimationFrame`。
10. 资源引入与图像策略：`link` / `@import`、图片格式、Sprites。
11. CSS 工程化与性能治理：预处理器、PostCSS（PostCSS）、Atomic CSS。
12. CSS 图形实战：三角形、扇形、圆、梯形与自适应正方形。
13. 发丝线与 1px 问题：`0.5px`、小字号与多端像素对齐。

## 推荐学习路径

- 入门顺序：层叠与优先级 -> display 与可见性 -> 盒模型与文本排版 -> 响应式单位与媒体查询 -> 经典布局与 Flex -> 定位与层叠上下文 -> 浮动、clear 与 BFC -> 动画与渲染时序 -> 工程化与性能治理
- 面试优先顺序：层叠与优先级 -> 经典布局与 Flex -> 定位与层叠上下文 -> 浮动、clear 与 BFC -> 盒模型与文本排版 -> 工程化与性能治理
- 项目落地顺序：先保证布局稳定与可见性正确 -> 再做响应式适配 -> 最后做渲染与工程化优化

## 常见混淆边界

- `display: none` vs `visibility: hidden`：是否占位、是否可交互、是否参与渲染树不同。
- `position` vs `transform: translate`：前者改布局，后者更多触发复合。
- `em` vs `rem`：前者相对父级，后者相对根元素。
- 伪类 vs 伪元素：前者描述状态，后者创建虚拟内容片段。

## 相关知识岛

- HTML 体系：结构语义与 CSS 选择器匹配效率直接相关。
- 浏览器原理体系：样式计算、布局、绘制链路直接影响 CSS 优化效果。
- 性能优化体系：回流重绘、资源体积、关键渲染路径与 CSS 强耦合。
