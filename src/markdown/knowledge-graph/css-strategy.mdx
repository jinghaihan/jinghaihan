## 概念定位

CSS 策略节点关注关键样式加载、非关键样式延后和样式组织成本控制。

它直接影响渲染阻塞、首屏稳定性和后续样式重算成本。

## 工作机制

1. 提取关键 CSS 保障首屏快速可见。
2. 非关键样式异步加载并延后应用。
3. 控制选择器复杂度和样式体积。
4. 按模块拆分样式并保持加载顺序稳定。
5. 结合缓存策略提升回访命中。

## 关键知识点

### 必须掌握

- CSS 默认阻塞渲染，关键样式必须优先。
- 样式体积和选择器复杂度都会影响渲染成本。
- 主题与业务样式要分层，避免覆盖冲突。
- 字体加载策略会影响首屏稳定性。
- 样式分包需要结合页面访问路径。
- 样式策略要与组件化约束协同。

### 选型与取舍

- 更多关键样式内联提速首屏，但增大 HTML。
- 样式过度拆分可降首包，但请求调度更复杂。
- 统一样式库维护方便，但可能引入冗余。

## 对页面性能与稳定性的影响

- 节点稳定时，Knowledge Graph 主线 的网络与渲染链路更可预测，性能治理收益更稳定。
- 节点失稳时，问题会向后续阶段扩散，常表现为首屏变慢、交互卡顿或版本一致性风险。
- 优化动作应结合上下游联动验证，避免局部优化把成本转移到其他节点。

## 观测与排障

### 核心指标

- 关键 CSS 体积
- 样式阻塞时长
- 样式重算耗时
- 首屏布局稳定性

### 排障顺序

1. 先确认首屏是否被样式阻塞。
2. 检查关键 CSS 是否缺失或过大。
3. 检查选择器复杂度和覆盖冲突。
4. 检查字体加载策略是否引发布局跳动。
5. 回归不同网络和设备。

### 常见故障场景

- 首屏白屏时间长。
- 样式加载后页面明显跳动。
- 主题切换冲突频发。
- 样式重算导致交互卡顿。

## 常见误区

- 忽视关键 CSS，全部依赖完整样式包。
- 选择器写法过深过重。
- 样式加载顺序不受控。

## 题库深挖补充

### 高频对比与边界（基于题库）

- HTML 解析、CSS 计算、渲染树构建是连续链路，任何一段阻塞都会延后首屏可见。
- async/defer/module 的选择会改变脚本执行时机与阻塞模式。
- 关键 CSS 提取能提速，但内联过多会抬高 HTML 体积。
- render tree 不包含所有 DOM 节点，display:none 等节点不会参与渲染树。
- 语义结构不仅影响 SEO，也影响后续样式和脚本维护成本。
- 面试常问“为什么首屏慢”，根因往往是关键路径资源组织不合理。

### 线上排障剧本

1. 先画出首屏关键资源链（HTML、CSS、关键脚本、字体）。
2. 检查阻塞脚本和阻塞样式是否可延后或拆分。
3. 检查首屏 DOM 规模与无效节点比例。
4. 检查样式计算热点是否由复杂选择器触发。
5. 回归冷缓存、弱网和中低端设备。

### 高频追问补充（附答案）

1. defer 一定优于 async 吗？

不一定。defer 更可控，async 更激进，取决于依赖关系。

2. 为什么说 HTML 结构会影响性能？

结构决定资源发现和解析路径，也决定渲染树规模。

3. render tree 和 DOM tree 的关系？

render tree 基于 DOM + CSSOM 生成，只包含参与渲染的节点。

## 进阶追问（补充）

1. 如果网络指标正常但用户仍感知慢，优先看哪一层？

优先看主线程任务和渲染阶段（Layout/Paint/Composite）是否拥塞，网络快不代表交互快。

2. 缓存策略和发布策略为什么必须一起设计？

缓存决定资源复用路径，发布决定资源版本切换；两者脱节会出现版本错配。

3. 如何验证一次链路优化不是偶然收益？

至少做对照组、分位数对比，并覆盖冷缓存与弱网样本。

## 案例追问加深（第三轮）

### 真实故障片段

- 故障表现：同一版本在不同地区首屏耗时差异明显，且刷新方式不同结果不同。
- 排查结论：网络链路、缓存路径和渲染调度叠加导致体验分化，单点优化无法彻底解决。
- 治理动作：建立链路分层观察面板，按导航类型、缓存命中层和渲染阶段联合分析。

### 追问补充

1. 为什么链路问题经常不是单点根因？

Knowledge Graph 本身是多阶段因果链，一个阶段的波动会在后续阶段被放大。

2. 如何快速缩小排障范围？

先固定触发方式和样本维度，再按网络、缓存、渲染三层逐步排除。

## 面试补充

### 30 秒说明

CSS 策略的核心是让“首屏可见”和“长期维护”同时成立，关键在分层、优先级和体积治理。

### 高频追问

1. 为什么 CSS 会阻塞渲染？

渲染树构建依赖样式计算，样式未就绪会推迟绘制。

2. 关键 CSS 应该多大合适？

以覆盖首屏必要样式为准，避免过度内联。

3. 字体策略为什么影响 CLS？

字体替换会导致文字重排，影响布局稳定。

4. 样式分包怎么做更合理？

按页面路径和复用关系拆分，控制请求与冗余平衡。
