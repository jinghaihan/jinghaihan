## 代码

```vue
<script setup lang="ts">
const props = defineProps<{ modelValue: string }>()
const emit = defineEmits<{ 'update:modelValue': [value: string] }>()
</script>

<template>
  <input
    :value="props.modelValue"
    @input="emit('update:modelValue', ($event.target as HTMLInputElement).value)"
  >
</template>
```

### 示例说明

- 这段代码解决什么问题：给自定义组件实现 `v-model` 协议。
- 关键语句或参数含义：`modelValue + update:modelValue` 是 Vue 3 默认双向绑定约定。
- 常见错误写法：只改子组件内部状态不发 `update:modelValue`，父值不会同步。

## 相关面试题

1. slot 是什么？有什么作用？原理是什么？
- 标准答案：结论是 slot 是父组件向子组件传递模板片段的机制。机制上子组件定义插槽位，父组件在渲染时注入内容。场景上卡片、弹窗、表格列渲染都常用具名插槽提高复用性。
- 考察点：是否能按阶段讲清核心流程、关键数据结构和边界条件。

2. v-if、v-show、v-html 的原理
- 标准答案：结论是 `v-if` 控制创建销毁，`v-show` 只切换 `display`，`v-html` 直接设置 `innerHTML`。机制上三者影响渲染成本和安全边界不同。场景上频繁切换用 `v-show`，低频条件块用 `v-if`，`v-html` 必须做 XSS 防护。
- 考察点：是否能按阶段讲清核心流程、关键数据结构和边界条件。

3. v-if 和 v-show 的区别
- 标准答案：结论是 `v-if` 首次惰性、切换销毁重建；`v-show` 首次渲染后仅改样式显示。机制上前者运行期开销在切换，后者开销在首次渲染。场景上高频切换选 `v-show`，权限/条件块选 `v-if`。
- 考察点：是否能从定义、触发时机、性能影响和适用场景四个维度做对比。

4. v-model 是如何实现的，语法糖实际是什么？
- 标准答案：结论是 `v-model` 是“值 + 事件”语法糖。机制上在原生输入框等价于 `:value` + `@input`，在 Vue 3 组件上等价于 `modelValue` + `update:modelValue`。场景上自定义组件要显式声明这对协议才能双向绑定。

- 标准答案：结论是可以。机制上子组件接收 `modelValue`，变更时 `emit("update:modelValue", value)`，父组件通过 `v-model` 自动绑定。场景上表单组件库普遍使用该约定保证统一体验。

12. 为什么不建议用 index 作为 key?
- 标准答案：结论是索引会随插入删除变化，导致节点身份漂移。机制上这会让 Diff 误复用旧节点，引发状态错位和动画异常。场景上只有静态、无重排列表才勉强可用 index。
